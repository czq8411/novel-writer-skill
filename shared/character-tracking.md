# 共享模块：人物出场追踪与一致性保障

> **按需加载**：涉及人物管理、OOC检测、群像调度时加载
> **全局规则**：遵循主控文件第二章核心铁律
> **版本**：v21.1

---

## 一、系统概述

人物出场追踪与一致性保障系统用于追踪所有人物的出场情况、状态变化，检测和预防人物OOC（Out of Character），确保长篇创作中人物形象的连贯性和一致性。

**核心目标**：
- 追踪所有人物出场情况
- 检测人物行为是否符合设定
- 预警长期未出场人物
- 防止人物OOC

## 二、人物档案系统

### 2.1 人物档案结构

```yaml
人物档案_模板:
  基础信息:
    姓名: [string]

    年龄: [number]

    人物类型: [主角/核心配角/次要配角/龙套]
  
  性格设定:
    核心性格: [string]  # 3-5个关键词
    性格描述: [string]  # 详细描述
    行为模式: [list]   # 典型行为
    语言特征: [string]  # 说话方式
    禁忌: [list]       # 绝不会做的事
  
  能力设定:
    境界/等级: [string]
    主要能力: [list]
    武器/法宝: [list]
    弱点: [list]
  
  关系网络:
    与主角关系: [string]
    关系变化: [list]   # [{章节, 变化描述}]
    盟友: [list]
    敌人: [list]
  
  状态追踪:
    当前状态:
      位置: [string]
      健康: [enum]
      情绪: [enum]
      境界: [string]
    状态历史: [list]   # [{章节, 状态}]
  
  出场记录:
    总出场次数: [number]
    最近出场: [章节号]
    出场频率: [number]  # 平均每N章出场一次
    关键出场: [list]   # 重要章节
  
  人物弧线:
    初始状态: [string]
    目标: [string]
    当前阶段: [string]
    预计结局: [string]
```

### 2.2 人物分类管理

| 类型 | 定义 | 管理策略 | 追踪精度 |
|------|------|---------|:-------:|
| **主角** | 故事核心人物 | 每章追踪，详细记录 | ⭐⭐⭐⭐⭐ |
| **核心配角** | 重要配角（>20次出场） | 每章追踪，重点记录 | ⭐⭐⭐⭐ |
| **次要配角** | 一般配角（5-20次出场） | 出场时记录，定期汇总 | ⭐⭐⭐ |
| **龙套** | 一次性角色（<5次出场） | 仅记录出场章节 | ⭐⭐ |

### 2.3 人物档案存储

**存储位置**：
- L1核心层：主角+核心配角档案
- L2缓存层：次要配角档案（活跃中）
- L3归档层：次要配角档案（长期未出场）+ 龙套档案

## 四、OOC检测系统

### 4.1 OOC检测维度

| 维度 | 检测内容 | 权重 |
|------|---------|:---:|
| **性格一致性** | 行为是否符合核心性格 | 35% |
| **能力一致性** | 能力使用是否符合设定 | 25% |
| **关系一致性** | 对待他人是否符合关系设定 | 20% |
| **状态一致性** | 当前状态是否连续 | 20% |

### 4.2 OOC检测流程

```
检测到人物出场
    ↓
提取人物当前行为
    ↓
查询人物档案
    ↓
逐项对比：
  - 性格对比
  - 能力对比
  - 关系对比
  - 状态对比
    ↓
发现异常？
    /        \
  是          否
  ↓           ↓
OOC评分    通过
  ↓
评分>阈值？
    /        \
  是          否
  ↓           ↓
标记OOC    记录轻微偏离
  ↓
生成修正建议
```

### 4.3 OOC评分标准

```
OOC评分 = Σ(维度偏离度 × 权重)

偏离度计算：
- 完全符合：0分
- 轻微偏离（有解释）：1分
- 明显偏离：2分
- 严重偏离：3分

评分等级：
- 0-1分：✅ 正常
- 1-2分：⚠️ 轻微OOC，建议调整
- 2-3分：🔶 明显OOC，需要修正
- >3分：🔴 严重OOC，必须修正
```

### 4.4 OOC修正建议

**修正策略**：

| OOC类型 | 修正策略 | 示例 |
|---------|---------|------|
| **性格偏离** | 增加心理描写解释行为动机 | "他平时不会这样做，但这次是为了救人" |
| **能力超限** | 增加代价或限制 | "使用这招后，他灵力耗尽" |
| **关系错误** | 调整对话语气或行为 | 将亲密行为改为礼貌距离 |
| **状态跳跃** | 补充过渡描写 | 增加伤势恢复过程 |

**修正建议格式**：
```markdown
## OOC检测报告

### 人物：林凡
### 当前评分：2.5/5 🔶 明显OOC

#### 检测到的OOC行为
1. **性格偏离**（权重35%，得分2分）
   - 行为：主动挑衅强敌
   - 设定：性格谨慎，从不主动惹事
   - 偏离度：明显
   - 建议：增加解释"他知道对方是杀害师父的凶手，才主动挑衅"

2. **能力超限**（权重25%，得分1分）
   - 行为：使用高级法术
   - 设定：当前境界无法使用
   - 偏离度：轻微
   - 建议：增加代价"强行使用导致经脉受损"

#### 修正方案
**方案A：增加解释**（推荐）
在行为前增加心理描写，解释为什么这次会违背性格

**方案B：修改行为**
改为更符合性格的行为，如"暗中观察，寻找机会"

#### 修正后预期评分
- 方案A：0.8/5 ✅ 正常
- 方案B：0.5/5 ✅ 正常
```

## 五、群像调度系统

### 5.1 群像调度规则

| 规则 | 说明 | 检查点 |
|------|------|--------|
| **核心配角保底** | 每个核心配角每10章至少出场1次 | 每10章检查 |
| **单章人数限制** | 单章新出场人物≤3人，总出场≤8人 | 每章检查 |
| **主次分明** | 主角戏份占比≥40%，核心配角≥20% | 每10章统计 |
| **避免扎堆** | 避免连续多章大量人物出场 | 实时监控 |

### 5.2 群像调度建议


## 六、集成到写作流程

### 6.1 与模式C的集成

在[mode-c-writing.md](mode-c-writing.md)中新增：

```markdown
### 人物一致性检查（v21.1新增）

#### 出场前检查
1. 查询人物档案
2. 确认当前状态
3. 检查是否长期未出场（>10章）
4. 如是，显示人物状态摘要

#### 出场中检查（实时）
1. 每段对话后检查语言特征
2. 每个行为后检查性格一致性
3. 能力使用后检查能力范围

#### 出场后检查
1. 更新出场记录
2. 更新状态变化
3. 更新关系变化
4. 生成OOC检测报告

#### 每10章汇总
1. 生成出场统计报表
2. 检查长期未出场人物
3. 生成群像调度建议
```

```markdown
## 人物档案：林凡

### 基础信息
- 姓名：林凡

- 年龄：18岁
- 身份：青云宗外门弟子→内门弟子

### 性格设定
- 核心性格：谨慎、坚韧、重情义
- 行为模式：谋定而后动，不轻易冒险
- 语言特征：简洁直接，不废话
- 禁忌：绝不会背叛朋友、绝不会滥杀无辜

### 能力设定
- 境界：筑基初期
- 主要能力：青云剑法、神秘玉佩
- 弱点：灵力储备不足、实战经验欠缺

### 关系网络
- 与林月：师妹，好感度85，关系亲密
- 与赵天霸：宿敌，仇恨度90
- 与苏婉清：师姐，好感度70

### 状态追踪
- 当前位置：青云宗内门
- 健康状态：良好
- 情绪状态：坚定

### 出场记录
- 总出场：50次
- 最近出场：第50章
- 出场频率：每章必出
```

### OOC自检清单

```markdown
## OOC自检清单

### 性格一致性
- [ ] 行为是否符合核心性格？
- [ ] 是否有外部因素导致性格变化？
- [ ] 变化是否有铺垫？

### 能力一致性
- [ ] 使用的能力是否符合当前境界？
- [ ] 是否使用了未提及的新能力？
- [ ] 能力使用是否有代价？

### 关系一致性
- [ ] 对待他人的态度是否符合关系设定？
- [ ] 关系变化是否有过渡？
- [ ] 是否有突兀的关系转变？

### 状态一致性
- [ ] 当前状态是否与前章连续？
- [ ] 伤势/情绪变化是否合理？
- [ ] 位置变化是否有说明？
```
