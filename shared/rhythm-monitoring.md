# 共享模块：实时节奏监控与干预系统

> **按需加载**：涉及节奏控制、爽点管理、情绪曲线时加载
> **全局规则**：遵循主控文件第二章核心铁律
> **版本**：v21.1

---

## 一、系统概述

实时节奏监控与干预系统用于实时监控小说创作节奏，包括爽点密度、情绪曲线、钩子强度等关键指标，及时发现节奏问题并提供干预建议，确保小说保持吸引力和阅读流畅性。

**核心目标**：
- 实时监控节奏指标
- 预警节奏失控风险
- 提供智能干预建议
- 维持最佳阅读体验

## 二、节奏指标体系

### 2.1 核心指标

| 指标 | 定义 | 理想范围 | 预警阈值 |
|------|------|:-------:|:-------:|
| **爽点密度** | 每章爽点数量 | 0.3-0.8个/章 | <0.2或>1.0 |
| **钩子强度** | 章末悬念吸引力 | 7-10分 | <6分 |
| **情绪强度** | 章节情绪峰值 | 6-9分 | <5或>9.5 |
| **冲突密度** | 每章冲突事件数 | 1-3个/章 | 0或>4 |
| **节奏变化率** | 相邻章节奏差异 | 20-50% | <10%或>80% |

### 2.2 辅助指标

| 指标 | 定义 | 用途 |
|------|------|------|
| **对话占比** | 对话字数/总字数 | 检测是否过于沉闷 |
| **场景切换次数** | 每章场景变化 | 检测是否过于跳跃 |
| **心理描写占比** | 心理描写字数/总字数 | 检测是否过于拖沓 |
| **新信息密度** | 每章新设定/情节数量 | 检测信息过载 |

### 2.3 指标权重

```yaml
节奏综合评分:
  爽点密度: 30%
  钩子强度: 25%
  情绪强度: 20%
  冲突密度: 15%
  节奏变化率: 10%
```

## 三、实时监控机制

### 3.1 监控触发点

| 触发时机 | 监控类型 | 说明 |
|---------|---------|------|
| **正文生成中** | 实时监 | 每完成500字检查一次 |
| **正文生成后** | 完整监 | 全章节奏分析 |
| **每5章完成** | 趋势监 | 分析节奏趋势 |
| **每30章完成** | 深度监 | 阶段性节奏复盘 |

### 3.2 实时监控流程

```
正文生成中
    ↓
每完成500字
    ↓
快速检测：
  - 当前爽点强度
  - 当前情绪强度
  - 对话占比
    ↓
指标异常？
    /        \
  是          否
  ↓           ↓
立即提示    继续生成
  ↓
建议调整
```

### 3.3 监控仪表盘

```markdown
## 实时节奏仪表盘（第[X]章，已生成1500字）

### 当前指标
| 指标 | 当前值 | 目标范围 | 状态 |
|------|:------:|:-------:|:----:|
| 爽点强度 | 6/10 | 7-10 | 🟡 偏低 |
| 情绪强度 | 5/10 | 6-9 | 🟡 偏低 |
| 对话占比 | 45% | 30-50% | 🟢 正常 |
| 节奏变化 | +15% | 20-50% | 🟡 偏平 |

### 预警
⚠️ 当前爽点强度偏低，建议在剩余1500字中增加一个小高潮

### 建议
1. 增加一个冲突事件，提升爽点强度
2. 加快情节推进速度
3. 减少过渡性描写
```

## 四、节奏预警系统

### 4.1 预警级别

| 级别 | 触发条件 | 响应方式 | 干预强度 |
|:----:|---------|---------|:-------:|
| 🟢 提示 | 指标偏离目标10-20% | 显示提示 | 可选 |
| 🟡 警告 | 指标偏离目标20-40% | 强制提示+建议 | 建议 |
| 🔴 严重 | 指标偏离目标>40% | 强制提示+修正方案 | 强烈建议 |
| 🚨 紧急 | 连续3章严重偏离 | 暂停生成+强制修正 | 必须 |

### 4.2 常见预警场景

#### 场景1：爽点不足

```markdown
## 节奏预警：爽点不足

### 当前状态
- 最近3章爽点密度：0.1个/章
- 目标密度：0.5个/章
- 偏离度：-80% 🔴 严重

### 风险
- 读者可能感到无聊
- 追读率可能下降
- 完读率可能降低

### 建议方案
**方案A：插入小爽点**（推荐）
- 在当前章增加一个打脸情节
- 预计增加爽点强度：+3分
- 实施难度：低

**方案B：加快节奏**
- 压缩过渡情节，提前进入高潮
- 预计提升节奏变化率：+30%
- 实施难度：中

**方案C：调整细纲**
- 修改后续章节规划，增加爽点密度
- 长期解决方案
- 实施难度：高
```

#### 场景2：情绪疲劳

```markdown
## 节奏预警：情绪疲劳

### 当前状态
- 最近5章情绪强度：8.5/10
- 持续时间：5章
- 风险等级：🔴 严重

### 风险
- 读者情绪疲劳
- 高潮失去冲击力
- 需要舒缓过渡

### 建议方案
**方案A：插入舒缓章节**（推荐）
- 下一章设计为日常/温馨情节
- 情绪强度降至5-6分
- 让读者情绪得到舒缓

**方案B：降低冲突强度**
- 当前章减少冲突描写
- 增加轻松对话
- 平衡情绪强度
```

#### 场景3：节奏拖沓

```markdown
## 节奏预警：节奏拖沓

### 当前状态
- 本章字数：3500字
- 核心事件：1个
- 信息增量：低
- 风险等级：🟡 警告

### 风险
- 读者感到"水字数"
- 跳过阅读
- 弃书风险

### 建议方案
**方案A：精简描写**
- 删减冗余场景描写
- 压缩过渡段落
- 目标字数：2500字

**方案B：增加信息密度**
- 增加新设定或伏笔
- 推进支线剧情
- 提升阅读价值
```

### 4.3 预警响应机制

```
触发预警
    ↓
判断预警级别
    ↓
🟢提示 → 显示提示信息，用户可选择忽略
🟡警告 → 强制显示建议，用户确认后继续
🔴严重 → 生成修正方案，建议立即调整
🚨紧急 → 暂停生成，必须调整后才继续
```

## 五、智能干预系统

### 5.1 干预策略

| 问题类型 | 干预策略 | 实施时机 |
|---------|---------|---------|
| **爽点不足** | 插入小高潮/加快节奏 | 实时 |
| **情绪疲劳** | 插入舒缓情节/降低强度 | 章末或下章 |
| **节奏拖沓** | 精简内容/增加信息 | 生成后 |
| **钩子无力** | 强化悬念/增加危机 | 章末 |
| **冲突不足** | 增加对抗/制造矛盾 | 实时 |

### 5.2 自动修正机制

**触发条件**：
- 严重预警（🔴）
- 用户选择"自动修正"

**修正流程**：
```
识别问题类型
    ↓
查询修正策略库
    ↓
生成修正方案
    ↓
用户确认
    ↓
执行修正
    ↓
验证效果
    ↓
重新评分
```

### 5.3 修正策略库

```yaml
修正策略:
  爽点不足:
    轻度:
      - 增加配角惊叹反应
      - 强化主角能力展示
    中度:
      - 插入小冲突事件
      - 增加意外转折
    重度:
      - 提前安排打脸情节
      - 增加金手指展示
  
  情绪疲劳:
    策略:
      - 插入日常温馨场景
      - 增加搞笑对话
      - 减少冲突密度
  
  节奏拖沓:
    策略:
      - 删减冗余描写
      - 压缩过渡段落
      - 加快情节推进
  
  钩子无力:
    策略:
      - 增加悬念揭示
      - 制造新危机
      - 强化反转效果
```

## 六、节奏趋势分析

### 6.1 趋势报告

```markdown
## 节奏趋势报告（第31-60章）

### 整体趋势
| 指标 | 平均值 | 趋势 | 评价 |
|------|:------:|:----:|------|
| 爽点密度 | 0.45/章 | ↗️ 上升 | 良好 |
| 钩子强度 | 7.2/10 | → 平稳 | 正常 |
| 情绪强度 | 6.8/10 | ↘️ 下降 | 需关注 |
| 冲突密度 | 1.8/章 | → 平稳 | 正常 |

### 问题识别
⚠️ 情绪强度呈下降趋势，最近10章平均6.2分

### 建议
1. 第61-70章适当增加情绪高潮
2. 安排一个重要转折点，提升情绪强度
3. 避免连续平淡章节

### 预测
按当前趋势，第70章可能进入情绪低谷，建议提前布局高潮
```
