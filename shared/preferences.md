## Memory分级存储策略（v22.0新增）

> **核心目的**：解决长篇创作中Memory容量受限、早期设定丢失的问题，通过分级存储实现智能管理

### 三级存储架构

| 级别 | 名称 | 存储位置 | 容量 | 更新策略 | 用途 |
|:---:|------|---------|:---:|---------|------|
| **L1** | 核心存储 | MCP Memory永久字段 | 无限制 | 仅在大纲/核心设定变更时更新 | 世界观底层规则、人物核心设定、核心锚点 |
| **L2** | 近期存储 | MCP Memory活跃字段 | 最近100章 | 每章更新，旧数据自动归档 | 章节进度、人物状态、活跃伏笔、风格指纹 |
| **L3** | 临时存储 | 章节级上下文 | 单章 | 每章重建 | 当前章细纲约束、AI味检测状态、写作模式 |

### 存储策略详解

#### L1：核心存储（永久有效）

> 存储内容变更即触发Memory更新，仅在大纲修改时更新

```json
{
  "core_memory": {
    "project_name": "《XXX》",
    "core_anchors": {
      "world_rules": { /* 世界观底层规则 */ },
      "character_cores": { /* 人物核心设定 */ },
      "plot_essentials": { /* 核心情节要素 */ }
    },
    "version": "1.0",
    "last_update": "2026-06-22T00:00:00"
  }
}
```

**更新触发**：
- 模式B大纲完成/修改 → 更新
- 模式E诊断发现核心设定冲突 → 更新
- 每50章回顾 → 确认后更新
- 用户输入"更新核心设定" → 手动更新

#### L2：近期存储（最近100章）

> 滚动存储，超过100章的数据自动归档到外部文件

```json
{
  "recent_memory": {
    "current_chapter": 42,
    "character_states": {
      "主角": { "realm": "金丹初期", "status": "active", "last_appearance": 42 },
      "苏婉儿": { "realm": "元婴中期", "status": "missing", "last_appearance": 38 }
    },
    "active_foreshadowings": ["F12", "F15", "F18"],
    "chapter_archive": {
      "range": "第1-42章",
      "archive_to": "./记忆库.md"
    },
    "style_fingerprint": {
      "avg_chapter_length": 3850,
      "dialogue_ratio": 0.35,
      "sensory_profile": { "visual": 60, "auditory": 20, "tactile": 15, "olfactory": 5 }
    }
  }
}
```

**归档机制**：
| 触发条件 | 处理方式 | 说明 |
|---------|---------|------|
| 章节数超过100章 | 自动归档最早50章到外部文件 | 保持Memory中只有最近100章数据 |
| 每50章 | 生成阶段性设定快照 | 与knowledge-graph.md中的快照系统联动 |
| 用户输入"清理记忆" | 手动归档 | 确认后执行归档 |

**外部归档文件格式**（写入 `./记忆库.md`）：
```markdown
## 记忆库归档（第1-50章）

### 人物发展轨迹
| 角色 | 出场章节 | 初始状态 | 第50章状态 | 关键变化节点 |
|------|:---:|:-------:|:---------:|------------|
| 主角 | 第1章 | 练气前期 | 筑基后期 | 第5章突破练气 → 第20章突破筑基 → 第45章突破筑基后期 |

### 已完成伏笔
| 编号 | 内容 | 埋设 | 回收 | 质量 |
|------|------|:---:|:---:|:---:|
| F01-F08 | <详情见快照文件> | 第1-20章 | 第10-48章 | ✅ 全部自然回收 |

### 活跃设定（持续有效）
- 修炼体系：练气→筑基→金丹→元婴（第1-50章有效）
- 主角身份：青云宗外门弟子→内门弟子
- 核心冲突：林家灭门真相
```

#### L3：临时存储（单章级）

> 每章重建，不持久化

```json
{
  "chapter_temp": {
    "chapter_num": 42,
    "chapter_title": "金丹初成",
    "outline_constraint": { /* 当前章细纲约束 */ },
    "ai_flavor_check": { /* AI味检测状态 */ },
    "writing_mode": "standard",
    "sections_completed": ["开头", "中段", "高潮", "结尾"],
    "current_word_count": 3850,
    "consistency_check_result": null
  }
}
```

### 智能压缩策略

| 压缩类型 | 执行时机 | 方法 | 效果 |
|---------|---------|------|------|
| **相似合并** | L2归档时 | 合并相似的人物状态记录（如连续5章主角都在修炼，合并为一条） | 减少30-50%存储 |
| **冗余删除** | 每章完成后 | 删除已确认的冗余信息（如已完成的小阶段目标） | 减少10-20%存储 |
| **关键提取** | 归档时 | 从早期章节中提取关键设定和变化节点 | 保留100%关键信息 |
| **快照替代** | 阶段归档时 | 用快照索引替代原始数据（详见knowledge-graph.md§7） | 减少80%早期数据存储 |

## 核心金句归档系统（v22.3新增）

> 解决金句前后呼应的「结构化存储」缺失问题。每章正文完成后自动提取潜在金句，存入Memory，供后续章节的「金句呼应场景模板」调用。

### 金句归档格式

```json
{
  "golden_lines": {
    "project_name": "《XXX》",
    "lines": [
      {
        "id": "GL01",
        "text": "<<金句全文（不超过30字）>>",
        "source": "<<出现场景（谁说的/在哪看到的）>>",
        "chapter": 3,
        "emotion_type": "治愈/燃/虐/爽",
        "emotion_intensity": 7,
        "planned_echo_chapters": [35, 70, 100],
        "echo_types": ["角色重述", "冲突反用", "终章回声"],
        "is_echoed": false
      }
    ]
  }
}
```

### 金句识别规则

| 识别场景 | 示例 | 自动归档 |
|---------|------|:--------:|
| 角色说出有记忆点的台词 | 「动物园不是关动物的笼子」 | ✅ 自动建议归档 |
| 主角的自我认知转折 | 「我以前没看懂这句话，现在好像看懂了一点点」 | ✅ 自动建议归档 |
| 旁白性的主题揭示 | 「你送的不是饭。是文明还在运转的证据」 | ✅ 自动建议归档 |
| 普通对话 | 「今天天气真好」 | ❌ 不归档 |

### 金句呼应调度

每次正文创作到「金句呼应场景模板」触发章节时，自动从Memory调取对应金句：

```markdown
【金句呼应调度】第35章需要呼应GL01
- 金句原文：<<动物园不是关动物的笼子，是关希望的笼子>>
- 来源：第3章·爷爷的记事本
- 建议呼应用法：
  第35章对应情节：陆远拒绝将动物园卖给开发商
  呼应方式：陆远在合同上签字前，想起了记事本上的那句话
  建议措辞：「他拿起笔的时候，想起了记事本上那句话。然后放下了笔。」
```

### 归档触发时机

| 时机 | 行为 |
|:----|------|
| 每章正文优化完成 | 自动扫描本章内容，识别潜在金句，生成归档建议 |
| 用户输入「记住这句」 | 强制将当前句加入金句归档 |
| 用户输入「金句列表」 | 展示当前项目所有归档金句及其呼应状态 |

## 偏好更新规则

| 时机 | 行为 |
|------|------|
| 每次模式A灵感构思完成 | 静默更新题材偏好、风格倾向 |
| 每次模式C章节完成 | 更新lastChapter、lastPosition、projectPath到Memory |
| 用户说"记住我的偏好" | 保存当前所有配置 |
| 用户说"忘记XX偏好" | 清除指定维度偏好 |
| 用户说"重置偏好" | 清空所有偏好数据 |
| 一部长篇创作完成 | 将作品信息追加到activeProjects |

## 偏好加载规则

- **新会话启动**：自动从Memory读取偏好，检测activeProjects中未完成项目
- **有未完成项目**：主动提示"检测到您正在创作《XXX》，当前进度第X章，是否继续？"
- **有偏好无项目**：显示个性化欢迎"欢迎回来！您的偏好：XX题材，XX风格"
- **无偏好**：进入标准新用户引导流程

## 中断续写智能检测

每次模式C完成章节后，自动在Memory中记录进度锚点。新会话启动时自动检测并提示续写。

**进度锚点结构**：
```json
{
  "resumePoint": {
    "projectName": "《XXX》",
    "lastChapter": 42,
    "lastPosition": "第42章结尾",
    "lastScene": "主角刚突破金丹期，正在稳固境界",
    "nextChapterPlan": "第43章：宗门来人",
    "projectPath": "./",
    "lastUpdateTime": "2026-05-12T14:30:00",
    "pendingHooks": ["宗门长老的真实身份", "秘境钥匙的下落"],
    "characterStates": {
      "主角": "金丹初期，灵力不稳",
      "苏婉儿": "元婴中期，在秘境中失踪"
    }
  }
}
```

**检测流程**：
1. 新会话启动 → 读取Memory中的resumePoint
2. 若存在且lastUpdateTime在30天内 → 主动提示续写
3. 若存在但超过30天 → 提示"检测到较早的创作记录《XXX》（最后更新：X天前），是否继续？"
4. 若不存在 → 正常启动

**续写提示格式**：
```markdown
📖 检测到未完成的创作

| 项目 | 详情 |
|------|------|
| 作品名 | 《XXX》 |
| 当前进度 | 第42章（共计划80章） |
| 上次更新 | 2026-05-12（3天前） |
| 上次结尾 | 主角刚突破金丹期，正在稳固境界 |
| 下一章 | 第43章：宗门来人 |
| 待处理钩子 | 宗门长老的真实身份、秘境钥匙的下落 |

输入 `继续` 从断点续写，或输入其他指令开始新任务。
```

## 创作里程碑系统

长篇创作缺乏阶段性正反馈，容易中途放弃。里程碑系统在关键节点提供成就感和数据回顾。

| 类别 | 里程碑 | 触发条件 |
|------|--------|---------|
| **字数** | 1万/5万/10万/50万/100万字 | 累计字数达标 |
| **章节** | 第10/50/100/500章 | 完成对应章节 |
| **连续更新** | 连续7/30/100天 | 连续创作天数达标 |
| **质量** | 首次爽点密度达标/首次追读率>20%/首次0毒点章节 | 质量指标达标 |
| **剧情** | 第一卷完成/主线转折/大高潮完成 | 剧情节点达成 |

**里程碑输出格式**：
```markdown
🎉 里程碑达成：10万字！

## 创作数据回顾
| 指标 | 数值 |
|------|------|
| 创作天数 | 45天 |
| 平均日更 | 2222字 |
| 爽点总数 | 23个 |
| 角色出场率 | 主角100%、苏婉儿87%、萧天65% |
| 伏笔回收率 | 78%（18/23） |
| 最长连续更新 | 12天 |

## 下一里程碑
- 50万字（预计还需180天）
- 第100章（当前第42章，预计还需58章）
```

里程碑达成记录追加到Memory的milestones数组，跨会话保持。

### 快照生成时机

| 时机 | 快照类型 | 内容 |
|------|---------|------|
| 每章完成 | 完整快照 | 包含上述全部字段 |
| 用户主动暂停 | 完整快照 | 同上，额外记录暂停原因 |
| 模式切换 | 轻量快照 | 仅记录当前模式、章节、最后操作 |
| 会话结束 | 完整快照 | 同上，额外记录会话摘要 |

### 快照恢复流程

1. 新会话启动 → 读取最新contextSnapshot
2. 若快照存在且snapshotTime在30天内 → 展示续写提示（含上一章摘要+下一章计划）
3. AI加载快照中的characterStates、pendingHooks、styleProfile
4. AI基于nextChapterPlan开始创作，确保无缝衔接
5. 创作完成后更新快照
