# 共享模块：自我诊断与提示词优化

> 本模块赋予 novel-writer-skill 自我进化能力。当诊断遗漏问题或用户反馈未被覆盖的缺陷时，自动分析根因并提出提示词优化方案，经用户确认后执行修改。

---

## 一、触发场景

### 场景A：诊断后自检（模式E触发）

每次完成14维诊断后，自动执行以下自检：

| 自检项 | 触发条件 | 说明 |
|--------|---------|------|
| 全维度满分自检 | 任一维度评分=10/10 | 满分可能意味着检测规则不够严格，需审视规则是否过于宽松 |
| 全维度零问题自检 | 所有维度P0/P1问题数=0 | 对于长篇网文几乎不可能，需审视检测阈值是否合理 |
| 阶段匹配自检 | 当前处于引入期(1-10章)但"世界观展开"评分>8 | 引入期世界观展开评分过高可能意味着信息倾倒 |
| 矛盾自检 | "爽点质量"评分>8但"前期铺垫"评分<5 | 铺垫不足不可能有高质量爽点，检测逻辑可能矛盾 |

### 场景B：用户反馈触发（模式C/E触发）

当用户指出以下类型问题时触发：

| 用户反馈类型 | 触发关键词 | 说明 |
|------------|-----------|------|
| 诊断遗漏 | "为什么没诊断出""没发现""漏了" | 用户发现的问题未被诊断系统捕获 |
| 提示词不足 | "不够""太简单""敷衍""不详细" | 现有提示词指导深度不够 |
| 规则不合理 | "不合理""不应该""有问题" | 检测规则本身存在逻辑缺陷 |
| 新需求 | "能不能""是否可以""需要增加" | 用户提出新的检测需求 |

### 场景C：创作中自检（模式C触发）

在正文创作过程中，当AI发现以下情况时主动触发：

| 自检项 | 触发条件 |
|--------|---------|
| 提示词冲突 | 两个子模块的指导建议相互矛盾 |
| 提示词过时 | 提示词引用的网文套路/模板已不符合当前市场趋势 |
| 覆盖盲区 | 当前创作场景在现有提示词中找不到对应指导 |

## 二、自我诊断流程

```
用户反馈/自检触发
       │
       ▼
┌──────────────────┐
│ 1. 问题定位       │ → 确定问题属于哪个模式/子模块/检测维度
└──────┬───────────┘
       ▼
┌──────────────────┐
│ 2. 根因分析       │ → 分析是规则缺失/阈值不当/逻辑缺陷/覆盖盲区
└──────┬───────────┘
       ▼
┌──────────────────┐
│ 3. 方案生成       │ → 生成具体的提示词修改方案（含修改前后对比）
└──────┬───────────┘
       ▼
┌──────────────────┐
│ 4. 用户确认       │ → 向用户展示方案，询问是否执行修改
└──────┬───────────┘
       ▼
     ┌─┴─┐
     │是 │ 否│
     └─┬─┘ └──→ 记录方案到改进建议库，下次可重新提出
       ▼
┌──────────────────┐
│ 5. 执行修改       │ → 修改对应文件，更新版本记录
└──────┬───────────┘
       ▼
┌──────────────────┐
│ 6. 验证生效       │ → 用修改后的规则重新诊断，确认问题可被捕获
└──────────────────┘
```

## 三、根因分析框架

### 3.1 问题分类

| 问题类型 | 代码 | 典型表现 | 分析方向 |
|---------|:---:|---------|---------|
| 规则缺失 | R01 | 某类问题完全没有对应检测规则 | 需新增检测维度或检测项 |
| 阈值不当 | R02 | 有规则但阈值过宽/过严导致漏报/误报 | 调整触发阈值 |
| 逻辑缺陷 | R03 | 规则存在但检测逻辑有矛盾或不可执行 | 修正检测逻辑 |
| 覆盖盲区 | R04 | 规则存在但未覆盖特定场景/阶段/题材 | 扩展规则适用范围 |
| 深度不足 | R05 | 规则存在但仅做表面检查，未深入质量层面 | 增加质量评分维度 |
| 联动缺失 | R06 | 多个维度各自为战，缺乏交叉验证 | 增加跨维度联动检测 |

### 3.2 根因定位表

| 用户反馈 | 可能根因 | 排查方向 |
|---------|---------|---------|
| "为什么没诊断出XX问题" | R01/R02 | 检查对应维度是否有检测规则，阈值是否合理 |
| "这个检测太表面了" | R05 | 检查检测规则是否仅检查"有无"而非"质量" |
| "XX和YY明显矛盾但没发现" | R06 | 检查是否有跨维度交叉验证规则 |
| "这个规则不合理" | R03 | 检查检测逻辑是否可执行，条件是否矛盾 |
| "XX场景没有被覆盖" | R04 | 检查规则适用范围是否遗漏当前场景 |

## 四、方案生成模板

### 4.1 新增检测规则模板

当发现规则缺失(R01)时，生成如下方案：

```markdown
## 自我诊断：检测规则缺失

### 发现的问题
[用户反馈的具体问题描述]

### 根因分析
- **问题类型**：R01-规则缺失
- **缺失位置**：[文件路径] → [章节/维度]
- **影响范围**：[哪些章节/场景会受影响]

### 建议方案
**新增检测维度/检测项**：

| 检测项 | 规则 | 级别 | 触发条件 |
|--------|------|:--:|---------|
| [检测项名称] | [具体检测规则] | [P0/P1/P2] | [何时触发] |

**修改文件**：[文件路径]
**修改位置**：[具体行/节]

### 修改前后对比
- **修改前**：[当前状态]
- **修改后**：[修改后状态]

### 预期效果
- 可捕获问题类型：[N]种
- 预计减少遗漏率：[X%]

---
是否执行此修改？
```

当发现深度不足(R05)时，生成如下方案：

```markdown

### 发现的问题
[当前仅检查"有无"，未检查"质量"]

- **问题类型**：R05-深度不足
- **当前检测**：仅检查[表面指标]
- **缺失检测**：未检查[深层质量指标]

### 建议方案

| 质量维度 | 评分标准(1-10) | 权重 |
|---------|---------------|:--:|
| [维度1] | [1分标准]...[10分标准] | [X%] |
| [维度2] | [1分标准]...[10分标准] | [X%] |

---

```


## 六、主动巡检机制

> 每完成一卷(约100章)或切换模式时，自动执行以下巡检。

### 6.1 巡检清单

| 巡检项 | 检查内容 | 频率 |
|--------|---------|:--:|
| 诊断覆盖率 | 最近10次用户反馈中，有多少是诊断系统未覆盖的 | 每卷 |
| 误报率 | 用户标记为"忽略"的P0/P1问题占比 | 每卷 |
| 规则时效性 | 检测规则中引用的网文套路是否过时 | 每卷 |
| 阈值合理性 | 各维度P0/P1/P2问题数量分布是否合理 | 每卷 |
| 提示词冲突 | 不同子模块对同一场景的指导是否存在矛盾 | 每卷 |
| 覆盖盲区 | 当前题材/阶段是否有未被任何提示词覆盖的创作场景 | 每卷 |

### 6.2 巡检报告格式

```markdown
## 技能自我巡检报告

### 巡检时间：YYYY-MM-DD
### 巡检范围：第X-Y章 / 模式[C/E]

### 诊断覆盖率
- 用户反馈问题总数：[N]
- 诊断系统覆盖：[M]（覆盖率：[X%]）
- 未覆盖问题：
  | # | 问题描述 | 缺失维度 | 建议 |
  |:--:|------|------|------|

### 阈值健康度
| 维度 | P0数 | P1数 | P2数 | 健康度 | 建议 |
|------|:---:|:---:|:---:|:---:|------|
| [维度] | [N] | [N] | [N] | [✅/⚠️] | [调整建议] |

### 改进建议
1. [建议1]
2. [建议2]
```

## 七、与各模式的集成

### 7.1 模式E集成

在 [mode-e-diagnostics.md](../modes/mode-e-diagnostics.md) 诊断流程末尾增加：

```
### 第十三步：技能自我诊断

诊断完成后，自动执行以下检查：
1. 全维度满分自检：是否有维度评分=10/10？→ 如有，审视规则是否过松
2. 全维度零问题自检：P0+P1=0？→ 如有，审视阈值是否合理
3. 阶段匹配自检：当前阶段与各维度评分是否匹配？
4. 矛盾自检：相关维度评分是否存在逻辑矛盾？

如发现问题，按本模块第二章流程生成优化方案并询问用户。
```

### 7.2 模式C集成

在 [mode-c-writing.md](../modes/mode-c-writing.md) 中增加：

```
### 用户反馈响应机制

当用户在创作过程中提出以下类型反馈时，触发技能自我诊断：
- "为什么没诊断出XX" → 触发场景B，分析诊断遗漏
- "这个XX不够详细" → 触发场景B，分析提示词深度
- "能不能增加XX检测" → 触发场景B，分析覆盖盲区

按 [shared/self-improvement.md](../shared/self-improvement.md) 流程生成优化方案并询问用户。
```
