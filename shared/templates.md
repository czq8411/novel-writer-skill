# 共享模块：网文创作模板市场

> 按需加载：模式A灵感构思、模式B大纲架构时，根据用户需求加载对应模板。
> 全局规则：遵循主控文件第二章核心铁律

---

## 一、退婚流模板

### 模板概述
开局被退婚羞辱→金手指觉醒→实力打脸→更高层次挑战。核心爽点是"当初你对我爱答不理，如今我让你高攀不起"。

### 标准章节分布

| 阶段 | 章节 | 核心事件 | 爽点 |
|------|------|---------|------|
| **羞辱** | 1-3章 | 主角被当众退婚，受尽嘲讽，家族蒙羞 | — |
| **觉醒** | 4-6章 | 意外获得传承/系统/前世记忆，金手指觉醒 | 金手指首次展示 |
| **修炼** | 7-15章 | 利用金手指快速成长，但隐藏实力 | 小试身手（暗中解决危机） |
| **打脸** | 16-20章 | 在某个公开场合展示实力，震惊众人 | 退婚对象后悔、家族震惊 |
| **离开** | 21-30章 | 离开新手村，进入更大世界 | 碾压曾经不可战胜的对手 |
| **崛起** | 31-50章 | 建立势力，在新世界站稳脚跟 | 收小弟、建宗门、夺资源 |

### 核心爽点节点
1. 第5章：金手指首次展示（读者知道主角变强了）
2. 第18章：退婚对象后悔（"我当初看走眼了"）
3. 第25章：碾压当年羞辱者（"三十年河东三十年河西"）
4. 第40章：家族承认错误（"是我们错了"）

### 常见陷阱与规避
| 陷阱 | 规避方法 |
|------|---------|
| 退婚理由牵强 | 给退婚方一个合理的利益驱动，不只是"嫌贫爱富" |
| 打脸过于容易 | 对手必须有真实实力，主角赢是因为成长而非对手弱 |
| 复仇后失去目标 | 提前规划退婚线之外的更大主线 |

## 二、废材逆袭模板

### 模板概述
被测出废材→意外获得传承→默默修炼→一鸣惊人。核心爽点是"所有人都以为我是废物，但我其实是天才"。

### 标准章节分布

| 阶段 | 章节 | 核心事件 | 爽点 |
|------|------|---------|------|
| **废材** | 1-3章 | 灵根测试/武魂觉醒垫底，被家族/宗门轻视 | — |
| **传承** | 4-6章 | 意外获得上古传承/隐藏血脉觉醒/戒指里的老爷爷 | 传承首次展示 |
| **隐藏** | 7-20章 | 表面废材，暗中修炼，扮猪吃虎 | 暗中解决危机，无人知晓 |
| **爆发** | 21-25章 | 宗门大比/学院考核中展现实力 | 一鸣惊人，所有人震惊 |
| **揭秘** | 26-40章 | 发现废材身份背后的秘密 | 身世揭秘，格局升级 |

### 核心爽点节点
1. 第5章：传承获得（读者知道主角要起飞了）
2. 第15章：第一次越级战斗（暗中击败强敌）
3. 第23章：宗门大比夺冠（公开打脸）
4. 第35章：身世揭秘（原来废材是最大天赋）

### 常见陷阱与规避
| 陷阱 | 规避方法 |
|------|---------|
| 隐藏期过长 | 隐藏期不超过20章，读者等不了太久 |
| 传承太强 | 传承必须有代价或限制，不能无脑变强 |
| 配角降智 | 轻视主角的人必须有合理理由，不能全员弱智 |

## 三、重生复仇模板

### 模板概述
前世惨死→重生回到关键节点→利用先知优势→改变命运。核心爽点是"这一次，我不会再犯同样的错误"。

### 标准章节分布

| 阶段 | 章节 | 核心事件 | 爽点 |
|------|------|---------|------|
| **惨死** | 1-3章 | 前世被背叛/杀害，死不瞑目 | — |
| **重生** | 4-6章 | 回到关键时间节点，带着前世记忆 | 重生时刻的震撼 |
| **布局** | 7-15章 | 利用先知优势，避开前世陷阱，暗中布局 | 第一次改变命运 |
| **清算** | 16-25章 | 逐一报复前世仇人 | 仇人发现主角变了 |
| **深挖** | 26-35章 | 发现前世悲剧背后有更大阴谋 | 格局升级 |
| **决战** | 36-50章 | 与幕后黑手最终对决 | 终极复仇 |

### 核心爽点节点
1. 第4章：重生时刻（"我回来了"）
2. 第10章：第一次改变命运（前世悲剧被避免）
3. 第20章：仇人震惊（"你怎么知道？"）
4. 第35章：幕后黑手浮出水面（"原来是你"）

### 常见陷阱与规避
| 陷阱 | 规避方法 |
|------|---------|
| 先知优势滥用 | 主角改变历史后，后续事件也应改变，不能刻舟求剑 |
| 复仇后空虚 | 提前规划复仇之外的人生目标 |
| 前世记忆太详细 | 记忆应有模糊和错误之处，增加真实感 |

## 四、系统流模板

### 模板概述
平凡人生→系统降临→任务驱动成长→系统背后的秘密。核心爽点是"别人靠天赋，我靠系统"。

### 标准章节分布

| 阶段 | 章节 | 核心事件 | 爽点 |
|------|------|---------|------|
| **降临** | 1-3章 | 平凡人生中突然出现系统面板 | 系统首次展示 |
| **新手** | 4-10章 | 完成初始任务，获得奖励，适应系统 | 第一次使用系统能力 |
| **成长** | 11-25章 | 系统发布任务，主角被动或主动完成 | 任务奖励的惊喜 |
| **升级** | 26-35章 | 系统解锁新功能，任务难度升级 | 系统升级的新能力 |
| **秘密** | 36-45章 | 发现系统不是凭空出现，有创造者和目的 | 系统背后的真相 |
| **超越** | 46-50章 | 主角不再依赖系统，或与系统创造者对决 | 超越系统 |

### 核心爽点节点
1. 第2章：系统面板首次展示
2. 第5章：第一次使用系统能力
3. 第26章：系统升级解锁新功能
4. 第40章：系统秘密揭示

### 常见陷阱与规避
| 陷阱 | 规避方法 |
|------|---------|
| 系统过于万能 | 系统必须有冷却/代价/限制，不能什么都能做 |
| 任务过于单调 | 任务类型应多样化，不只是"杀X只怪" |
| 主角沦为系统傀儡 | 主角必须有独立于系统的目标和人格 |

## 五、穿越种田模板

### 模板概述
穿越到古代/异世界→利用现代知识→逐步积累资源→改变世界。核心爽点是"用现代思维碾压古代社会"。

### 标准章节分布

| 阶段 | 章节 | 核心事件 | 爽点 |
|------|------|---------|------|
| **穿越** | 1-3章 | 穿越到古代/异世界，身份低微 | 穿越时刻的冲击 |
| **生存** | 4-15章 | 利用现代知识改善生活，逐步积累资源 | 第一次用现代知识解决问题 |
| **扩展** | 16-25章 | 从个人生存到影响村庄/城镇 | 权贵被打脸 |
| **碰撞** | 26-35章 | 引起权贵注意，卷入更大格局 | 现代思维vs古代制度 |
| **改变** | 36-50章 | 用现代思维改造世界，成为一方势力 | 建立自己的势力 |

### 核心爽点节点
1. 第3章：穿越时刻（"这是哪？"）
2. 第8章：第一次用现代知识解决问题
3. 第22章：权贵被打脸（"你一个平民怎么懂这些？"）
4. 第40章：建立自己的势力

### 常见陷阱与规避
| 陷阱 | 规避方法 |
|------|---------|
| 现代知识万能 | 主角的知识应有边界，不能什么都会 |
| 古代人降智 | 古代人只是知识体系不同，不是智商低 |
| 改变太快 | 社会变革需要时间，不能一蹴而就 |

## 七、模板参数化与自定义

> 用户可基于标准模板进行参数化调整，生成个性化变体。

### 模板参数表

| 参数 | 说明 | 默认值 | 可调范围 |
|------|------|--------|---------|
| `pace` | 节奏快慢 | 标准 | 快速/标准/慢热 |
| `cruelty` | 残酷程度 | 中等 | 轻松/中等/残酷/黑暗 |
| `romance` | 感情线比重 | 轻度 | 无/轻度/中度/重度 |
| `humor` | 幽默感比重 | 轻度 | 无/轻度/中度/重度 |
| `scale` | 世界观规模 | 中等 | 小/中等/大/史诗 |
| `cheats` | 金手指数量 | 1个 | 0/1/2/3 |
| `pov` | 视角模式 | 单主角 | 单主角/双主角/多视角 |

### 参数化示例

```
用户输入：用退婚流模板，但是节奏快一点，感情线重一点，世界观小一点

生成变体：
- 基础模板：退婚流
- pace=快速 → 压缩修炼期，第10章即打脸
- romance=重度 → 退婚对象不是单纯反派，有复杂情感纠葛
- scale=小 → 聚焦一个宗门内部，不扩展到大陆格局

调整后章节分布：
| 阶段 | 原章节 | 调整后 | 变化 |
|------|--------|--------|------|
| 羞辱 | 1-3章 | 1-2章 | 压缩 |
| 觉醒 | 4-6章 | 3-4章 | 压缩 |
| 修炼 | 7-15章 | 5-8章 | 大幅压缩 |
| 打脸 | 16-20章 | 9-12章 | 提前+增加情感戏 |
| 离开 | 21-30章 | 13-20章 | 压缩规模 |
| 崛起 | 31-50章 | 21-40章 | 聚焦宗门内部 |
```

### 模板组合

支持将多个模板的元素组合，创建混合模板：

| 组合方式 | 示例 | 说明 |
|---------|------|------|
| **主模板+副模板** | 退婚流（主）+系统流（副） | 退婚流框架，金手指为系统 |
| **阶段替换** | 废材逆袭开局+重生复仇中段 | 废材开局，中期发现前世记忆 |
| **元素融合** | 穿越种田+系统流 | 穿越者带系统种田 |

**组合示例**：
```
用户输入：废材逆袭+系统流组合

生成混合模板：
- 主角被测出废材（废材逆袭开局）
- 绝望中获得系统（系统流金手指）
- 系统任务驱动成长（系统流核心）
- 宗门大比一鸣惊人（废材逆袭高潮）
- 发现废材身份与系统来源有关（双线揭秘）

爽点叠加效应：
- 废材逆袭的"从零到巅峰" + 系统流的"任务奖励惊喜" = 双重爽感
- 注意：需控制金手指强度，避免系统过于万能导致废材逆袭失去张力
```

## 八、模板社区机制

> 用户可保存自定义模板、分享模板、导入他人模板，形成创作模板生态。

### 模板保存

用户完成参数化调整或组合后，可保存为自定义模板：

```
用户输入：保存当前模板为"快节奏退婚流"

系统响应：
✅ 模板已保存

模板信息：
- 名称：快节奏退婚流
- 基础：退婚流
- 参数：pace=快速, romance=重度, scale=小
- 章节分布：40章（原50章压缩）
- 保存时间：2026-05-12 14:30
- 模板ID：TPL-USER-001
```

### 模板管理

| 指令 | 行为 |
|------|------|
| `我的模板` | 列出所有已保存的自定义模板 |
| `查看模板 [名称/ID]` | 查看模板详情 |
| `使用模板 [名称/ID]` | 加载指定模板开始创作 |
| `删除模板 [名称/ID]` | 删除指定模板 |
| `导出模板 [名称/ID]` | 导出模板为可分享的JSON文件 |
| `导入模板` | 从JSON文件导入他人模板 |

### 模板导出格式

```json
{
  "templateMeta": {
    "name": "快节奏退婚流",
    "version": "1.0",
    "author": "用户自定义",

    "description": "基于退婚流模板的快节奏变体，适合追求紧凑剧情的创作",
    "tags": ["退婚流", "快节奏", "感情线", "小世界观"]
  },
  "baseTemplate": "退婚流",
  "parameters": {
    "pace": "快速",
    "romance": "重度",
    "scale": "小"
  },
  "chapterDistribution": [
    {"stage": "羞辱", "chapters": "1-2", "coreEvent": "主角被当众退婚"},
    {"stage": "觉醒", "chapters": "3-4", "coreEvent": "金手指觉醒"},
    {"stage": "修炼", "chapters": "5-8", "coreEvent": "快速成长+情感线发展"},
    {"stage": "打脸", "chapters": "9-12", "coreEvent": "公开打脸+情感纠葛"},
    {"stage": "离开", "chapters": "13-20", "coreEvent": "离开新手村"},

  ],
  "customNotes": "节奏紧凑，每5章一个爽点，感情线与主线交织推进"
}
```

### 模板推荐

基于用户创作历史和偏好，智能推荐模板：

| 用户特征 | 推荐模板 | 推荐理由 |
|---------|---------|---------|
| 首次创作 | 退婚流（标准） | 结构清晰，爽点明确，最适合新手 |
| 喜欢快节奏 | 系统流 | 任务驱动，节奏天然快 |
| 喜欢深度人物 | 重生复仇 | 前世今生的对比提供人物深度 |
| 喜欢轻松日常 | 穿越种田 | 节奏舒缓，侧重生活细节 |
| 有创作经验 | 模板组合 | 可尝试混合模板创造新意 |

### 模板迭代

用户可在使用模板过程中提出改进，系统记录迭代历史：

```
模板迭代记录：
v1.0 (2026-05-12)：初始版本，基于退婚流标准模板
v1.1 (2026-05-20)：根据用户反馈，增加第3章情感铺垫
v1.2 (2026-06-01)：优化打脸阶段，增加反派心理描写
```
