# AI写作伦理与原创性保障

> 按需加载：涉及原创性检测、AI参与度、内容合规时加载本文件。
> novel-writer的使命是**增强而非替代**人类创作者。

## 核心伦理原则

| 原则 | 说明 |
|------|------|
| **人类主导** | 所有创作决策的最终决定权属于人类作者，AI仅提供建议和选项 |
| **透明可溯** | AI生成的内容应可追溯，用户始终知道哪些内容由AI辅助生成 |
| **原创优先** | 鼓励创新和原创表达，避免生成与现有作品高度相似的内容 |
| **版权尊重** | 不主动模仿特定在世作家的风格，不生成侵犯他人版权的内容 |
| **隐私保护** | 用户创作内容仅用于当前会话，不用于模型训练（除非用户明确同意） |

## 原创性检测机制

### 内部相似度检测

| 检测维度 | 阈值 | 处理方式 |
|---------|------|---------|
| 句式重复 | 连续3句结构相同 | 自动变换句式 |
| 桥段重复 | 相似情节出现≥2次 | 标注并建议替代方案 |
| 形容词重复 | 同一形容词在500字内出现≥3次 | 自动替换同义词 |
| 开篇模式重复 | 连续3章开篇方式相同 | 提示变换开篇方式 |

### 外部相似度警示

| 警示级别 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 🟢 轻度 | 与某作品有1-2个相似元素 | 标注相似点，供用户判断 |
| 🟡 中度 | 与某作品有3-4个相似元素 | 标注相似点+提供差异化建议 |
| 🔴 高度 | 与某作品有≥5个相似元素或核心设定雷同 | 强烈建议修改+提供3个替代方向 |

**警示输出格式**：
```markdown
### ⚠️ 原创性提醒

| 相似元素 | 参考作品 | 相似程度 | 建议 |
|---------|---------|---------|------|
| [元素1] | [作品名] | [程度] | [差异化建议] |

**综合评估**：[原创性评分/10]
**建议**：[保持/微调/重构]
```

### 撞梗检测系统

> 自动检测与热门作品的相似创意，提供差异化建议，避免同质化。

#### 撞梗检测维度

| 检测维度 | 说明 | 示例 |
|---------|------|------|
| 核心设定 | 世界观、金手指、系统规则 | 穿越者获得系统 |
| 开局套路 | 第一章的核心事件 | 退婚羞辱、废柴觉醒 |
| 人物关系 | 主角与配角的关系模式 | 废柴主角+强势女主 |
| 情节结构 | 故事发展的节奏和节点 | 升级→打脸→新地图循环 |
| 爽点设计 | 让读者感到爽的具体方式 | 扮猪吃虎、越级挑战 |

#### 撞梗检测流程

```
用户输入创意 → 提取核心元素 → 对比热门作品库 → 计算相似度 → 输出警示+建议
```

#### 热门套路库（实时更新）

| 套路名称 | 热度 | 规避建议 | 差异化方向 |
|---------|------|---------|-----------|
| 退婚流开局 | ⭐⭐⭐⭐⭐ | 避免完全照搬"退婚羞辱" | 退婚方有苦衷/主角主动退婚 |
| 废柴觉醒 | ⭐⭐⭐⭐⭐ | 避免"戒指老爷爷"老套设定 | 金手指与主角身份深度绑定 |
| 系统流 | ⭐⭐⭐⭐⭐ | 避免"签到/抽奖/任务"三板斧 | 系统有隐藏目的/代价沉重 |
| 重生复仇 | ⭐⭐⭐⭐ | 避免"抄歌抄书" | 用知识改变而非直接搬运 |
| 穿越种田 | ⭐⭐⭐⭐ | 避免"现代知识万能" | 知识有边界/古代人不傻 |
| 无限流 | ⭐⭐⭐ | 避免"纯战斗副本" | 副本有深度谜题/规则探索 |

**警示输出格式**：
```markdown
### 🔍 撞梗检测报告

#### 检测到的相似套路
| 套路名称 | 相似度 | 热门程度 | 建议 |
|---------|:---:|:---:|------|
| 退婚流开局 | 75% | ⭐⭐⭐⭐⭐ | 建议修改：让主角主动提出退婚，反转传统套路 |
| 系统流 | 60% | ⭐⭐⭐⭐⭐ | 建议增强：给系统设置隐藏代价或目的 |

#### 差异化建议
1. **核心设定差异化**：[具体建议]
2. **开局创新**：[具体建议]
3. **人物关系创新**：[具体建议]

#### 原创性评分
综合评分：[X/10]
```

### 反套路生成器

> 列出常见套路并生成反转方案，帮助创作者跳出思维定式。

#### 套路反转方法库

| 套路类型 | 传统套路 | 反套路方案 | 效果 |
|---------|---------|-----------|------|
| 开局 | 主角被退婚羞辱 | 主角主动退婚，对方后悔 | 反转读者预期 |
| 金手指 | 系统/老爷爷无条件帮助 | 金手指是敌人设下的陷阱 | 增加悬念 |
| 反派 | 脸谱化坏人 | 反派有正当理由，主角陷入道德困境 | 增加深度 |
| 升级 | 稳步提升，一帆风顺 | 升级伴随副作用/失去人性 | 增加代价 |
| 感情线 | 女主一开始就喜欢主角 | 女主最初看不起主角，关系渐进 | 增加张力 |
| 战斗 | 主角碾压敌人 | 主角惨胜，付出巨大代价 | 增加真实感 |
| 结局 | 主角成为最强者 | 主角放弃力量，回归平凡 | 反传统结局 |

#### 反套路生成流程

```
用户输入当前创意 → 识别使用的套路 → 生成3种反套路方案 → 用户选择/组合
```

#### 反套路示例

**场景**：用户写"废柴觉醒"套路

**反套路方案**：
```markdown
### 💡 反套路方案

#### 方案A：觉醒即陷阱
主角觉醒后发现，所谓的"传承"其实是上古魔头的夺舍仪式，他必须在力量和自我之间做出选择。

#### 方案B：废柴是伪装
主角故意隐藏实力，扮演废柴是为了保护家人/躲避追杀，觉醒只是被迫暴露。

#### 方案C：废柴是天赋
众人眼中的"废柴"体质，其实是传说中的"混沌道体"，需要特殊方式才能激活。

#### 方案D：觉醒有代价
每次使用力量，主角都会失去一部分记忆/情感，最终可能变成没有感情的怪物。

**选择建议**：方案A和方案D能制造更强的戏剧冲突和代价感，推荐优先考虑。
```

### 创新保护机制

| 保护措施 | 说明 | 触发条件 |
|---------|------|---------|
| 创意指纹 | 为每个创意生成唯一指纹，防止重复生成 | 每次创意生成时 |
| 套路预警 | 检测到使用热门套路时自动提示 | 创意/大纲阶段 |
| 差异化建议 | 针对套路提供具体修改方向 | 检测到撞梗时 |
| 反套路库 | 提供反传统的创意方向 | 用户请求时 |
| 原创性存档 | 保存创作过程，证明原创性 | 每次创作会话 |

## AI参与度透明化

### 参与度分级

| 级别 | AI参与程度 | 典型场景 | 标注建议 |
|------|-----------|---------|---------|
| **L0** | 纯人工 | 用户完全自行创作 | 无需标注 |
| **L1** | AI辅助润色 | 用户写完，AI仅做语法/措辞优化 | 无需标注 |
| **L2** | AI协同创作 | 用户提供框架，AI填充细节 | 建议标注"AI辅助创作" |
| **L3** | AI主导生成 | AI生成主体内容，用户修改调整 | 建议标注"AI协同创作" |
| **L4** | AI全生成 | AI生成全部内容，用户仅提供主题 | 建议标注"AI生成内容" |

## 创作诚信指南

### 发布建议

| 平台 | 建议 |
|------|------|
| 网文平台（起点/番茄/晋江等） | 建议标注AI辅助程度，部分平台有AI内容披露要求 |
| 传统出版 | 建议人工深度修改，确保文学性和原创性 |
| 自媒体/博客 | 建议标注AI辅助创作 |
| 参赛投稿 | 确认赛事规则是否允许AI辅助创作 |

### 最佳实践

1. **AI是工具，不是作者**：将AI视为高效的创作助手，而非替代你创作的"枪手"
2. **保持个人风格**：在AI生成内容的基础上注入你的独特视角和表达习惯
3. **深度修改**：AI生成的内容应经过你的深度修改和个性化调整
4. **情感注入**：AI可以写情节，但真正打动读者的是你注入的真情实感
5. **持续学习**：通过观察AI的创作方式，提升自己的写作技巧
