# 共享模块：Memory分级存储优化系统

> **按需加载**：涉及长篇创作（>100章）、Memory容量管理时加载
> **全局规则**：遵循主控文件第二章核心铁律
> **版本**：v21.1

---

## 一、问题分析

### 1.1 原有问题

| 问题 | 影响 | 发生场景 |
|------|------|---------|
| Memory容量限制 | 超过500章后早期设定丢失 | 长篇创作 |
| 信息冗余 | 相似剧情重复存储 | 日常/修炼章节 |
| 检索效率低 | 关键信息淹没在大量文本中 | 紧急查询 |
| 无优先级区分 | 核心设定和临时状态同等存储 | 所有场景 |

### 1.2 优化目标

- 支持百万字级长篇创作（1000章+）
- 核心设定永不丢失
- 关键信息快速检索
- 存储空间高效利用

## 二、分级存储架构

### 2.1 三级存储体系

```
┌─────────────────────────────────────────────────────────────┐
│                     Memory分级存储架构                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐  │
│  │   L1核心层    │ ←→ │   L2缓存层    │ ←→ │   L3归档层    │  │
│  │  (永久存储)   │    │  (近期100章)  │    │  (历史章节)   │  │
│  └──────────────┘    └──────────────┘    └──────────────┘  │
│         ↑                   ↑                   ↑          │
│    核心设定/人物      当前剧情状态         历史记录          │
│    世界观/伏笔        临时状态            已回收伏笔         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 各层定义

#### L1核心层（永久存储）

**存储内容**：
```yaml
核心设定:
  世界观:
    - 力量体系
    - 地理设定
    - 势力分布
    - 规则体系
  
  人物档案:
    主角:
      - 基础信息（姓名、年龄、背景）
      - 核心性格
      - 成长轨迹（境界变化记录）
      - 关键关系
    核心配角:
      - 基础信息
      - 核心性格
      - 与主角关系
      - 命运轨迹
  
  关键伏笔:
    - 长期伏笔（>100章）
    - 核心谜题
    - 终极BOSS相关
  
  主线脉络:
    - 核心冲突
    - 主要阶段划分
    - 关键转折点
```

**存储策略**：
- 永不删除，只追加更新
- 每次变更记录版本
- 优先加载，常驻内存

**容量预估**：
- 核心设定：≤10KB
- 人物档案：≤20KB（10个核心人物）
- 关键伏笔：≤5KB
- **总计：≤50KB**

#### L2缓存层（近期100章）

**存储内容**：
```yaml
近期剧情:
  章节摘要:
    - 最近100章每章核心事件（50字/章）
    - 关键场景记录
    - 重要对话摘要
  
  人物状态:
    - 当前状态（位置、境界、情绪）
    - 最近10章状态变化
    - 活跃配角列表
  
  活跃伏笔:
    - 待回收伏笔（≤50章内）
    - 新埋设伏笔（最近10章）
    - 回收预警（≤5章）
  
  临时设定:
    - 新引入的设定
    - 当前地图信息
    - 临时势力关系
```

**存储策略**：
- 滚动更新，保留最近100章
- 每10章压缩归档一次
- 快速检索，优先访问

**容量预估**：
- 章节摘要：≤10KB（100章×100字）
- 人物状态：≤5KB
- 活跃伏笔：≤3KB
- **总计：≤30KB**

#### L3归档层（历史章节）

**存储内容**：
```yaml
历史记录:
  压缩摘要:
    - 每50章生成一个阶段摘要
    - 每卷生成一个卷摘要
    - 关键事件索引
  
  已回收伏笔:
    - 回收时间
    - 回收方式
    - 读者反馈
  
  次要人物:
    - 长期未出场人物档案
    - 出场记录
    - 再次出场提醒
  
  废弃设定:
    - 已变更的设定记录
    - 变更原因
    - 变更章节
```

**存储策略**：
- 文件存储，按需加载
- 智能压缩，定期归档
- 索引建立，快速定位

**容量预估**：
- 压缩摘要：≤20KB（1000章）
- 已回收伏笔：≤10KB
- 次要人物：≤20KB
- **总计：≤60KB**

### 2.3 数据流转机制

```
新章节生成
    ↓
写入L2缓存层
    ↓
每10章检查
    ↓
需要归档？
    /        \
  是          否
  ↓           ↓
压缩→L3    保留在L2
  ↓           ↓
更新索引   继续累积
    ↓
超过100章？
    /        \
  是          否
  ↓           ↓
最早章归档L3  继续累积
  ↓
更新L2
```

## 三、智能压缩算法

### 3.1 章节摘要压缩

**原始内容**（第42章）：
```markdown
第42章：宗门大比

林凡在宗门大比中表现出色，连续击败三名内门弟子，引起长老关注。
在决赛中，他面对大师兄赵天霸，虽然境界落后但凭借灵活战术一度占优。
最终因灵力耗尽惜败，但获得长老赏识，破格晋升内门。
同时，他在比赛中使用的神秘玉佩引起某人注意，埋下伏笔。
```

**压缩后**（L2存储）：
```yaml
第42章:
  核心事件: 宗门大比，林凡惜败赵天霸，破格晋升内门
  关键场景: 比武台
  人物: 林凡、赵天霸、长老
  境界变化: 外门→内门
  伏笔: 玉佩引起注意（F015）
  爽点: 越级挑战、破格晋升
```

**压缩率**：约80%

### 3.2 相似章节合并

**检测条件**：
- 核心事件类型相同（如"修炼"、"日常"）
- 人物相同
- 无关键剧情推进
- 无伏笔回收/埋设

**合并策略**：
```yaml
第50-55章（修炼期）:
  类型: 修炼
  人物: 林凡
  内容: 闭关修炼，境界巩固
  关键信息: 无
  合并原因: 连续修炼，无剧情推进
  保留: 仅保留第50章和第55章摘要
```

### 3.3 关键信息提取

**提取规则**：
```python
关键信息优先级:
  P0: 境界突破、关系变化、重要物品获得、核心伏笔
  P1: 战斗胜利、势力变化、新人物登场
  P2: 日常事件、修炼进展、次要对话
  
提取策略:
  - P0信息：完整保留
  - P1信息：摘要保留
  - P2信息：选择性保留（首次出现时）
```

## 四、快速检索机制

### 4.1 多级索引

```yaml
索引结构:
  人物索引:
    林凡:
      - 第1章: 出场
      - 第10章: 突破
      - 第42章: 晋升
    赵天霸:
      - 第5章: 首次提及
      - 第42章: 对决
  
  伏笔索引:
    F001:
      - 埋设: 第5章
      - 回收: 第50章
    F015:
      - 埋设: 第42章
      - 状态: 待回收
  
  设定索引:
    青云宗:
      - 第1章: 介绍
      - 第42章: 大比
    筑基期:
      - 第10章: 林凡突破
```

### 4.2 检索优先级

```
用户查询"林凡的境界"
    ↓
L1核心层检索
    ↓
找到"林凡成长轨迹"
    ↓
返回最新境界
    ↓
同时查询L2
    ↓
补充最近10章变化
    ↓
合并返回
```

**检索时间**：
- L1查询：<10ms
- L2查询：<50ms
- L3查询（按需）：<200ms

### 4.3 智能预加载

**预加载策略**：
```yaml
启动时预加载:
  - L1核心层（必须）
  - L2最近10章（默认）
  
创作时预加载:
  - 当前章细纲
  - 前3章摘要
  - 相关人物档案
  - 待回收伏笔
  
切换场景时预加载:
  - 新场景设定
  - 相关人物状态
  - 相关势力信息
```

## 五、集成到写作流程

### 5.1 模式C集成

在[mode-c-writing.md](mode-c-writing.md)中更新MCP联动：

```markdown
## MCP联动（v21.1优化版）

### Memory分级访问

#### 写入策略
```
新章节生成
    ↓
提取关键信息
    ↓
写入L2缓存层
    ↓
检查是否涉及核心设定
    ↓
是 → 同步更新L1核心层
    ↓
每10章触发归档检查
    ↓
需要归档 → 压缩写入L3归档层
```

#### 读取策略
```
需要查询设定
    ↓
L1核心层 → 命中？
    /        \
  是          否
  ↓           ↓
返回        L2缓存层 → 命中？
              /        \
            是          否
            ↓           ↓
          返回        L3归档层
                        ↓
                      返回（较慢）
```

### 5.2 自动归档触发

**触发条件**：
| 条件 | 行为 |
|------|------|
| 每完成10章 | 检查L2容量，必要时归档 |
| 每完成50章 | 生成阶段摘要，归档到L3 |
| 每完成1卷 | 生成卷摘要，更新索引 |
| Memory警告 | 立即归档最旧内容 |
