# 共享模块：大纲反向补充机制

> 按需加载：模式B-2细纲工程、模式E诊断复盘、批量自动诊断时加载本文件。
> 全局规则：遵循主控文件第二章核心铁律

## 触发方式

| 触发词 | 功能 | 示例 |
|--------|------|------|
| `大纲补充` `反向补充大纲` `细纲反馈大纲` | 基于细纲诊断结果，识别大纲缺口并生成补充方案 | `大纲补充 基于第1-60章细纲` |
| `自动补充大纲` | 自动识别缺口并直接补充到大纲对应模块 | `自动补充大纲 基于全部细纲` |
| `大纲缺口检测` `大纲完整性检查` | 仅检测大纲缺口，不自动补充 | `大纲缺口检测` |

## 功能概述

大纲反向补充机制打破了传统"大纲→细纲"的单向流程，建立**细纲→大纲**的向上反馈闭环：

```
传统单向流程：          反向补充闭环：
                        ┌──────────────┐
  大纲                    │   大纲        │←── 补充建议
   ↓                      │   ↑          │
  细纲                    │   │ 缺口识别  │
   ↓                      │   │          │
  正文                    │  细纲(诊断)   │──→ 发现缺口
                         └──────────────┘
```

当对N章细纲进行批量诊断时，系统不仅检查细纲本身的质量，还会：

1. **聚合分析**：汇总N章细纲中反复出现的"设定不足"信号
2. **缺口定位**：将信号映射到大纲的7大模块，精确定位缺失的子模块
3. **补充方案生成**：为每个缺口生成具体的大纲补充内容
4. **自动/手动补充**：支持自动写入大纲或输出建议供人工审核

## 补充方案生成规则

### 按缺口严重度分级处理

| 严重度 | 标识 | 处理策略 | 示例 |
|:--:|:--:|------|------|
| 🔴 严重 | P0 | 立即生成补充方案，自动写入大纲对应模块 | 力量体系缺失→自动生成3.1力量来源章节 |
| 🟠 重要 | P1 | 生成补充方案，标注建议，等待用户确认后写入 | 配角设定缺失→生成5.4配角补充条目 |
| 🟡 一般 | P2 | 生成补充建议，仅输出报告不自动写入 | 时间流速未定义→建议补充2.3时间流速体系 |

### 补充内容生成模板

对每个识别到的缺口，按以下模板生成补充内容：

```markdown
### 大纲补充：[缺口名称]

- **目标模块**：[大纲模块编号+名称，如：2.4 社会阶层与势力]
- **缺口类型**：[新增子节/扩展现有/细化描述]
- **严重度**：[🔴P0/🟠P1/🟡P2]
- **发现来源**：[细纲第X-Y章诊断信号摘要]
- **影响范围**：[涉及细纲章节数/正文章节数]

#### 建议补充位置
在大纲 [模块编号] 的 [具体位置，如：2.4.3之后] 新增/扩展以下内容：

#### 补充内容
[具体的大纲补充文本，格式与大纲原文一致，包含必要的子节编号]

#### 补充理由
[为什么需要这个补充？不补充会导致什么问题？]

#### 关联影响
- 需同步更新的细纲章节：[列出受影响的细纲章节]
- 需同步更新的正文：[列出受影响的正文]
- 需同步更新的其他大纲模块：[列出关联模块]
```

## 与批量诊断的集成

当执行 `细纲批量诊断` 时，大纲反向补充作为**可选附加阶段**自动运行：

```
细纲批量诊断流程（增强版）：
    │
    ├── 阶段1：逐章细纲诊断（原有）
    │   └── 对每章细纲执行8维诊断
    │
    ├── 阶段2：P0/P1自动修复（原有）
    │   └── 修复细纲层面的问题
    │
    ├── 阶段3：大纲缺口聚合分析（新增）
    │   ├── 汇总阶段1中所有"设定不足"信号
    │   ├── 按大纲7大模块分类聚合
    │   ├── 过滤偶发信号，保留高频/跨章信号
    │   └── 生成缺口清单
    │
    ├── 阶段4：补充方案生成（新增）
    │   ├── 对每个🔴缺口自动生成补充内容
    │   ├── 对🟠缺口生成补充建议
    │   └── 对🟡缺口标记观察
    │
    └── 阶段5：输出综合报告
        ├── 细纲诊断报告（原有）
        ├── 大纲反向补充报告（新增）
        └── 补充前后对比
```

### 聚合分析规则

| 规则 | 说明 |
|------|------|
| **高频阈值** | 同一信号在≥3章细纲中出现，才触发缺口检测 |
| **跨章聚合** | 同一大纲模块的信号合并为一个缺口 |
| **偶发过滤** | 仅在1-2章出现的信号标记为"观察"，不生成补充方案 |
| **严重度加权** | 影响章节数越多，严重度越高 |

## 自动补充 vs 手动确认

### 自动补充（默认）

以下情况自动写入大纲：
- 🔴P0级缺口：影响≥5章细纲，不补充将导致系统性质量问题
- 补充内容为纯新增（不修改现有大纲内容）
- 补充内容与现有大纲无冲突

### 手动确认

以下情况输出建议，等待用户确认：
- 🟠P1级缺口：影响3-4章细纲
- 补充内容涉及修改现有大纲设定
- 补充内容可能与其他模块产生关联影响
- 用户指定 `仅诊断不修复` 模式

### 用户确认指令

| 指令 | 行为 |
|------|------|
| `全部补充` | 执行所有待确认的补充方案 |
| `补充P0` | 仅执行🔴P0级补充 |
| `补充模块X` | 仅补充指定模块（如：`补充模块2`） |
| `跳过补充` | 不执行任何补充，仅保留报告 |
| `补充第X项` | 仅执行指定编号的补充方案 |
