# 模式F：全自动创作流水线（v20.3）

> **所属**：novel-writer 核心能力库  
> **触发词**：`一键创作`、`自动写`、`全自动`、`自动创作`、`恢复流水线`、`重试阶段`、`回滚阶段`  
> **全局规则**：遵循主控文件第二章核心铁律

---

## 流水线概述

模式F将6大模式串联为自动化流水线，用户输入初始需求后，按A→B→B-2→C→E→D顺序自动执行。每阶段完成后自动进入下一阶段，仅在关键决策点暂停等待用户确认。

```
模式F触发 → 灵感构思(A) → 大纲生成(B) → 细纲规划(B-2) → 逐章创作(C)+自动校验 → 质量诊断(E) → 包装输出(D) → 完成
```

---

## 流水线阶段

### 阶段0：需求收集

| 信息项 | 必填 | 说明 |
|--------|------|------|
| 小说题材 | ✅ | 玄幻/都市/言情/悬疑/末世/系统流/历史/科幻 |
| 核心创意 | ✅ | 一句话描述核心卖点（金手指/世界观/人设亮点） |
| 目标字数 | ✅ | 计划总字数或章节数 |
| 写作风格 | 可选 | 从10种风格库中选择，不选则AI自动匹配 |
| 主角设定 | 可选 | 姓名/性别/初始身份，不填则AI自动生成 |
| 特殊要求 | 可选 | 额外要求（如：必须有系统面板、禁止后宫等） |

### 阶段1：灵感构思（模式A）

自动调用模式A生成：
- 3个创意方案（核心梗概+金手指设计+世界观亮点）
- 用户选择方案后进入阶段2

**暂停点**：等待用户选择创意方案

### 阶段2：大纲生成（模式B）

自动调用模式B生成：
- 7模块核心大纲（世界观/力量体系/主角成长/势力格局/人物关系/爽点密度/伏笔规划）
- 用户确认或修改后进入阶段3

**暂停点**：等待用户确认大纲

### 阶段3：细纲规划（模式B-2）

自动调用模式B-2生成：
- 逐章细纲（核心事件/阶段拆分/爽点指令/伏笔操作/感官锚点/人物状态）
- 默认生成前10章细纲

**暂停点**：等待用户确认细纲格式

### 阶段4：逐章创作（模式C）

**循环流程**：
```
for 章节 in 细纲列表:
    1. 读取当前章细纲
    2. 读取前一章结尾（衔接用）
    3. 创作正文（遵循模式C规则）
    4. 自动校验（字数/禁忌词/元数据/钩子/五感/心理描写/文风）
    5. 输出正文+校验报告
    6. 更新Memory（伏笔/人物状态/剧情变动）
    7. 询问：继续下一章？/修改本章？/暂停？
```

**暂停点**：每章完成后等待用户确认

**批量模式**：用户可选择"连续写N章"

**并行模式**（总章节≥30章时可用）：
- 每批5-8章并行创作
- 创作前统一同步设定基线
- 完成后校验文风一致性和章节衔接

### 阶段5：质量诊断（模式E）

全部章节完成后自动调用模式E：
- 创作数据分析仪表板（进度/字数/爽点/伏笔/人物/质量/效率）
- 7维诊断报告
- 跨卷连续性检测
- 章节节奏评分
- 输出优化建议清单

**暂停点**：展示诊断报告，等待用户决定是否修复

### 阶段6：包装输出（模式D）

自动调用模式D生成：
- 15个书名方案（5种流派各3个）
- 多平台简介适配（起点/番茄/晋江/纵横）
- 标签SEO优化方案
- 封面文案

**暂停点**：展示包装方案，等待用户选择

### 阶段7：完成

输出创作总结：
- 总章节数/总字数
- 创作耗时统计
- 里程碑达成情况
- 下一步建议（发布平台推荐/续集规划/番外建议）

## 流水线控制

### 暂停与恢复

| 指令 | 行为 |
|------|------|
| `暂停` | 保存进度到Memory，下次输入`继续`从断点恢复 |
| `跳过XX` | 跳过当前阶段，直接进入下一阶段 |
| `重做XX` | 重新执行当前阶段 |
| `只做XX` | 仅执行指定阶段 |
| `从XX开始` | 从指定阶段开始执行 |

### 断点续传机制

#### 断点保存

每个阶段完成后自动保存状态：
```yaml
流水线状态:
  项目名称: "[书名]"
  当前阶段: [阶段编号]
  阶段内进度:
    阶段4_逐章创作:
      已完成章节: [第X章]
      下一章节: [第Y章]
  已完成阶段: [阶段列表]
  已生成文件: [文件路径列表]
  最后保存时间: "[时间戳]"
```

#### 断点恢复流程

1. 读取Memory中的流水线状态
2. 验证断点数据完整性（文件存在性、数据依赖）
3. 用户确认后从断点恢复
4. 继续流水线执行

### 重试机制

| 重试粒度 | 触发方式 | 适用场景 |
|---------|---------|---------|
| 章节级 | `重试第X章` | 单章质量不达标 |
| 批次级 | `重试批次[N]` | 并行模式某批次文风偏差大 |
| 阶段级 | `重试阶段[X]` | 大纲/细纲需要大幅调整 |
| 全量重试 | `重试全部` | 核心设定变更 |

### 异常回滚机制

| 异常类型 | 严重度 | 处理策略 |
|---------|:---:|---------|
| 文件写入失败 | P1 | 自动重试3次 |
| Memory更新失败 | P1 | 自动重试+降级保存 |
| 生成内容质量不达标 | P2 | 标记+通知用户 |
| 设定冲突检测 | P2 | 暂停+展示冲突+等待决策 |
| 依赖文件丢失 | P0 | 立即暂停+回滚到上一完整阶段 |
| 连续3次重试失败 | P0 | 暂停+建议手动介入 |

## 跨技能协作接口

### 协作技能

| 触发词 | 协作技能 | 说明 |
|--------|---------|------|
| `碰撞灵感` `交叉诊断` `风格参考` | chinese-novelist | 灵感碰撞、风格参考、交叉诊断 |
| `设计封面` `世界地图` `人物立绘` | canvas-design | 封面设计、地图生成、人物立绘 |
| `作品提案` `创作计划PPT` | pptx | 生成提案PPT、创作计划PPT |
| `导出Word` `创作报告` | docx | 导出为Word文档、生成创作报告 |

### 协作原则

1. **主从关系**：novel-writer为主技能，负责最终决策
2. **数据隔离**：协作数据不污染Memory和创作状态
3. **结果融合**：协作结果需经审核后融合
4. **可追溯**：所有协作记录保存至collaborationHistory

---

## 与手动模式的关系

- 模式F是6大模式的自动化编排，不替代任何模式
- 用户可随时退出流水线，切换到手动模式
- 流水线中每个阶段的质量标准与手动模式一致
- 暂停点确保用户对关键决策保持控制权
