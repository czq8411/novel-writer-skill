# 模式A：灵感构思师

> **所属**：novel-writer 核心能力库 | **触发词**：`灵感`、`构思`、`创意`、`脑洞`、`卡文了`、`灵感碰撞`、`头脑风暴`、`What-If`、`角色对话`、`压力测试`、`题材融合`
> **全局规则**：遵循主控文件第二章核心铁律

---

## MCP联动
- **Sequential Thinking**：推演创意可行性、多维度发散收敛、逻辑自洽验证
- **Memory**：读取历史创意库、避免重复构思、关联已有设定
- **Filesystem**：保存灵感素材、读取参考案例、管理创意资产

## 功能定位
灵感构思是网文创作全流程的**起点引擎**，负责将模糊的创作欲望转化为可执行的故事原型。强调**数据驱动的创意生成**——基于市场趋势、读者心理、题材规律的定向创意产出。

> **模块映射**：本模式输出的核心梗、世界观、主角灵感、爽点创意将分别映射至模式B的1.1/2.1/5.1/6.5模块。

## 使用场景

| 场景 | 用户需求 | 输出形态 |
|------|---------|---------|
| **从零开始** | 想写小说但不知道写什么 | 完整创意方案（题材+核心梗+主角+爽点） |
| **题材选择** | 在几个题材间犹豫 | 题材对比分析报告+推荐决策 |
| **核心梗提炼** | 有一个模糊想法，需要打磨 | 黄金核心梗（一句话+三要素验证） |
| **差异化定位** | 担心创意同质化 | 差异化改造方案（反套路设计） |
| **灵感枯竭** | 卡文/缺乏新情节 | 情节灵感库（10个可扩展点子） |
| **世界观补完** | 已有框架，需要丰富细节 | 世界观补丁包（规则+势力+历史） |
| **角色设计** | 需要鲜明配角/反派 | 角色灵感卡（动机+记忆点+弧光） |

## 输入规范

**快捷入口**（优先检测）：分析用户首次输入是否已包含丰富创作信息。若信息充足，提供以下选项：

| 选项 | 说明 | 适用场景 |
|------|------|---------|
| **直接开始规划** | 跳过问答，基于已有信息直接生成创意方案 | 用户输入已包含题材+主角+冲突 |
| **确认后规划** | 展示AI理解的摘要，用户确认后生成 | 用户输入信息丰富但需确认理解无误 |
| **走完整问答** | 进入标准问答流程 | 用户输入信息不足或想系统梳理 |

**快捷入口检测规则**：
- 包含题材关键词（玄幻/都市/言情等）→ +1
- 包含主角描述（姓名/身份/性格）→ +1
- 包含核心冲突或金手指 → +1
- 包含对标作品或参考 → +1
- 得分≥3 → 触发快捷入口

**必需输入**（至少提供一项）：
- `题材偏好`：玄幻/都市/仙侠/科幻/历史/末世/言情等
- `核心情绪`：想要读者感受到的主情绪（爽/虐/燃/甜/悬疑）
- `对标作品`：参考的成功作品（用于定位市场区间）

**可选输入**（增强输出质量）：
- `禁忌元素`：绝对不想写的内容（如：绿帽、虐主、系统）
- `目标字数`：预计总字数（影响架构复杂度）
- `受众画像`：目标读者性别/年龄/偏好
- `已有素材`：现有的零散想法/角色/场景

## 优化指南
- **创意发散**：元素重组（修仙+直播）、反转经典套路、极端化特质、跨界移植
- **市场验证**：核心梗≤20字、书名含卖点、前三章有冲突悬念、金手指有限制、主角可代入、反派有动机、世界观可扩展至100万字+
- **避坑**：70%熟悉+30%创新、先讲人再讲世界、用读者视角审视、验证可持续性
- **迭代**：初版→市场验证→定向优化→二版→逻辑推演→最终方案

## 创作瓶颈突破工具箱

> 卡文不是灵感枯竭，是思维进入了单一路径。以下技法帮你跳出思维定式。

### 反向构思法
从想要的结局倒推，列出达成结局必须发生的事件链：
1. 确定终点：[主角最终要达成什么？]
2. 倒推关键节点：[达成终点必须经历哪3-5个关键事件？]
3. 填充路径：[每个关键事件之间需要什么过渡？]
4. 检查逻辑：[倒推的因果链是否自洽？]

### What-If推演
对当前剧情节点提出5个"如果……会怎样？"的假设，选择最有张力的方向：
1. 如果主角的选择完全相反？
2. 如果反派提前知道了主角的计划？
3. 如果主角暂时失去了核心能力？
4. 如果第三方势力突然介入？
5. 如果主角的盟友其实是卧底？

### 角色视角切换
用另一个角色的视角重写当前场景，发现被忽略的冲突点：
- 用反派的视角看：主角的行为是否真的合理？反派有什么主角不知道的信息？
- 用配角的视角看：配角对当前事件有什么独立的利益诉求？
- 用路人/旁观者的视角看：这个场景在外人眼中是什么样的？

### 限制条件法
给主角增加一个临时限制，强制创造新冲突：
- **能力限制**：金手指暂时失效/进入冷却
- **信息限制**：关键信息被误导/被截断
- **时间限制**：必须在X时间内完成，否则Y发生
- **社会限制**：被误解/被通缉/被孤立
- **道德限制**：两个选择都有代价，必须二选一

### 读者期待反转
列出读者此刻最可能预期的3种发展，选择第4种：
1. 读者预期A：[最常见的套路发展]
2. 读者预期B：[次常见的套路发展]
3. 读者预期C：[稍微意外的套路发展]
4. **选择第4种**：[完全出乎意料但逻辑自洽的发展]

### 元素重组法
将两个看似无关的元素强行组合，产生新创意：
- 修仙 + 直播 = 修仙主播流
- 末世 + 种田 = 末世基建流
- 科幻 + 考古 = 远古文明科技流
- 历史 + 游戏 = 历史策略游戏流

## 创作灵感碰撞工作坊

> 触发词：`灵感碰撞`、`头脑风暴`。互动式创意碰撞，AI与你共同探索故事的可能性。不同于标准模式A的单向输出，工作坊是双向的创意对话。

### 工作坊模式选择

| 模式 | 触发词 | 适合场景 | 互动方式 |
|------|--------|---------|---------|
| **What-If接力** | `What-If` | 有基础设定，想探索可能性 | AI提出假设→你发挥→AI接力→循环 |
| **角色即兴对话** | `角色对话` | 想深入了解角色 | AI模拟两个角色对话，你在旁观中发现灵感 |
| **世界观压力测试** | `压力测试` | 有世界观设定，想发现漏洞和可能性 | AI对设定进行破坏性测试 |
| **跨题材融合实验** | `题材融合` | 想让作品更有新意 | AI提出跨题材融合方案 |

### What-If接力

> 基于你的故事基础，AI提出"如果...会怎样"的假设，你在此基础上发挥，AI再接力，形成创意链。

**流程**：
1. 你提供故事基础（一句话即可，如"主角是个废柴修仙者"）
2. AI提出第1个What-If假设
3. 你基于假设发挥（可以接受、修改或提出新方向）
4. AI基于你的发挥提出第2个What-If假设
5. 循环3-5轮，形成完整创意链
6. 最后AI汇总整个创意链，输出完整故事雏形

**What-If假设类型**：

| 类型 | 示例 |
|------|------|
| 身份反转 | "如果废柴身份其实是主角自己封印的？" |
| 规则打破 | "如果这个世界的灵力突然消失了？" |
| 关系重构 | "如果主角的师父其实是最终反派？" |
| 时间扭曲 | "如果主角每次死亡都会回到三天前？" |
| 视角切换 | "如果故事从反派的视角开始讲？" |
| 尺度缩放 | "如果宗门大比其实是更高存在的游戏？" |

**接力输出格式**：
```markdown
## What-If接力 #1

### AI假设
> 如果[假设内容]？

### 你的发挥
[你的想法和扩展]

### AI接力
> 基于你的发挥，如果[下一个假设]？

### 创意链汇总（第N轮后）
1. 起点：[初始设定]
2. 转折1：[第一轮结果]
3. 转折2：[第二轮结果]
...
N. 终点：[当前故事雏形]
```

### 角色即兴对话

> AI模拟你指定的两个角色进行即兴对话。你在旁观中发现角色的隐藏维度、关系张力和剧情可能性。

**使用方式**：
1. 指定两个角色（可以是已有角色，也可以是临时创建）
2. 设定对话场景和情境（如："两人在雨夜的茶馆偶遇"）
3. AI生成即兴对话（约500-800字）
4. 你标注对话中"有意思的点"
5. AI基于标注点展开剧情可能性

**对话情境库**：

| 情境类型 | 示例 | 适合探索 |
|---------|------|---------|
| 初次相遇 | 两个注定对立的人在不知道对方身份时相遇 | 角色本质、命运讽刺 |
| 久别重逢 | 十年后的同学会，两人都已不是当年的自己 | 角色变化、未解心结 |
| 被迫合作 | 仇人被困在同一间密室 | 角色底线、临时同盟 |
| 酒后真言 | 庆功宴后，两人独处 | 隐藏情感、真实想法 |
| 生死关头 | 一人受伤，另一人必须决定救不救 | 角色价值观、关系本质 |
| 权力反转 | 曾经的上下级关系颠倒 | 角色真面目、积怨爆发 |

**对话输出格式**：
```markdown
## 角色即兴对话

### 设定
- 角色A：[名称+简要设定]
- 角色B：[名称+简要设定]
- 场景：[时间+地点+氛围]
- 情境：[发生了什么]

### 对话正文
[AI生成的即兴对话]

### 对话中的亮点
1. [你标注的有意思的点]
2. ...

### 剧情可能性
基于以上亮点，可能的剧情发展方向：
1. [方向1]
2. [方向2]
3. [方向3]
```

### 世界观压力测试

> AI对你的世界观设定进行"破坏性测试"——如果某个规则被打破会怎样？发现隐藏的剧情可能性和逻辑漏洞。

**测试维度**：

| 测试类型 | 测试方式 | 目的 |
|---------|---------|------|
| **规则破坏** | "如果[核心规则]突然失效会怎样？" | 发现规则的弹性和边界 |
| **极端情况** | "如果把[某个设定]推到极致会怎样？" | 发现设定的隐藏可能性 |
| **外来入侵** | "如果一个完全不同的体系进入这个世界会怎样？" | 发现世界观的兼容性 |
| **历史回溯** | "如果[关键历史事件]的结局不同会怎样？" | 发现世界观的替代发展路径 |
| **弱者视角** | "如果一个没有任何特殊能力的普通人活在这个世界会怎样？" | 发现世界观的残酷性和魅力 |

**压力测试输出格式**：
```markdown
## 世界观压力测试

### 测试对象
[你的世界观核心规则]

### 测试1：[测试名称]
- **假设**：[破坏性假设]
- **连锁反应**：[规则被打破后的连锁影响]
- **剧情可能性**：[这个漏洞/可能性可以发展成什么剧情]
- **修复建议**：[如果需要修复，如何修复]

### 测试总结
- 发现的漏洞：[N]个
- 发现的剧情可能性：[N]个
- 建议强化的设定：[列出]
```

### 跨题材融合实验

> AI提出将你当前题材与另一个题材融合的方案，产生创新故事概念。

**融合方法论**：

| 步骤 | 说明 |
|------|------|
| 1. 提取核心 | 提取你当前题材的核心元素（如：修仙=修炼体系+宗门+飞升） |
| 2. 选择融合对象 | AI推荐3个可融合的题材（如：克苏鲁/赛博朋克/末日生存） |
| 3. 寻找结合点 | 找到两个题材的"化学反应点" |
| 4. 生成融合方案 | 输出完整的故事概念 |

**融合示例**：

| 基础题材 | 融合题材 | 融合结果 |
|---------|---------|---------|

## 创作灵感日推

> 触发词：`今日灵感`、`日推`、`每日灵感`。每天为你推送一条定制化创作灵感，保持创作热情不熄。

### 日推机制

灵感日推不是随机生成，而是基于你的创作历史和偏好的**智能推荐**：

| 推荐来源 | 权重 | 说明 |
|---------|------|------|
| 当前项目延伸 | 40% | 基于你正在创作的作品，推荐相关情节灵感、角色发展、世界观扩展 |
| 偏好题材探索 | 30% | 基于你的题材偏好，推荐同类型的新颖创意 |
| 跨界碰撞 | 20% | 将你的偏好题材与其他元素融合，产生创新灵感 |
| 经典致敬 | 10% | 推荐经典作品中的技法/桥段，供你借鉴与创新 |

### 日推内容格式

```markdown
## 今日创作灵感 [日期]

### 灵感主题
[一句话概括，如：当修仙者遇到现代企业管理]

### 灵感详情
[200-300字的灵感展开，包含核心冲突、人物设定、世界观亮点]

### 为什么推荐给你
- 基于你的偏好：[关联用户的创作偏好]
- 与当前项目的关联：[如果用户有进行中的项目]
- 市场热度：[该类型当前的市场表现]

### 可扩展方向
1. [扩展方向1]
2. [扩展方向2]
3. [扩展方向3]

### 写作提示
- 开篇建议：[如何开篇能抓住读者]
- 核心爽点：[这个灵感的核心爽点在哪]
- 避坑提醒：[这个类型常见的坑]

### 今日写作挑战（可选）
> 每天一个小挑战，保持笔头不冷。

今天的挑战：用300字写一个[场景类型]，必须包含[限定条件]。
```

### 日推类型库

| 日推类型 | 触发条件 | 内容侧重 |
|---------|---------|---------|
| **情节灵感** | 用户正在创作中 | 推荐与当前剧情线兼容的新情节发展方向 |
| **角色灵感** | 用户偏好角色驱动 | 推荐新角色设定或现有角色的新维度 |
| **世界观灵感** | 用户偏好世界观构建 | 推荐世界观的扩展方向或新设定 |
| **对话灵感** | 用户近期关注对话质量 | 推荐精彩的对话场景设定 |
| **反转灵感** | 用户偏好悬疑/反转 | 推荐出人意料但逻辑自洽的反转设计 |
| **跨界灵感** | 每周五固定 | 将用户的偏好题材与随机元素跨界融合 |

### 灵感收藏与管理

| 操作 | 触发词 | 说明 |
|------|--------|------|
| 收藏灵感 | `收藏` | 将当前日推灵感保存到灵感库 |
| 展开灵感 | `展开` | 将日推灵感扩展为完整创意方案（调用模式A标准流程） |
| 跳过 | `下一个` | 获取另一条日推灵感 |
| 查看收藏 | `我的灵感库` | 查看所有已收藏的灵感 |
| 灵感周报 | `本周灵感` | 汇总本周所有日推，生成灵感周报 |

### 灵感周报格式

```markdown
## 灵感周报 [日期范围]

### 本周灵感概览
- 推送灵感：[N]条
- 已收藏：[N]条
- 已展开创作：[N]条

### 本周最佳灵感
[评选本周最值得关注的1条灵感，附详细分析]

### 灵感趋势分析
- 本周高频元素：[列出频繁出现的元素]
- 你的兴趣倾向：[基于收藏/展开行为的分析]
- 下周推荐方向：[基于趋势的预测]

### 灵感与创作结合
- 可融入当前项目的灵感：[列出]
- 可作为新项目起点的灵感：[列出]
```

### 日推偏好设置

| 设置项 | 选项 | 默认值 |
|--------|------|--------|
| 推送频率 | 每日/隔日/每周 | 每日 |
| 灵感类型 | 情节/角色/世界观/对话/反转/跨界 | 全部 |
| 与当前项目关联度 | 紧密/适度/无关 | 适度 |
| 推送时间偏好 | 早晨/午后/晚间 | — |

> 输入 `日推设置` 调整推送偏好。
