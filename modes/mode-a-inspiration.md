## 新手引导（6步渐进式）

**步骤1：题材选择**
```markdown
## 选择题材
1. 玄幻/仙侠 2. 都市 3. 历史 4. 科幻 5. 末世 6. 言情 7. 其他
输入数字或描述偏好。
```

**步骤2：核心情绪**
```markdown
## 你想让读者感受到什么？
1. 爽 2. 燃 3. 虐 4. 甜 5. 悬疑
输入数字或描述。
```

**步骤3：主角设定**
```markdown
## 你的主角是什么样的人？
- 表面身份（如：废柴少年）
- 性格特点（如：聪明、腹黑）
- 最想要的东西（如：变强、复仇）
```

**步骤4：金手指设计**
```markdown
## 给主角一个特殊能力
1. 系统 2. 天赋 3. 知识 4. 重生/穿越 5. 其他
输入数字或描述。
```

**步骤5：开篇钩子**
```markdown
## 故事从哪里开始？
描述一个画面或场景（如：主角被退婚，意外觉醒力量）
```

**步骤6：生成创意方案**
```markdown
## 创意方案已生成
包含：书名建议、核心梗、世界观、主角卡、金手指、黄金三章框架
```

## 输入规范

**快捷入口检测**（≥3项触发）：
- 包含题材关键词 +1
- 包含主角描述 +1
- 包含冲突/金手指 +1
- 包含对标作品 +1

**必需输入**（至少一项）：
- 题材偏好：玄幻/都市/仙侠/科幻/历史/末世/言情
- 核心情绪：爽/虐/燃/甜/悬疑
- 对标作品：参考成功作品

**可选输入**：禁忌元素、目标字数、受众画像、已有素材

## 瓶颈突破工具箱

### 卡文类型诊断
| 类型 | 症状 | 根因 |
|------|------|------|
| 情节卡 | 不知写什么 | 大纲不够细 |
| 人物卡 | 角色不动 | 动机不清晰 |
| 设定卡 | 世界观矛盾 | 设定不严谨 |
| 节奏卡 | 快慢不当 | 爽点分布不均 |
| 动力卡 | 不想写 | 创作疲劳 |
| 质量卡 | 不满意 | 完美主义 |

### 突破技法
- **反向构思**：从结局倒推事件链
- **What-If推演**：5个"如果……会怎样"假设
- **视角切换**：用反派/配角/路人视角重写
- **限制条件法**：增加临时限制（能力/时间/道德）
- **读者期待反转**：列出3种预期，选第4种
- **元素重组**：强行组合无关元素

## 文件输出与目录管理

### 目录结构
```
当前工作目录/
├── 灵感/
│   └── [书名]_创意方案.md
├── 大纲/
│   └── [书名]_大纲.md
├── 细纲/
│   └── [卷名]/
│       └── [卷名]_细纲.md
├── 正文/
│   └── [卷名]/
│       └── 第[章节号]章_[章节名].md
└── metadata.json
```

### 文件输出规则
- **灵感文件**：`灵感/[书名]_创意方案.md`
- **触发条件**：创意方案生成完成后立即保存
- **文件格式**：UTF-8编码，Markdown格式

### 强制保存流程
1. **执行 pwd**：获取当前工作目录
2. **检查目录**：确认 `灵感/` 目录存在，不存在则创建
3. **生成文件名**：`[书名]_创意方案.md`
4. **写入文件**：调用 Filesystem.write_file() 保存灵感方案
5. **记录元数据**：更新 Memory 中的文件路径记录

### MCP联动增强
- **Filesystem**：
  - `create_directory()` - 创建灵感目录
  - `write_file()` - 写入灵感文件（路径：`./灵感/[书名]_创意方案.md`）
- **Memory**：记录书名、文件路径等元数据

### 保存成功反馈
```markdown
## ✅ 灵感方案已保存

### 保存路径
📁 `./灵感/[书名]_创意方案.md`

### 文件信息
- 📝 灵感模块：[N]个
- 📊 文件大小：[X]KB
- ⏱️ 保存时间：[时间戳]

### 下一步
- 输入"大纲"进入模式B生成故事框架
- 输入"诊断大纲"进行质量评估
- 输入"灵感"继续完善创意
```

### 保存失败降级策略
| 失败类型 | 处理方式 |
|---------|---------|
| 目录创建失败 | 提示权限不足，引导用户检查目录权限 |
| 文件写入失败 | 重试1次，仍失败则提示用户手动保存 |
| 文件名非法 | 自动替换非法字符为 `-` |
