## 🎲 抽卡功能总览

模式A的每个步骤都支持「抽卡」功能，让AI为您生成创意灵感。

### 抽卡使用方式

| 步骤 | 触发方式 | 生成内容 |
|------|---------|---------|
| **步骤1** | 输入「抽卡」 | 3个创意题材 |
| **步骤2** | 输入「抽卡」 | 3个情绪组合 |
| **步骤3** | 输入「抽卡」 | 3个主角人设 |
| **步骤4** | 输入「抽卡」 | 3个金手指方案 |
| **步骤5** | 输入「抽卡」 | 3个开篇场景 |
| **步骤6** | 输入「抽卡」 | 3套完整深度信息方案 |

### 抽卡交互流程

```
用户输入「抽卡」
    ↓
AI生成3个选项（A/B/C）
    ↓
用户选择：
  ├─ 输入 A/B/C → 确认选择，进入下一步
  ├─ 输入「再抽」→ 重新生成3个新选项
  └─ 输入数字 → 选择传统选项
```

### 抽卡设计原则

1. **上下文感知**：AI根据前面已选内容生成相关灵感
2. **差异化设计**：3个选项各有特色，避免同质化
3. **无限循环**：不满意可无限「再抽」直到满意
4. **连贯性保障**：选择后流程与手动选择完全一致

### 文件输出规则
- **灵感文件**：`灵感/[书名]_创意方案.md`
- **触发条件**：创意方案生成完成后立即保存
- **文件格式**：UTF-8编码，Markdown格式
- **路径支持**：支持用户指定保存路径，详见下方"动态路径支持"

### 动态路径支持（v20.8新增）

**用户指定路径方式**：
- 在请求中明确指定路径，如：`将灵感保存到 /Users/xxx/小说/`
- 使用当前工作目录（自动检测）
- 使用默认路径（有MCP权限的目录）

**路径解析优先级**：
```
1. 用户明确指定的绝对路径
2. 当前工作目录（pwd）
3. 默认路径：/Users/chenzhiqiang/Desktop/T/document/novel_1
```

**自动降级机制**：

当用户指定的路径无MCP写入权限时，自动切换到 **RunCommand** 工具：

```bash
# 创建目录结构
mkdir -p [目标路径]/[书名]/灵感

# 创建文件
echo "[灵感方案内容]" > [目标路径]/[书名]/灵感/[书名]_创意方案.md
```

**优势**：
- ✅ 无路径限制，可以保存到任意目录
- ✅ 支持动态路径，如 `/Users/xxx/Desktop/小说/novel_2/`
- ✅ 无需修改MCP配置

### 强制保存流程
1. **获取目标路径**：优先级：用户指定 > 当前工作目录 > 默认路径
2. **检查目录权限**：确认目标路径是否有MCP写入权限
3. **无权限时切换**：自动使用 RunCommand 工具创建目录和文件
4. **生成文件名**：`[书名]_创意方案.md`
5. **写入文件**：保存灵感方案内容
6. **记录元数据**：更新 Memory 中的文件路径记录

### MCP联动增强
- **Filesystem**（首选）：
  - `create_directory()` - 创建灵感目录
  - `write_file()` - 写入灵感文件
- **RunCommand**（备选）：无路径限制的目录和文件创建
- **Memory**：记录书名、文件路径等元数据

