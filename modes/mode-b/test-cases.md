# 模式B子模块：多类型大纲测试案例集

> 按需加载：大纲评估时自动加载对应类型的测试案例，或输入"大纲测试"手动触发。
> 全局规则：遵循主控文件第二章核心铁律

---

## 一、测试案例设计原则

1. **覆盖性**：覆盖玄幻、言情、悬疑、科幻、历史五大核心类型
2. **分层性**：每种类型包含简单（30章）、中等（100章）、复杂（300章+）三个复杂度级别
3. **可量化**：每个案例有明确的通过标准和评分细则
4. **真实性**：案例基于真实网文创作场景设计

## 三、通用测试流程

### 测试执行步骤

```
1. 选择测试案例
   ├── 输入"大纲测试 案例1" → 玄幻·废材逆袭
   ├── 输入"大纲测试 案例2" → 言情·都市甜宠
   ├── 输入"大纲测试 案例3" → 悬疑·连环案件
   ├── 输入"大纲测试 案例4" → 科幻·星际殖民
   └── 输入"大纲测试 案例5" → 历史·权谋争霸

2. 系统按案例要求生成大纲
   └── 严格遵循案例的核心梗和复杂度要求

3. 自动执行检测清单
   └── 逐项检查10个必检要素

4. 输出测试报告
   └── 包含通过/未通过项+评分+改进建议
```
