# 模式B子模块：基础设定完整性检测

> 按需加载：大纲输出完成后自动加载，或输入"设定检测"、"完整性检查"手动触发。
> 全局规则：遵循主控文件第二章核心铁律

---

## 一、功能概述

在大纲输出完成后，系统自动检测用户是否遗漏了网文创作中常见的基础设定细节。若检测到缺失项，主动向用户发起提示或询问，帮助全面完善大纲。

---

## 二、检测维度与大纲模块映射

| 检测维度 | 对应大纲模块 | 检测通过条件 | 缺失严重度 |
|---------|------------|------------|:--:|
| 世界观背景 | 2.1 世界观核心 + 2.2 核心时间线 | 至少包含世界运行规则+历史背景+地理环境描述 | 🔴 P0 |
| 核心冲突类型 | 1.2 逻辑推演过程 | 至少明确一种冲突类型（个人成长/势力斗争/生存挑战等） | 🔴 P0 |
| 主要角色设定 | 5.1 主角设定 + 5.2 主角成长路径 | 至少包含性格关键词+核心驱动力+成长弧线 | 🔴 P0 |
| 关键情节转折点 | 6.1 全书结构总览 + 6.2 剧情节点设计 | 每卷至少1个关键高潮事件+1个转折点 | 🟠 P1 |
| 金手指/系统规则 | 1.3 核心设定与逻辑自洽 | 至少包含金手指功能+限制条件+来源逻辑 | 🔴 P0 |
| 势力分布与关系 | 2.4 社会阶层与势力 | 至少包含统治阶级+中层势力+底层生态的描述 | 🟠 P1 |

| 修炼/升级体系 | 3.1 体系总览 + 3.2 等级阶梯 | 至少包含体系名称+等级划分+战力天花板 | 🔴 P0 |
| 内部冲突设计 | 1.2 + 5.1 | 主角是否有内在矛盾（想要vs需要、恐惧vs渴望） | 🟠 P1 |

| 类型特有元素 | 4.7 | 是否包含目标小说类型的核心特有元素 | 🟠 P1 |

## 三、逐模块检测规则

### 模块1：核心定位与逻辑推演

| 检测项 | 检测规则 | 缺失提示 |
|--------|---------|---------|
| 核心冲突类型 | 1.2中是否明确了冲突类型（个人成长/势力斗争/生存挑战/资源争夺/理念对抗） | "你的核心冲突类型还不够明确。网文中常见的冲突类型包括：个人成长型（突破自我限制）、势力斗争型（阵营对抗）、生存挑战型（环境/资源危机）、资源争夺型（宝物/领地竞争）、理念对抗型（价值观冲突）。你的故事主要属于哪种？建议在1.2逻辑推演中明确标注。" |
| 内部冲突 | 1.2或5.1中是否描述了主角的内在矛盾（如：想要复仇但害怕失去人性、渴望力量但恐惧代价） | "主角缺少内部冲突设计。优秀的角色不仅有外部目标，还有内在矛盾——'想要'与'需要'之间的张力。请补充：主角内心深处恐惧什么？他的性格缺陷是什么？他在追求目标的过程中需要克服什么内在障碍？" |
| 外部冲突 | 1.2中是否明确了外部对抗力量（具体反派/敌对势力/环境威胁/制度压迫） | "外部冲突不够具体。请明确：谁/什么在阻碍主角？这个阻碍力量的具体形态是什么？它为何必须与主角为敌？" |
| 金手指规则 | 1.3中是否包含金手指的功能+限制+来源逻辑 | "金手指设定缺少关键约束。请补充：金手指的具体能力边界是什么？使用代价或冷却时间？它的来源是否与世界观逻辑自洽？严禁无代价的机械降神。" |

### 模块2：世界观与基础架构

| 检测项 | 检测规则 | 缺失提示 |
|--------|---------|---------|
| 地理环境 | 2.1中是否描述了世界的地理特征（大陆分布/气候/特殊地域） | "世界观缺少地理环境描述。请补充：这个世界的地理特征是怎样的？有哪些特殊地域（禁地/秘境/异空间）？地理环境如何影响势力分布和剧情发展？" |
| 社会结构 | 2.4中是否包含完整的社会阶层描述 | "社会结构设定不够完整。请补充：这个社会的阶层如何划分？各阶层之间的流动通道是什么？主角从底层崛起的路径是否合理？" |
| 历史背景 | 2.2中是否包含至少3个关键历史事件 | "缺少历史背景铺垫。请补充：这个世界有哪些塑造当前格局的关键历史事件？这些事件如何影响主角的处境和选择？" |
| 势力关系 | 2.4中是否描述了势力间的互动关系（敌对/联盟/制衡/依附） | "势力间的关系不够清晰。请补充：各势力之间是敌对、联盟还是制衡关系？主角属于或将要建立哪个势力？势力格局会如何演变？" |
| 货币体系 | 2.5中是否包含货币单位+购买力锚点 | "经济体系缺失。请补充：这个世界的货币是什么？1单位货币能购买什么？不同等级货币的兑换比例？主角如何获取财富和资源？" |
| 资源获取 | 2.5中是否描述了资源获取方式 | "资源获取方式未定义。请补充：主角主要通过什么方式获取修炼资源/财富/信息？获取方式是否与世界观设定一致？是否存在资源稀缺性带来的冲突？" |

### 模块3：力量体系与等级架构

| 检测项 | 检测规则 | 缺失提示 |
|--------|---------|---------|
| 体系名称 | 3.1中是否定义了力量体系名称 | "力量体系缺少正式命名。建议为你的力量体系起一个有辨识度的名称（如：序列途径/修真境界/魂力等级），增强世界观的仪式感。" |
| 等级划分 | 3.2中是否包含完整的等级阶梯（至少5级） | "等级划分不够详细。请补充：每个等级的名称、能力表现、社会地位、晋升条件。等级不仅是数值，更是社会地位和剧情推进的阶梯。" |
| 战力天花板 | 3.1中是否定义了战力天花板 | "缺少战力天花板设定。请明确：这个世界的最强战力表现是什么？这有助于防止后期战力通胀和崩坏。" |
| 力量代价 | 3.4中是否定义了使用力量的代价 | "力量代价未定义。请补充：使用该力量体系需要付出什么代价？（消耗理智/寿命/金钱/生命力/精神力）严禁无代价的变强。" |

### 模块5：人物设定与关系网

| 检测项 | 检测规则 | 缺失提示 |
|--------|---------|---------|
| 性格特征 | 5.1中是否包含3-5个核心性格关键词 | "主角性格特征不够鲜明。请补充3-5个核心性格关键词，并确保性格特征在剧情中有具体的表现场景。" |

| 特殊能力 | 5.1中是否描述了特殊能力进化路线 | "主角的特殊能力/技能进化路线未定义。请补充：主角的核心能力如何随剧情进化？有哪些关键的能力觉醒节点？" |

### 模块6：情节线设计

| 检测项 | 检测规则 | 缺失提示 |

## 四、检测流程

```
大纲输出完成
        ↓
逐模块检测13个基础设定维度
        ↓
    ┌───────────────────────────┐
    │ 全部通过？                 │
    └──────┬──────────┬─────────┘
           ↓ 是       ↓ 否
    输出"基础设定完整，   按严重度排序缺失项
    可进入细纲阶段"           ↓
                    P0级缺失：立即提示，建议补充后再进入细纲
                    P1级缺失：提示用户，可先进入细纲但标注风险
                           ↓
                    用户补充后重新检测
                           ↓
                    全部通过 → 进入下一阶段
```

## 六、交互规则

1. **主动检测**：大纲输出完成后，自动运行检测，若存在缺失项，主动展示提示报告
2. **分级处理**：P0级缺失强烈建议立即补充，P1级缺失可延后但需标注风险
3. **模块定位**：每个缺失项明确标注对应的大纲模块编号，方便用户快速定位修改
4. **选项式引导**：为每个缺失维度提供2-3个具体补充方向，降低用户决策成本
5. **跳过机制**：用户可回复"跳过"暂缓某项补充，系统记录跳过项并在细纲阶段适时提醒
6. **关联影响**：提示某项缺失可能对后续细纲创作和正文写作的具体影响
