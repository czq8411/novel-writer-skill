### 已有章节定义导入模式（v22.2新增）

> 当用户已有逐章定义（如从《九转废体》级别的大纲中提取），B-2优先导入而非重新生成，以保留原大纲的叙事节拍/情绪曲线/伏笔编号等信息完整性。

#### 导入方式

| 方式 | 触发词 | 流程 |
|:----:|:-------|------|
| 直接粘贴 | `导入细纲` | 用户粘贴已有章节定义文本 → 自动识别字段映射 → 输出匹配的细纲单元 |
| 引用大纲 | `引用大纲定义` | 用户指定大纲文件路径 → 提取逐章定义区域 → 批量生成章节索引 |
| 手工输入 | 默认进入模式模板 | 按标准/详细/深度模板逐章填写 |

#### 字段映射规则

| 原大纲定义中查找 | 映射到B-2深度模板 |
|:---------------:|:----------------:|
| `核心情节` / `核心故事` | 核心情节 |
| `场景：` | 基本信息·场景 |
| `时间：` / `日期` | 基本信息·时间 |
| `年龄：` | 基本信息·年龄 |
| `境界：` | 基本信息·境界 |
| `关键事件` 下的每行 `-` 条目 | 叙事节拍（每条一个节拍） |
| `【性格缺陷】` / `【成长】` 标注 | 人物状态·行为注释 |
| `本章节伏笔：F\d+` | 伏笔管理（保留编号+状态） |
| `情绪曲线：\d+→\d+→` | 情绪曲线（数值序列） |
| `爽点：` | 爽点设计 |
| `章末钩子` | 章末钩子 |
| `---`（章节分隔线） | 章节边界 |

#### 导入后的处理

1. **保留原始内容**：导入后原定义的叙事节拍/伏笔编号/情绪曲线全部保留
2. **仅增补缺失**：只对原定义中缺失的字段（如跨章连续性）进行增补
3. **不重排结构**：不改变原定义的章节顺序、事件顺序和情感节奏
4. **字段标记**：对AI生成的增补内容使用 `【AI补全】` 标记，让用户区分原始定义和AI补充

### 动态路径支持（v20.8新增）

**用户指定路径方式**：
- 在请求中明确指定路径，如：`将细纲保存到 /Users/xxx/小说/`
- 使用当前工作目录（自动检测）
- 使用默认路径（有MCP权限的目录）

**路径解析优先级**：
```
1. 用户明确指定的绝对路径
2. 当前工作目录（pwd）
3. 默认路径：/Users/chenzhiqiang/Desktop/T/document/novel_1
```

**自动降级机制**：

当用户指定的路径无MCP写入权限时，自动切换到 **RunCommand** 工具：

```bash
# 创建目录结构
mkdir -p <目标路径>/<书名>/细纲/<卷名>

# 创建文件
echo "<细纲内容>" > <目标路径>/<书名>/细纲/<卷名>/<卷名>_细纲.md
```

**优势**：
- ✅ 无路径限制，可以保存到任意目录
- ✅ 支持动态路径，如 `/Users/xxx/Desktop/小说/novel_2/`
- ✅ 无需修改MCP配置

### 保存失败降级策略
| 失败类型 | 处理方式 |
|---------|---------|
| 目录创建失败（权限不足） | 自动切换到RunCommand工具，无路径限制 |
| 目录创建失败（其他原因） | 重试1次，仍失败则提示用户手动保存 |
| 文件写入失败 | 重试1次，仍失败则提示用户手动保存 |
| 文件名非法 | 自动替换非法字符为 `-` |

### 强制保存流程
1. **获取目标路径**：优先级：用户指定 > 当前工作目录 > 默认路径
2. **检查目录权限**：确认目标路径是否有MCP写入权限
3. **无权限时切换**：自动使用 RunCommand 工具创建目录和文件
4. **生成文件名**：`<卷名>_细纲.md`
5. **写入文件**：保存细纲内容
6. **记录元数据**：更新 Memory 中的文件路径记录

### MCP联动增强
- **Filesystem**（首选）：
  - `create_directory()` - 创建细纲卷目录
  - `write_file()` - 写入细纲文件
- **RunCommand**（备选）：无路径限制的目录和文件创建
- **Memory**：记录当前卷名、章节范围、文件路径等元数据
