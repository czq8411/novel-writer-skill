## 一、书名优化

### 书名方案生成（5种流派各3个）

| 流派 | 方案 |
|------|------|
| 反差流 | 《<表面身份>：只有我能<特殊能力>》、《让你<做A>，没让你<做B>》、《全球<事件>：我<反常识行为>》 |
| 脑洞流 | 《<夸张数字>级，开局<震撼事件>》、《<身份>？<实际身份>！》、《<事件>后，我成了<反差身份>》 |
| 无敌流 | 《一刀<数字>级，开局<震撼事件>》、《<身份>：从<起点>到<终点>》、《全球<事件>：我<无敌能力>》 |
| 情感流 | 《<情感冲突>：我<反杀行为>》、《被<事件>后，我<逆袭行为>》、《<背叛/伤害>之后，我<重生/崛起>》 |
| 系统流 | 《<系统名称>：开局<奖励>》、《激活<系统>，我<逆天改命>》、《<职业>系统：我在<世界>无敌了》 |

### 书名黄金法则
- 前10字必须包含核心卖点（职业/金手指/特殊身份）
- 优先直白吸睛，避免文艺晦涩
- 数字和反差是点击率最高的元素
- 书名长度≤15字（移动端完整显示）

### 书名A/B测试
| 书名 | 流派 | 点击率预估 | 适用平台 | 优势 | 风险 |
|------|------|-----------|---------|------|------|
| <<方案1>> | <<类型>> | <<高/中/低>> | <<平台>> | <<优势>> | <<风险>> |

### 书名生成量化要求
- **备选数量**：每种流派至少生成3个书名，总计≥12个备选书名
- **风格覆盖**：必须覆盖以下至少4种风格：霸气型/悬念型/反差型/文艺型/直白型
- **字数限制**：每个书名4-12字（番茄系放宽至15字）
- **测试维度**：每个书名标注吸引度(1-5)、记忆度(1-5)、辨识度(1-5)

## 二、简介打磨

### 黄金三段式
| 段落 | 内容 | 字数 | 目标 |
|------|------|------|------|
| 第一段 | 主角身份+核心困境+独特处境 | ≤50字 | 3秒抓住眼球 |
| 第二段 | 特殊能力/系统/机遇+差异化优势 | ≤50字 | 制造期待感 |
| 第三段 | 爽点预告+世界观亮点+悬念钩子 | ≤50字 | 驱动点击 |

### 多平台适配
| 平台 | 读者特征 | 简介风格 | 重点 | 避坑 |
|------|---------|---------|------|------|
| 起点 | 男频为主，追求爽感 | 直白有力 | 金手指、升级体系 | 避免文艺腔 |
| 番茄 | 免费阅读，碎片化时间 | 钩子前置 | 开局冲突、反转 | 避免慢热铺垫 |
| 晋江 | 女频为主，重视人设情感 | 突出人设反差 | CP设定、情感冲突 | 避免纯爽文腔 |
| 纵横 | 老白读者多，重视逻辑深度 | 突出世界观深度 | 设定严谨、权谋智斗 | 避免无脑爽 |

### 简介量化要求
- **平台版本**：至少生成3个平台版本（起点版/番茄版/综合版）
- **起点版**：200-300字，强调世界观深度和伏笔设计
- **番茄版**：100-150字，强调爽点节奏和冲突密度
- **综合版**：150-200字，平衡深度与节奏

## 八、文件输出与目录管理

### 目录结构
```
当前工作目录/
├── 灵感/
│   └── <书名>_创意方案.md
├── 大纲/
│   └── <书名>_大纲.md
├── 细纲/
│   └── <卷名>/
│       └── <卷名>_细纲.md
├── 正文/
│   └── <卷名>/
│       └── 第<章节号>章_<章节名>.md
├── 包装/
│   └── <书名>_包装方案.md
└── metadata.json
```

### 文件输出规则
- **包装文件**：`包装/<书名>_包装方案.md`
- **触发条件**：包装方案生成完成后立即保存
- **文件格式**：UTF-8编码，Markdown格式
- **路径支持**：支持用户指定保存路径，详见下方"动态路径支持"

### 动态路径支持（v20.8新增）

**用户指定路径方式**：
- 在请求中明确指定路径，如：`将包装方案保存到 /Users/xxx/小说/`
- 使用当前工作目录（自动检测）
- 使用默认路径（有MCP权限的目录）

**路径解析优先级**：
```
1. 用户明确指定的绝对路径
2. 当前工作目录（pwd）
3. 默认路径：/Users/chenzhiqiang/Desktop/T/document/novel_1
```

**自动降级机制**：

当用户指定的路径无MCP写入权限时，自动切换到 **RunCommand** 工具：

```bash
# 创建目录结构
mkdir -p <目标路径>/<书名>/包装

# 创建文件
echo "<包装方案内容>" > <目标路径>/<书名>/包装/<书名>_包装方案.md
```

**优势**：
- ✅ 无路径限制，可以保存到任意目录
- ✅ 支持动态路径，如 `/Users/xxx/Desktop/小说/novel_2/`
- ✅ 无需修改MCP配置

### 强制保存流程
1. **获取目标路径**：优先级：用户指定 > 当前工作目录 > 默认路径
2. **检查目录权限**：确认目标路径是否有MCP写入权限
3. **无权限时切换**：自动使用 RunCommand 工具创建目录和文件
4. **生成文件名**：`<书名>_包装方案.md`
5. **写入文件**：保存包装方案内容
6. **记录元数据**：更新 Memory 中的文件路径记录

### MCP联动增强
- **Filesystem**（首选）：
  - `create_directory()` - 创建包装目录
  - `write_file()` - 写入包装文件
- **RunCommand**（备选）：无路径限制的目录和文件创建
- **Memory**：记录书名、文件路径、平台选择等元数据

### 保存成功反馈
```markdown
## ✅ 包装方案已保存

### 保存路径
📁 `./包装/<书名>_包装方案.md`

### 文件信息
- 📝 书名方案：<N>个
- 📊 简介方案：<N>个（适配<N>个平台）
- 🏷️ 标签组合：<N>组
- 📊 文件大小：<X>KB
- ⏱️ 保存时间：<时间戳>

### 下一步
- 输入"写正文"开始创作
- 输入"一键发布"进行全流程打包
- 输入"包装"重新生成方案
```

### 保存失败降级策略
| 失败类型 | 处理方式 |
|---------|---------|
| 目录创建失败 | 提示权限不足，引导用户检查目录权限 |
| 文件写入失败 | 重试1次，仍失败则提示用户手动保存 |
| 文件名非法 | 自动替换非法字符为 `-` |
