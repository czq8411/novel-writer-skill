# 模式E子模块：大纲-剧情深度诊断与联动修正

> 按需加载：涉及大纲变更后自动剧情诊断、番茄平台内容标准评估、大纲深度关联丰富、跨章节联动修正时加载本文件。
> 全局规则：遵循主控文件第二章核心铁律

---

## 一、功能概述

本模块是模式E诊断功能的高阶扩展，解决以下4项核心能力缺口：

| 能力 | 说明 | 触发场景 |
|------|------|---------|
| **能力1：大纲变更自动触发** | 大纲发生实质性变更后，自动启动关联章节的剧情诊断 | 大纲补充/修改完成后 |
| **能力2：番茄平台标准评估** | 按番茄小说平台爆款标准量化评估章节，判定内容扩充必要性 | 剧情诊断时 |
| **能力3：大纲深度关联丰富** | 从章节内容反向深度诊断大纲，智能生成丰富扩展方案 | 章节诊断发现大纲不足时 |
| **能力4：跨章节联动修正** | 章节内容变动后，自动识别关联设定并级联修正 | 章节修复后 |

## 二、能力1：大纲变更自动触发剧情诊断

### 2.1 触发机制设计

```
大纲变更事件
    │
    ├── 事件类型识别
    │   ├── 大纲模块新增（如新增势力/角色/技能体系）
    │   ├── 大纲模块修改（如修改境界体系/核心冲突/金手指规则）
    │   ├── 大纲模块删除（如删除角色/势力/伏笔线）
    │   └── 大纲反向补充（细纲诊断触发的大纲自动补充）
    │
    ├── 影响范围计算
    │   ├── 直接关联章节：大纲模块中明确引用的章节范围
    │   ├── 间接关联章节：涉及相同角色/势力/伏笔的其他章节
    │   └── 全量关联：核心设定变更（如境界体系）→ 全部章节
    │
    └── 自动触发剧情诊断
        ├── 仅诊断受影响章节（默认）
        ├── 全量诊断（核心设定变更时）
        └── 输出变更影响报告 + 诊断结果
```

### 2.2 触发条件判断（严谨条件机制）

**仅在以下条件同时满足时触发**，避免干扰正常编辑流程：

| 条件 | 判断规则 | 说明 |
|------|---------|------|
| **条件A：大纲发生实质性变更** | 大纲文件内容发生修改（非格式调整） | 通过对比大纲文件哈希值或修改时间判断 |
| **条件B：变更涉及剧情关联模块** | 变更涉及模块1-7中的剧情关联子节（1.2/1.3/2.1-2.5/3.1-3.5/5.1-5.5/6.1-6.6/7.1-7.4） | 仅格式/编号调整不触发 |
| **条件C：存在已完成的章节内容** | 细纲或正文目录中存在≥1章已完成内容 | 无章节内容时不触发 |
| **条件D：非用户主动编辑模式** | 当前不处于模式C（正文创作）的连续写作状态 | 避免打断创作流 |

**不触发场景（白名单）**：
- 用户正在连续创作（模式C进行中，最近5分钟内有过正文输出）
- 大纲变更仅为格式调整（如编号重排、标题修改）
- 大纲变更仅为错别字修正
- 用户明确指定"仅诊断不修复"模式且大纲变更为手动修改
- 大纲变更范围仅涉及模块1.1（核心卖点描述）等非剧情关联模块

## 三、能力2：番茄小说平台爆款内容标准评估

### 3.1 平台标准体系

基于番茄小说平台TOP100爆款长篇小说的共性特征，建立以下量化评估标准：

#### 3.1.1 章节充实度标准

| 评估维度 | 爆款标准 | 权重 | 评分规则 |
|---------|---------|:--:|---------|
| **情节推进量** | 每章≥1个实质性情节推进点 | 25% | 0点=1分/1点=6分/2点=8分/≥3点=10分 |
| **信息增量** | 每章≥2个新信息点（设定/人物/世界观） | 20% | 0点=1分/1点=5分/2点=7分/≥3点=10分 |
| **情感波动** | 每章≥1次情感曲线波动（≥3分振幅） | 15% | 无波动=2分/1次=6分/2次=8分/≥3次=10分 |
| **对话信息密度** | 对话中信息增量占比≥60% | 10% | <30%=2分/30-60%=6分/>60%=10分 |
| **感官描写覆盖** | 每章≥3种非视觉感官细节 | 10% | 0-1种=2分/2种=6分/3种=8分/≥4种=10分 |
| **章末钩子强度** | 钩子引发"必须看下一章"的冲动≥7/10 | 10% | <4=2分/4-6=6分/7-8=8分/9-10=10分 |
| **爽点/冲突密度** | 每章≥1个爽点或冲突升级 | 10% | 无=1分/小型=6分/中型=8分/大型=10分 |

#### 3.1.2 内容扩充必要性判定

| 综合评分 | 扩充必要性 | 判定说明 |
|:---:|:---:|------|
| ≥8.5分 | **无需扩充** | 章节充实度达标，符合爆款标准 |
| 7.0-8.4分 | **建议微调** | 部分维度可优化，建议针对性补充50-150字 |
| 5.5-6.9分 | **需要扩充** | 章节内容偏薄，建议扩充200-500字 |
| 4.0-5.4分 | **急需扩充** | 章节内容明显不足，建议扩充500-1000字 |
| <4.0分 | **严重不足** | 章节空洞，建议重新规划或大幅扩充1000+字 |

#### 3.1.3 番茄平台专项检测项

| 检测项 | 爆款特征 | 检测规则 | 级别 |
|--------|---------|---------|:--:|
| **开篇吸引力** | 前500字内出现核心冲突或金手指 | 检查前500字是否包含冲突/金手指描写 | P0 |
| **黄金三章** | 第1章铺垫≤1500字，第3章完成首次小高潮 | 检查前3章铺垫字数和高潮位置 | P0 |
| **爽点节奏** | 每3-5章1个小释放，每30-50章1个大闭环 | 检查爽点间隔和大剧情闭环周期 | P1 |
| **追读钩子** | 每章结尾必须有开放式悬念 | 检查章末300字是否包含有效钩子 | P0 |
| **信息释放节奏** | 新设定分批释放，单章新设定≤3个 | 检查单章新设定数量和释放节奏 | P1 |
| **情感投入点** | 每10章≥1个情感爆发场景 | 检查情感爆发场景密度 | P1 |
| **反派层次** | 反派有独立动机和成长线 | 检查反派是否脸谱化 | P1 |
| **世界观沉浸** | 每5章≥1处世界观细节深化 | 检查世界观细节分布 | P2 |
| **字数健康** | 单章2000-5000字，波动≤30% | 检查字数和波动 | P2 |

## 四、能力3：大纲深度关联诊断与智能丰富

### 4.1 深度关联诊断维度

区别于现有的大纲一致性校验（仅检查"是否偏离"），深度关联诊断从以下维度进行：

| 诊断维度 | 说明 | 诊断方法 |
|---------|------|---------|
| **大纲覆盖度** | 章节内容中涉及的元素，大纲是否都有对应设定 | 提取章节中的角色/地点/道具/事件→逐一比对大纲 |
| **大纲粒度** | 大纲设定的细化程度是否足以支撑当前章节的细节描写 | 检查大纲子节的详细程度 vs 章节描写的需求 |
| **大纲前瞻性** | 大纲是否已规划当前章节之后3-5章的关键节点 | 检查大纲6.2/6.3对未来章节的覆盖 |
| **大纲一致性压力** | 章节内容是否对大纲设定产生了潜在冲突压力 | 检查章节新增设定是否与大纲预留空间兼容 |
| **大纲丰富度** | 大纲中是否有足够的细节储备供后续章节调用 | 检查大纲各模块的信息密度 |

### 4.2 章节→大纲反向深度诊断

```
逐章深度诊断
    │
    ├── 步骤1：提取章节中的所有设定元素
    │   ├── 人物元素：出场角色/角色状态/角色关系变化
    │   ├── 地点元素：场景位置/地域特征/环境细节
    │   ├── 道具元素：出现物品/功法/资源
    │   ├── 事件元素：核心事件/支线事件/伏笔操作
    │   └── 设定元素：新规则/新概念/新势力
    │
    ├── 步骤2：逐一比对大纲覆盖情况
    │   └── for 每个设定元素:
    │       ├── 大纲中有明确定义 → ✅ 覆盖
    │       ├── 大纲中有隐含提及 → ⚠️ 需细化
    │       └── 大纲中完全缺失 → ❌ 大纲缺口
    │
    ├── 步骤3：聚合分析
    │   ├── 统计各模块的覆盖率和缺口分布
    │   ├── 识别高频缺口（≥3章出现同一缺口）
    │   └── 评估缺口对后续章节的影响
    │
    └── 步骤4：生成大纲丰富方案
        ├── 对每个🔴缺口生成具体的大纲补充内容
        ├── 对🟠缺口生成细化建议
        └── 输出大纲丰富度评分
```

## 五、能力4：跨章节内容联动修正

### 5.1 关联设定识别模型

当某章节内容发生实质性变动时，系统自动识别以下关联维度：

```
章节变动事件
    │
    ├── 维度1：角色状态关联
    │   ├── 境界/等级变更 → 影响后续所有涉及该角色战力的章节
    │   ├── 关系阶段变更 → 影响后续所有涉及该角色互动的章节
    │   ├── 身份/地位变更 → 影响后续所有涉及该角色身份的章节
    │   └── 角色新增/删除 → 影响所有该角色出场的章节
    │
    ├── 维度2：世界观设定关联
    │   ├── 地域/地图变更 → 影响所有涉及该地域的章节
    │   ├── 势力格局变更 → 影响所有涉及该势力的章节
    │   ├── 时间线变更 → 影响所有涉及时间节点的章节
    │   └── 规则/体系变更 → 影响所有涉及该规则的章节
    │
    ├── 维度3：道具/功法关联
    │   ├── 道具新增/删除 → 影响所有涉及该道具的章节
    │   ├── 功法体系变更 → 影响所有涉及该功法的章节
    │   └── 资源/货币变更 → 影响所有涉及经济系统的章节
    │
    ├── 维度4：伏笔/悬念关联
    │   ├── 伏笔提前回收 → 影响后续所有涉及该伏笔的章节
    │   ├── 伏笔埋设变更 → 影响回收章节的衔接
    │   └── 悬念解答变更 → 影响所有涉及该悬念的章节
    │
    └── 维度5：情节线关联
        ├── 主线事件变更 → 影响所有关联的支线章节
        ├── 支线新增/删除 → 影响主线交叉章节
        └── 章节顺序调整 → 影响所有衔接章节
```

### 5.2 级联修正流程

```
步骤1：变动捕获
    ├── 识别变动的具体内容（角色/设定/道具/伏笔/情节）
    ├── 记录变动前后的差异
    └── 标记变动类型（新增/修改/删除）

步骤2：关联扫描
    ├── 扫描所有已完成的细纲章节
    ├── 扫描所有已完成的正文章节
    ├── 扫描大纲中的关联设定
    └── 生成关联影响清单

步骤3：影响评估
    ├── 对每个关联项评估影响程度（高/中/低）
    ├── 高影响：直接矛盾，必须修正
    ├── 中影响：间接不一致，建议修正
    └── 低影响：细节偏差，可选修正

步骤4：修正方案生成
    ├── 对高影响项：自动生成修正文本
    ├── 对中影响项：生成修正建议
    └── 对低影响项：标记观察

步骤5：级联修正执行
    ├── 按章节顺序逐一修正
    ├── 每修正一章，重新扫描后续章节的关联影响
    ├── 修正后自动验证一致性
    └── 输出修正日志

步骤6：修正后验证
    ├── 重新运行大纲一致性诊断
    ├── 重新运行前后章衔接诊断
    ├── 确认未引入新的设定冲突
    └── 输出验证报告
```

### 5.4 级联修正安全机制

| 安全规则 | 说明 |
|---------|------|
| **修正前备份** | 每次级联修正前，自动创建受影响章节的版本快照 |
| **逐章验证** | 每修正一章，立即验证该章与前后章的一致性 |
| **回滚机制** | 若修正后评分下降或引入新问题，自动回滚到修正前版本 |
| **人工确认门槛** | 单次修正影响≥5章时，先输出影响报告，等待用户确认后执行 |
| **最大修正范围** | 单次级联修正不超过20章，超出部分标记为待人工处理 |
| **冲突检测** | 修正内容与用户手动修改冲突时，优先保留用户修改 |
