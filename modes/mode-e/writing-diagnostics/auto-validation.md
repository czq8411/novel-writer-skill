# 模式C子模块：自动校验闭环

> 按需加载：正文创作完成后自动触发，或输入`字数检查`手动触发。
> 全局规则：遵循主控文件第二章核心铁律

---

## 一、校验清单

正文创作完成后，自动触发以下校验（无需用户介入）：

| 校验项 | 检查方式 | 不通过处理 |
|--------|---------|-----------|
| **字数检查** | 统计中文字符数，目标3500-4500字 | 不足自动补充，超出提示精简 |
| **禁忌词扫描** | 扫描2.3黑名单词汇（因此/然而/仿佛/似乎/一丝/莫名/不由得等） | 自动替换为自然表达，输出替换清单 |
| **元数据泄漏检查** | 扫描伏笔编号（FXX）、细纲指令标记（【爽点】）是否误入正文 | 自动删除标记，转化为自然描写 |
| **钩子强度检查** | 检查章末300字是否包含悬念/危机/反转元素 | 强度不足自动强化钩子 |
| **五感调用检查** | 确认正文包含至少2种非视觉感官描写 | 不足自动补充感官细节 |
| **心理描写长度** | 检查单次心理独白是否超过3行 | 超标自动拆分并穿插动作 |
| **文风一致性** | 对比本章与前3章的句式分布/感官偏好/对话占比/情绪密度 | 偏差>15%标记警告，偏差>25%自动调整 |

---

## 二、文风一致性深度校验

| 校验维度 | 检查方式 | 阈值 | 不通过处理 |
|---------|---------|------|-----------|
| **句式复杂度分布** | 统计短句(≤15字)/中句(16-30字)/长句(>30字)比例，与前3章均值对比 | 偏差>15% | 标记警告，输出分布对比表 |
| **感官调用偏好** | 统计视觉/听觉/嗅觉/触觉/味觉调用频率，与前3章均值对比 | 偏差>15% | 标记警告，建议补充缺失感官 |
| **对话占比** | 统计对话字数占总字数比例，与前3章均值对比 | 偏差>10% | 标记警告，提示调整 |
| **情绪密度** | 统计情感词汇出现频率和分布，与前3章均值对比 | 偏差>20% | 标记警告，输出情绪曲线对比 |

## 三、修复规则

- **自动修复**：禁忌词替换、元数据清理、感官补充（最多2轮）
- **标记修复**：字数偏差>20%、钩子缺失——标记问题，输出修复建议，由用户决定是否重写
- **校验报告**：每次校验后输出简要报告

```markdown
## 自动校验报告

| 校验项 | 状态 | 详情 |
|--------|------|------|
| 字数 | ✅/⚠️ | [实际字数]字 |
| 禁忌词 | ✅/⚠️ | 发现[N]处，已自动替换 |
| 元数据泄漏 | ✅/⚠️ | 发现[N]处，已自动清理 |
| 钩子强度 | ✅/⚠️ | [强度评估] |
| 五感调用 | ✅/⚠️ | 已调用[N]种非视觉感官 |
| 心理描写 | ✅/⚠️ | [最长段落行数] |
```

## 四、校验流程

```
正文创作完成
    │
    ├── 1. 字数检查
    │   ├── 3500-4500字 → ✅ 通过
    │   ├── <3500字 → 自动补充（加载content-expansion.md）
    │   └── >4500字 → 提示精简
    │
    ├── 2. 禁忌词扫描
    │   └── 自动替换 → 输出替换清单
    │
    ├── 3. 元数据泄漏检查
    │   └── 自动清理 → 转化为自然描写
    │
    ├── 4. 钩子强度检查
    │   └── 强度不足 → 自动强化钩子
    │
    ├── 5. 五感调用检查
    │   └── 不足 → 自动补充感官细节
    │
    ├── 6. 心理描写长度检查
    │   └── 超标 → 自动拆分并穿插动作
    │
    ├── 7. 文风一致性校验
    │   ├── 偏差<15% → ✅ 通过
    │   ├── 偏差15-25% → ⚠️ 标记警告
    │   └── 偏差>25% → 🔧 自动调整
    │
    └── 8. AI设定漂移检测（v20.9新增）
        ├── 漂移评分<10% → ✅ 通过
        ├── 漂移评分10-25% → ⚠️ 提示关注
        ├── 漂移评分25-50% → ⚠️ 生成修正建议
        └── 漂移评分>50% → 🔴 暂停创作，要求确认
```

## 五、AI设定漂移检测（v20.9新增）

> **核心目的**：检测AI在创作过程中无意识引入的新设定与初始大纲的冲突，防止设定漂移。

### 5.1 检测维度

| 维度 | 检测内容 | 提取方法 |
|------|---------|---------|
| **新角色引入** | 正文中出现的未在设定快照中定义的角色 | NER命名实体识别 |
| **新势力出现** | 正文中出现的未在设定快照中定义的势力/宗门/家族 | 关键词匹配 + 上下文分析 |
| **新境界/功法/法宝** | 正文中出现的未在设定快照中定义的修炼境界、功法、法宝 | 关键词匹配 + 规则库对比 |
| **规则变化** | 时间流速、修炼规则、世界法则的变化描述 | 语义相似度对比 |
| **人物状态异常** | 角色境界、年龄、身份与设定快照不一致 | 实体属性提取 + 对比 |

### 5.2 设定提取规则

#### 5.2.1 角色提取
- 识别模式：`[姓名]，[身份/职业]`、`[姓名]是[身份]`、`[姓名]来自[势力]`
- 提取内容：姓名、身份、势力归属、境界

#### 5.2.2 势力提取
- 识别模式：`[势力名]` + 常见后缀（宗、门、派、帮、族、帝国、联盟）
- 提取内容：势力名称、势力类型、核心人物

#### 5.2.3 修炼体系提取
- 识别模式：`[境界名]` + 常见后缀（期、境、阶、层）
- 提取内容：境界名称、等级关系、突破条件

#### 5.2.4 规则提取
- 识别模式：时间描述（X天后、Y年后）、规则描述（必须、不能、可以）
- 提取内容：时间流速、约束条件、特殊规则

### 5.3 冲突检测逻辑

| 冲突类型 | 判定条件 | 示例 |
|---------|---------|------|
| **境界冲突** | 新提及的境界不在初始修炼体系列表中 | 初始最高元婴期，新增化神期 |
| **规则冲突** | 新规则与初始规则直接矛盾 | 初始1章≈1天，当前1章≈3个月 |
| **人物冲突** | 角色属性与快照不一致且无合理解释 | 快照年龄16岁，当前描述18岁（仅过5章） |
| **势力冲突** | 已覆灭势力再次出现 | 第30章覆灭的黑风寨在第45章再次出现 |
| **逻辑冲突** | 设定违反基本逻辑或物理法则 | 筑基期击败元婴期（无特殊设定支持） |

### 5.4 漂移检测执行流程

```
1. 读取当前章正文
    │
2. 提取设定信息（角色/势力/境界/规则）
    │
3. 读取最新设定快照
    │
4. 对比提取信息与快照
    │
5. 标记差异类型（新增/修改/删除/冲突）
    │
6. 计算漂移评分
    │
7. 根据评分执行响应策略
```

### 5.5 响应策略

| 漂移评分 | 等级 | 响应策略 | 是否暂停 |
|---------|------|---------|---------|
| 0-10% | ✅ 稳定 | 记录变化，无需干预 | 否 |
| 10%-25% | ⚠️ 轻微漂移 | 输出警告，提示关注 | 否 |
| 25%-50% | ⚠️ 中等漂移 | 生成详细报告和修正建议 | 否（连续模式）/ 是（单章模式） |
| >50% | 🔴 严重漂移 | 暂停创作，要求用户确认 | 是 |

### 5.6 详细报告格式

```markdown
## AI设定漂移检测报告

### 检测信息
- **章节**: 第45章
- **基准快照**: INITIAL（大纲完成时）
- **漂移评分**: 23%（轻微漂移）
- **检测时间**: 2026-05-21 10:30

### 发现的设定变化

#### 🔴 冲突项
| 类型 | 原文内容 | 初始设定 | 差异说明 |
|------|---------|---------|---------|
| 境界 | "化神期强者俯瞰众生" | 最高境界：元婴期 | 新增未定义境界 |

#### ⚠️ 警告项
| 类型 | 原文内容 | 初始设定 | 差异说明 |
|------|---------|---------|---------|
| 时间 | "三个月后，林凡..." | 1章≈1天 | 时间流速不一致 |
| 人物 | "林凡已经18岁了" | 16岁（第1章） | 年龄增长过快（仅5章） |

#### ✅ 正常项
| 类型 | 原文内容 | 初始设定 | 说明 |
|------|---------|---------|------|
| 境界 | "筑基中期" | 筑基期 | 正常境界提升 |
| 势力 | "青云宗" | 青云宗 | 设定一致 |

### 处理建议
1. 🔴 请确认"化神期"是否为有意扩展世界观，若是请更新初始设定
2. ⚠️ 检查第40章后的时间线一致性
3. ⚠️ 确认主角年龄增长是否符合剧情设定

### 后续操作
- 输入"继续"：忽略警告继续创作
- 输入"修正"：根据建议修正本章内容
- 输入"更新设定"：将新设定添加到快照中
```

### 5.7 自动修复规则

| 修复类型 | 自动修复条件 | 修复方式 |
|---------|-------------|---------|
| **时间流速不一致** | 单章时间跳跃≤30天 | 自动调整描述使其符合时间流速 |
| **年龄计算错误** | 年龄增长超出合理范围 | 提示修正，需人工确认 |
| **境界表述不一致** | 境界名称拼写错误 | 自动修正为标准名称 |
| **势力名称错误** | 势力名称拼写错误 | 自动修正为标准名称 |
