# 模式C子模块：质量保障体系

> 按需加载：涉及AB测试、章节过渡检测、跨章节校验时加载本文件。
> 全局规则：遵循主控文件第二章核心铁律

---

## 章节级AB测试（可选）

> 对关键场景生成多个版本，对比选择最优方案。AI记录偏好并应用到后续章节。

### 适用场景
- 高潮/反转/情感爆发点
- 读者预期分歧点（不同读者可能期待不同走向）
- 作者对两种写法犹豫不决时

### AB测试模式

| 版本 | 侧重方向 | 适用场景 | 节奏特征 |
|------|---------|---------|---------|
| **版本A：动作驱动** | 侧重动作与节奏，快速推进 | 爽文/战斗/追逐场景 | 快节奏，短句为主 |
| **版本B：心理驱动** | 侧重心理与氛围，深度刻画 | 精品文/悬疑/情感场景 | 慢节奏，中长句为主 |
| **版本C：对话驱动** | 侧重对话与互动，人物塑造 | 人物驱动/群像/日常场景 | 中等节奏，对话为主 |

### 使用方式
1. 在关键场景前输入 `AB测试` 触发
2. AI生成2-3个版本（各约300-500字）
3. 用户选择偏好版本，AI记录选择倾向
4. 后续章节自动偏向用户偏好风格

### 偏好记录
```json
{
  "abPreferences": {
    "actionDriven": 0.6,
    "psychologyDriven": 0.3,
    "dialogueDriven": 0.1,
    "lastChoice": "versionA",
    "history": <
      {"chapter": 15, "scene": "宗门大比", "chosen": "versionA"}
    >
  }
}
```

### AB测试→质量校验反馈闭环

> AB测试不仅是选择偏好，更是持续优化创作质量的反馈系统。每次选择都在训练AI理解你的审美。

#### 闭环流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 生成AB   │───▶│ 用户选择  │───▶│ 偏好更新  │───▶│ 质量校验  │
│ 多版本   │    │ 最优版本  │    │ 权重调整  │    │ 自动应用  │
└──────────┘    └──────────┘    └──────────┘    └────┬─────┘
      ▲                                              │
      │              ┌──────────┐                    │
      └──────────────│ 反馈学习  │◀───────────────────┘
                     └──────────┘
```

#### 选择理由记录

每次AB选择后，记录选择理由，用于深度理解用户偏好：

| 记录项 | 说明 |
|--------|------|
| 选择的版本 | <<版本A/B/C>> |
| 选择理由 | <<节奏更好/心理描写更细腻/对话更自然/动作更精彩>> |
| 未选版本的问题 | <<节奏太慢/描写过于直白/对话生硬/动作描写不足>> |
| 希望保留的元素 | <<未选版本中值得保留的亮点>> |

#### 偏好权重自动调整

| 选择倾向 | 权重调整 | 影响范围 |
|---------|---------|---------|
| 连续3次选动作驱动 | 动作驱动权重+0.15 | 战斗/冲突场景自动偏向动作描写 |
| 连续3次选心理驱动 | 心理驱动权重+0.15 | 情感/转折场景自动增加心理描写 |
| 连续3次选对话驱动 | 对话驱动权重+0.15 | 日常/群像场景自动增加对话占比 |
| 混合选择 | 权重不变 | 保持当前风格混合比例 |

#### 质量校验联动

AB测试结果自动触发以下质量校验：

| AB选择 | 联动校验项 | 校验重点 |
|--------|-----------|---------|
| 选择动作驱动 | 节奏检查、爽点密度 | 确认动作场景节奏是否达标，爽点是否充分释放 |
| 选择心理驱动 | 心理描写长度、情感层次 | 确认心理描写是否超标，情感是否有层次递进 |
| 选择对话驱动 | 对话占比、语音辨识度 | 确认对话是否推动剧情，角色语音是否有辨识度 |
| 混合选择 | 全面校验 | 确认各元素比例是否协调 |

#### 跨章节偏好一致性

| 检测项 | 说明 |
|--------|------|
| 风格漂移检测 | 对比最近10章的AB选择分布，检测是否存在无意识的风格漂移 |
| 场景-风格匹配 | 检测AB选择是否与场景类型匹配（战斗场景选动作驱动为正常，日常场景选动作驱动需关注） |
| 读者预期一致性 | 同一类型场景（如连续3场战斗）的风格选择是否一致，避免读者困惑 |

## 智能章节过渡检测

> 章节间衔接是读者体验的关键盲区。此模块在每章创作前自动检测与前一章的衔接质量。

### 过渡检测触发
- 每次创作新章节前自动触发
- 输入 `检查过渡` 手动触发指定章节的过渡检测

### 检测维度

#### 场景承接检测

读取前一章结尾300字和本章开头300字，检测场景转换是否自然：

| 场景关系 | 检测重点 | 通过标准 |
|---------|---------|---------|
| **同一场景延续** | 环境细节一致性、物品位置、光线/时间变化 | 环境描写无矛盾，时间流逝有暗示 |
| **场景切换** | 切换是否有过渡句、是否交代移动过程 | 有至少1句过渡描写，非"瞬间移动" |
| **时间跳跃** | 跳跃时长是否合理、是否有时间流逝暗示 | 有环境/人物状态变化暗示时间流逝 |

#### 时间线连贯检测

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 时间顺序 | 前一章结尾时间 → 本章开头时间 | 前一章是深夜，本章开头是正午且无过渡 |
| 时间流速 | 事件发生所需时间是否合理 | 从A城到B城需要3天，但下一章开头已在B城 |
| 并行事件 | 多线叙事的时间对齐 | A线已过3天，B线还在同一时刻 |

#### 情绪连贯检测

| 检测项 | 通过标准 | 问题示例 |
|--------|---------|---------|
| 情绪承接 | 本章开头情绪与前一章结尾情绪有合理过渡 | 前一章结尾主角悲痛欲绝，本章开头嬉笑打闹 |
| 情绪变化 | 如有情绪变化，有触发事件或过渡描写 | 情绪突变无原因交代 |
| 情绪强度 | 情绪强度变化有渐进过程 | 从平静直接跳到暴怒，无触发描写 |

#### 钩子回应检测

| 检测项 | 检查方式 |
|--------|---------|
| 钩子状态 | 前一章结尾的钩子是否在本章得到回应（解决/延续/升级） |
| 钩子遗忘 | 钩子超过3章未回应 → 标记警告 |
| 钩子质量 | 钩子回应是否满足读者期待（不能敷衍了事） |

### 过渡质量评分

| 维度 | 权重 | 评分(1-10) |
|------|------|-----------|
| 场景承接 | 30% | <<X>> |
| 时间连贯 | 25% | <<X>> |
| 情绪连贯 | 25% | <<X>> |
| 钩子回应 | 20% | <<X>> |
| **综合过渡评分** | — | **<X/10>** |

### 过渡问题自动修复

| 问题类型 | 自动修复方案 |
|---------|-------------|
| 场景断裂 | 在本章开头插入1-2句过渡描写（环境变化/移动过程/时间流逝暗示） |
| 时间跳跃过大 | 插入时间流逝暗示（"三天后"、"当阳光再次照进窗户时"） |
| 情绪不连贯 | 在本章开头插入情绪过渡句（"那股悲伤还没散去"、"他深吸一口气，把情绪压了下去"） |
| 钩子未回应 | 在本章前500字内插入钩子回应（哪怕只是"他还在想昨晚那句话"） |

## 跨章节自动校验修复闭环

> 每完成5章自动触发跨章节连贯性校验，自动发现问题并修复，形成质量保障闭环。无需用户手动触发诊断。

### 触发条件

| 条件 | 行为 |
|------|------|
| 每完成5章（第5/10/15...章） | 自动触发跨章节校验 |
| 每完成一卷 | 自动触发跨卷校验（更深度） |
| 用户输入 `跨章校验` | 手动触发指定范围校验 |

### 校验流程

```
完成第N章正文
      │
      ▼
  N是5的倍数？───否──▶ 仅执行单章自动校验
      │
     是
      ▼
┌─────────────────────────────────────┐
│  跨章节自动校验（第N-4章 ~ 第N章）    │
├─────────────────────────────────────┤
│  1. 读取最近5章正文+对应细纲          │
│  2. 执行5维连贯性校验                 │
│  3. 发现问题 → 自动修复（最多2轮）     │
│  4. 修复后重新校验                    │
│  5. 输出修复前后对比报告              │
│  6. 仅在修复失败时暂停等待用户决策     │
└─────────────────────────────────────┘
```

### 校验维度

#### 人物状态连续性

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 境界/等级 | 逐章追踪主角境界，检测跳跃 | 第3章筑基初期，第5章筑基后期且无修炼描写 |
| 装备/物品 | 追踪关键物品的持有状态 | 第2章获得的神器，第4章战斗中未使用 |
| 伤势/状态 | 追踪受伤和恢复时间线 | 第3章重伤，第4章开场就生龙活虎 |
| 位置/移动 | 追踪角色位置变化 | 第3章在A城，第5章在B城，中间无移动描写 |

#### 伏笔链路完整性

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 伏笔回收 | 已埋伏笔是否在计划章节回收 | F015计划第10章回收，第10章未提及 |
| 伏笔遗忘 | 超过20章未回收的伏笔 | F008已埋25章未回收 → 标记警告 |
| 伏笔冲突 | 新埋伏笔是否与已回收伏笔矛盾 | F012已回收"张三死了"，新章又出现张三 |

#### 战力体系一致性

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 战力天花板 | 是否出现超出设定上限的战力表现 | 设定上限元婴，出现化神级战斗 |
| 等级压制 | 越级战斗是否有合理解释 | 筑基胜金丹，无特殊手段说明 |
| 金手指限制 | 金手指使用是否超出设定限制 | 设定每天3次，一章内用了5次 |

#### 时间线合理性

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 时间流速 | 事件发生所需时间是否合理 | 从A城到B城需3天，但事件在1天内完成 |
| 时间标记 | 是否有明确的时间参照 | 连续5章无任何时间标记 → 读者时间感混乱 |
| 并行事件对齐 | 多线叙事的时间是否对齐 | A线已过3天，B线还在同一时刻 |

#### 爽点与节奏分布

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 爽点密度 | 5章内爽点数量 | 5章仅1个爽点 → 密度偏低 |
| 爽点类型 | 是否连续重复同类型 | 连续3个爽点都是"打脸" → 审美疲劳 |
| 高潮分布 | 高潮是否集中在某几章 | 5章中3章有高潮，2章纯过渡 → 节奏不均 |

#### 人物关系发展连续性（新增）

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 关系阶段跳跃 | 逐章追踪每对角色关系阶段，检测单章跨越>2级 | 第5章S0→第6章S4，中间无互动 |
| 关系升级触发 | 关系升级时检查是否有共同经历事件（≥200字描写） | S2→S3但对应章节无合作/对话场景 |
| 关系互动频率 | 已建立S3+关系的角色，检查每3章是否至少有1次互动 | 连续5章无互动 |
| 新关系建立节奏 | 同一章节建立≥3对新关系（S0→S2+） | 第5章同时与3人建立S2+关系 |

#### 角色身份交代连续性（新增）

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 首次出场身份 | 角色首次出场300字内是否包含≥2个身份维度 | 出场500字后仍未提及其身份 |
| 身份变化同步 | 角色身份变化时（如晋升），是否在正文中明确描写 | 第9章夺冠→第10章直接以内门弟子身份出现 |
| 群体身份一致 | 同一场景/同一批次的角色，身份交代是否一致 | A交代了外门弟子，B/C/D未交代 |
| 身份信息密度 | 单章新出场角色>3个时，是否每个都交代了核心身份 | 第5章4个新角色同时出场，仅主角身份完整 |

#### 神秘感与悬念连续性（新增）

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 终极反派过早出现 | 前20章是否直接点名/描述最终boss | 第3章直接描述"魔帝"身份和能力 |
| 悬念层次检查 | 每10章是否至少维持3层悬念（L1-L4） | 第1-10章仅有章级钩子，无卷级/终极悬念 |
| 谜题解答节奏 | 单章是否同时解答≥3个悬念 | 第15章一次性揭示3个谜底 |
| 神秘信息保留率 | 前30章揭示的世界观信息是否>60% | 第25章已揭示80%世界观设定 |

#### 世界观展开连续性（新增）

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 重大事件前置铺垫 | 宗门大比/秘境等事件前是否有≥3章铺垫 | 第8章入门→第10章直接宗门大比 |
| 场景跳跃检测 | 5章内是否跨越≥3个大地域 | 家族→宗门→秘境→王都，4章完成 |
| 宗门/组织描写 | 加入宗门后，宗门结构/规则/派系描写是否≥500字 | 宗门仅用2句话带过 |
| 赛事信息完整度 | 大比/赛事前是否交代参赛资格/赛制/奖励/历史 | 直接开始比赛，无任何规则说明 |
| 日常细节密度 | 每10章是否有≥3处日常生活描写 | 连续15章无衣食住行描写 |

#### 人物群像连续性（新增）

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 工具人检测 | 角色出场≥5章，是否所有行为仅服务于主角 | 角色A出场10章，每次都是给主角送信息 |
| 背景故事密度 | 主要角色（≥10章出场）背景交代是否≥200字 | 角色B出场15章，背景仅"猎户之子"4字 |
| 独立高光检测 | 主要角色每30章是否有≥1次独立高光 | 角色C连续40章无独立解决问题场景 |
| 对话区分度 | 不同角色对话风格/口头禅是否有区分 | 3个配角说话方式完全一样 |

#### 爽点质量连续性（新增）

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 重大事件精彩度 | 入门测试/大比等事件是否有铺垫+受众反应+对比 | 入门测试3段话写完，无任何观众反应 |
| 受众反应描写 | 主角高光后是否有旁观者反应（震惊/质疑/崇拜） | 主角展示实力后直接切场景 |
| 对比凸显 | 主角优秀是否通过同级对比/前辈评价来凸显 | 只说"天赋异禀"，无任何对比参照 |
| 胜利代价 | 重大胜利是否有代价/付出描写 | 主角轻松获胜，毫发无伤 |
| 涟漪效应 | 爽点后主角地位/关系/剧情是否有变化 | 夺冠后一切照旧，无人提及 |

#### 前期铺垫连续性（新增）

| 检测项 | 检查方式 | 问题示例 |
|--------|---------|---------|
| 章节充实度 | 每章是否有≥1个情节推进+≥2个新信息 | 第12章纯对话，无情节推进 |
| 伏笔密度 | 前30章埋设伏笔是否≥5个 | 前30章仅2个伏笔 |
| 设定展开率 | 大纲已设定的势力/人物，提及后10章内是否展开 | 第5章提及"丹堂"，第20章仍未展开 |
| 势力/人物活跃度 | 已引入势力/人物是否在15章内再次出场 | 第8章引入角色D，第25章才再次出现 |
| 过渡章比例 | 连续过渡章（无高潮/无转折）是否≤3章 | 第15-20章连续6章纯过渡 |

| 伏笔密度过低 | 在合适位置埋设新伏笔（每10章≥2个） | 1-2轮 |

### 修复失败处理

> 以下情况自动修复可能失败，需暂停等待用户决策：

| 失败类型 | 表现 | 处理 |
|---------|------|------|
| 修复引入新问题 | 修复后校验发现新问题 | 回退修复，输出问题分析，等待用户决策 |
| 修复需要大幅改动 | 修复需要改动>500字 | 输出修复方案，等待用户确认后执行 |
| 连续2轮修复失败 | 同一问题2轮修复后仍未通过 | 标记为"需人工处理"，输出详细诊断 |
