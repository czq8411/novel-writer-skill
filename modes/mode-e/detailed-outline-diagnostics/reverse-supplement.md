# 模式B-2子模块：细纲→大纲反向补充

> **所属**：novel-writer 模式B-2细纲工程 | **触发**：每完成10章细纲自动触发
> **版本**：v1.0 | **依赖**：<shared/outline-supplement.md](../../shared/outline-supplement.md)

---

## 功能定位

细纲→大纲反向校验是"合规检查"（细纲是否偏离大纲），反向补充是"缺口发现"（大纲本身是否缺失设定）。两者互补，形成完整的双向反馈闭环。

| 对比维度 | 反向校验 | 反向补充（本模块） |
|---------|---------------|---------------|
| **目的** | 检查细纲是否偏离大纲 | 发现大纲本身的设定缺失 |
| **方向** | 细纲→大纲（对比差异） | 细纲→大纲（发现缺口） |
| **触发** | 每章细纲完成后自动触发 | 每完成10章细纲自动触发 |
| **输出** | 偏离清单+修正指令 | 缺口清单+补充方案 |
| **处理** | 修正细纲对齐大纲 | 补充大纲对应模块 |

---

## 触发条件

| 触发方式 | 条件 | 行为 |
|---------|------|------|
| **自动触发** | 每完成10章细纲 | 自动聚合分析，生成补充报告 |
| **手动触发** | 输入 `大纲补充` `反向补充大纲` `细纲反馈大纲` | 基于当前所有细纲执行缺口分析 |
| **批量触发** | 输入 `细纲批量诊断 第1-N章` | 诊断流程中包含大纲反向补充阶段 |

## 反向补充流程

```
细纲完成N章
    │
    ├── 1. 聚合N章细纲中的"设定不足"信号
    │   └── 如：连续3章势力描写单薄、配角背景空白、技能无体系归属
    │
    ├── 2. 按大纲7大模块分类映射
    │   └── 势力单薄→模块2.4 | 配角空白→模块5.4 | 技能无归属→模块3.3
    │
    ├── 3. 过滤偶发信号（<3章出现则忽略）
    │
    ├── 4. 生成补充方案
    │   ├── 🔴严重缺口：自动生成补充内容，写入大纲
    │   ├── 🟠重要缺口：生成补充建议，等待确认
    │   └── 🟡一般缺口：标记观察，输出建议
    │
    └── 5. 输出补充报告
```

## 用户确认指令

| 指令 | 行为 |
|------|------|
| `全部补充` | 执行所有待确认的补充方案 |
| `补充P0` | 仅执行🔴P0级补充 |
| `补充模块X` | 仅补充指定模块（如：`补充模块2`） |
| `跳过补充` | 不执行任何补充，仅保留报告 |
