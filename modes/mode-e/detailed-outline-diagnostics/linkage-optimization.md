# 模式B-2子模块：多章节细纲联动优化

> **所属**：novel-writer 模式B-2细纲工程 | **触发**：每完成5章细纲自动触发
> **版本**：v1.0 | **依赖**：细纲核心输出、大纲模块(6)

---

## 功能定位

单章细纲优秀不等于连续章节阅读体验好。此模块检测连续3-5章的宏观节奏和体验，确保跨章节的阅读流畅性和爽点多样性。

---

## 触发条件

| 触发方式 | 条件 | 行为 |
|---------|------|------|
| **自动触发** | 每完成5章细纲 | 自动执行3维联动检测 |
| **手动触发** | 输入 `联动优化` `章节联动` `节奏检测` | 对最近5章执行联动检测 |
| **范围触发** | 输入 `联动优化 第X-Y章` | 对指定范围执行联动检测 |

---

## 检测维度

### 维度一：爽点类型轮换检测

| 检测项 | 健康标准 | 问题示例 |
|--------|---------|---------|
| 连续爽点类型 | 同类型爽点连续≤3次 | 连续5章都是"打脸"爽点 → 审美疲劳 |
| 爽点强度波动 | 强度应有起伏，非匀速 | 连续5章爽点强度都是7分 → 缺乏高潮 |
| 爽点间隔 | 每3-5章1个爽点 | 连续8章无明确爽点 → 读者流失 |

### 维度二：节奏单调检测

| 检测项 | 健康标准 | 问题示例 |
|--------|---------|---------|
| 章节功能轮换 | 铺垫/发展/高潮/过渡应交替出现 | 连续4章都是"过渡"功能 → 节奏拖沓 |
| 情绪曲线多样性 | 连续章节情绪曲线应有不同形态 | 连续3章都是"直线上升型" → 读者疲劳 |
| 场景类型轮换 | 室内/室外/战斗/日常应交替 | 连续5章都是"宗门大殿"场景 → 空间单调 |

### 维度三：信息密度均衡检测

| 检测项 | 健康标准 | 问题示例 |
|--------|---------|---------|
| 新信息点分布 | 每章1-3个新信息点，连续章节不堆积 | 第10章5个新信息点，第11章0个 → 信息过载后断档 |
| 伏笔密度 | 每3章埋1-2个伏笔，不过度集中 | 连续2章各埋4个伏笔 → 回收压力大 |
| 人物出场均衡 | 核心配角每3章至少出场1次 | 某配角连续8章未出场 → 读者遗忘 |

### 维度四：冲突节奏检测（新增）

| 检测项 | 健康标准 | 问题示例 |
|--------|---------|---------|
| 内部冲突连续性 | 内部冲突每2-3章有进展体现 | 连续5章无内部冲突描写 → 人物扁平化 |
| 外部冲突升级感 | 外部冲突每5章有可感知的升级 | 连续10章外部冲突强度不变 → 缺乏紧张感 |
| 冲突类型轮换 | 内部/外部冲突交替推进 | 连续8章只有外部冲突 → 人物缺乏深度 |

### 维度五：高潮/结局推进检测（新增）

| 检测项 | 健康标准 | 问题示例 |
|--------|---------|---------|
| 高潮铺垫进度 | 每章对卷高潮有可感知的推进 | 连续5章与高潮无关 → 高潮突兀 |
| 结局伏笔密度 | 每5-8章埋1个结局相关伏笔 | 前50章无结局伏笔 → 结局缺乏铺垫 |
| 情感峰值分布 | 每10章至少1个情感小高峰 | 连续15章情感平淡 → 读者疲劳 |

## 联动问题严重度判定

| 问题类型 | 判定条件 | 严重度 |
|---------|---------|:---:|
| 爽点类型连续重复 | ≥5章同类型 | 🔴 严重 |
| 爽点类型连续重复 | 4章同类型 | 🟠 中等 |
| 爽点类型连续重复 | 3章同类型 | 🟡 一般 |
| 章节功能连续重复 | ≥4章同功能 | 🔴 严重 |
| 章节功能连续重复 | 3章同功能 | 🟠 中等 |
| 信息密度失衡 | 相邻章信息点差≥4 | 🔴 严重 |
| 信息密度失衡 | 相邻章信息点差2-3 | 🟠 中等 |
| 核心配角缺席 | ≥8章未出场 | 🔴 严重 |
| 核心配角缺席 | 5-7章未出场 | 🟠 中等 |
| 内部冲突缺失 | ≥5章无进展 | 🔴 严重 |
| 内部冲突缺失 | 3-4章无进展 | 🟠 中等 |
| 高潮铺垫断裂 | ≥5章无推进 | 🔴 严重 |
| 高潮铺垫断裂 | 3-4章无推进 | 🟠 中等 |
