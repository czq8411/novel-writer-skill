## 智能路由引擎（v20.6新增）

> 触发词：`智能诊断`、`自动诊断`、`全面诊断`
> 系统自动分析诊断请求，智能识别所需子技能并按优先级排序调用。

### 一、智能识别算法

```markdown
## 🎯 智能诊断流程

### 输入解析阶段
1. **意图识别**
   - 用户输入 → 诊断意图分类 → 诊断对象判定
   
2. **上下文分析**
   - 当前创作阶段（大纲/细纲/正文）
   - 当前章节位置
   - 历史诊断数据
   - 近期修复记录

3. **子技能匹配**
   - 意图 → 子技能映射表
   - 上下文 → 辅助子技能匹配
   - 历史数据 → 关注子技能优先级
```

### 二、意图分类与子技能映射

```markdown
## 意图-子技能映射表

| 用户输入 | 诊断意图 | 核心子技能 | 辅助子技能 |
|---------|---------|-----------|-----------|
| `诊断 第X章` | 正文诊断 | auto-validation, quality-assurance | style-fingerprint, reader-retention |
| `大纲诊断` | 大纲诊断 | outline-evaluation, setting-detection | outline-plot-deep-diagnosis |
| `细纲诊断` | 细纲诊断 | quality-scoring, reverse-verification | consistency-check, linkage-optimization |
| `批量诊断` | 批量诊断 | batch-auto-fix | quality-scoring, auto-validation |
| `跨卷检测` | 跨卷诊断 | cross-volume-rhythm | 跨卷伏笔检测, 跨卷人物检测 |
| `仪表板` | 综合诊断 | dashboard | 所有子技能（按需加载） |
| `模拟评论` | 读者诊断 | reader-visualization | reader-retention |
| `疲劳检测` | 长篇诊断 | 长篇专项检测 | cross-volume-rhythm |
```

### 三、智能优先级排序

```markdown
## 子技能调用优先级

### 优先级计算公式
优先级 = 基础优先级 × 上下文系数 × 历史系数 × 关联系数

### 优先级分级
| 级别 | 标识 | 调用时机 |
|:---:|:---:|---------|
| P0 | 🔴 强制 | 必须调用，存在P0级问题 |
| P1 | 🟠 高优 | 应该调用，关联度高 |
| P2 | 🟡 标准 | 建议调用，提升完整性 |
| P3 | 🟢 可选 | 可选调用，性能优先时跳过 |

### 优先级判定规则
| 判定条件 | 优先级 | 示例 |
|---------|:---:|------|
| 存在P0级问题 | P0 | 战力崩坏、伏笔未回收 |
| 存在P1级问题 | P1 | 章节衔接不畅、爽点不足 |
| 关联子技能发现异常 | P1 | 细纲异常→大纲一致性检查 |
| 章节数≥50章 | P1 | 自动触发长篇专项检测 |
| 用户明确指定 | P2 | 用户请求特定诊断 |
| 性能敏感场景 | P3 | 连续批量诊断 |
```

### 四、智能路由执行

```markdown
## 智能路由执行流程

### 执行阶段
1️⃣ **输入解析**
   └─ 分析用户输入 → 判定诊断意图

2️⃣ **子技能识别**
   └─ 查询意图-子技能映射表 → 匹配核心/辅助子技能

3️⃣ **优先级排序**
   └─ 计算优先级 → 按优先级排序子技能

4️⃣ **依赖分析**
   └─ 分析子技能依赖关系 → 确定加载顺序

5️⃣ **并行加载**（可选）
   └─ 独立子技能并行执行 → 提升诊断效率

6️⃣ **顺序执行**（必要）
   └─ 依赖子技能按序执行 → 确保数据可用性

7️⃣ **结果聚合**
   └─ 聚合各子技能诊断结果 → 生成综合报告

8️⃣ **优化触发**
   └─ 分析诊断结果 → 触发相关优化功能
```

## 诊断协同引擎（v20.6新增）

> 实现多子技能协同诊断，确保诊断结果的完整性和一致性。

### 一、协同诊断模式

```markdown
## 协同诊断架构

### 星型模式（推荐）
```
           协同引擎
           /  │  \
          /   │   \
         /    │    \
        v     v     v
   子技能1 子技能2 子技能3
        \    │    /
         \   │   /
          \  │  /
           v  v  v
         结果聚合
```

### 链式模式
```
子技能1 → 子技能2 → 子技能3 → 结果聚合
   ↑           ↓
   └───────────┘
        数据反馈
```

### 并行模式
```
子技能1 ─┐
子技能2 ─┤─→ 结果聚合
子技能3 ─┘
```

### 混合模式（默认）
```
┌─────────────────────────────────────┐
│           协同引擎                    │
│  ┌─────────────┐   ┌────────────┐ │
│  │ 并行执行区   │   │ 顺序执行区  │ │
│  │ (独立技能)   │   │ (依赖技能)  │ │
│  │ auto-valid  │   │ 细纲诊断    │ │
│  │ style-fing  │──→│ 大纲诊断    │ │
│  │ reader-ret  │   │ 联动优化    │ │
│  └─────────────┘   └────────────┘ │
│           \           /            │
│            v         v             │
│       结果聚合 ← 数据总线            │
└─────────────────────────────────────┘
```

### 二、协同诊断规则

```markdown
## 协同诊断规则

### 1. 数据共享规则
| 数据类型 | 共享范围 | 共享方式 |
|---------|---------|---------|
| 大纲数据 | 所有子技能 | 统一数据总线 |
| 细纲数据 | 细纲/正文相关技能 | 统一数据总线 |
| 正文数据 | 正文诊断技能 | 统一数据总线 |
| 诊断结果 | 协同引擎 | 结果聚合 |

### 2. 依赖处理规则
| 依赖类型 | 处理方式 | 示例 |
|---------|---------|------|
| 数据依赖 | 前置子技能完成后加载 | 细纲评分→正文校验 |
| 结果依赖 | 前置结果聚合后执行 | 大纲评分→一致性检查 |
| 交叉依赖 | 迭代处理，限定2轮 | 大纲↔细纲双向校验 |

### 3. 冲突处理规则
| 冲突类型 | 处理方式 | 示例 |
|---------|---------|------|
| 评分冲突 | 取加权平均值 | auto-validation vs style-fingerprint |
| 建议冲突 | 优先级裁决 | 修复建议互相矛盾 |
| 数据冲突 | 以最新为准 | 不同子技能返回不一致数据 |
```

### 三、智能协调器

```markdown
## 智能协调器

### 协调功能
1. **任务分配**
   - 分析子技能依赖关系
   - 分配执行顺序和并行度
   - 监控执行状态

2. **数据协调**
   - 管理数据共享和传递
   - 处理数据格式转换
   - 维护数据一致性

3. **异常处理**
   - 检测子技能执行异常
   - 自动重试失败任务
   - 降级处理不可用子技能

4. **结果整合**
   - 聚合多子技能诊断结果
   - 消除重复诊断项
   - 生成统一诊断报告
```

## 大纲绑定强制机制

**前置检查**：诊断前必须执行
1. 读取大纲文件，按模块优先级提取：6.3章节摘要(核心情节/爽点类型/境界) → 6.2单元目标 → 5.2/5.4人物阶段 → 7.3伏笔
2. 读取细纲文件，确认细纲与大纲的模块映射关系
3. 读取前3-5章细纲和正文，确认人物状态连续性、伏笔一致性和剧情连贯性

**过程检查**：每个问题诊断必须标注
- **大纲依据**：[对照大纲哪个模块子节，如：6.3/5.2/7.3]
- **偏离程度**：[无偏离/轻微偏离/中等偏离/严重偏离]
- **是否违背大纲设定**：[是/否]

**建议检查**：每条优化建议必须包含
- **对照大纲要求**：[具体引用大纲模块原文]
- **修改后大纲一致性**：[是否仍符合大纲设定]
- **预期评分提升**：[量化指标]

**后置检查**：修改后必须重新验证
- 重新运行大纲一致性诊断
- 确认修改未引入新的设定冲突
- 验证人物状态连续性

| C级 | 及格，需较大调整 | 处理P0/P1级问题 |
| D级 | 不及格 | 全面重构 |

## 优先级排序机制

### 问题严重程度分级
| 级别 | 标识 | 定义 | 处理优先级 | 量化评分影响 |
| :--- | :--- | :--- | :--- | :--- |
| **P0** | 🔴 致命 | 严重毒点（虐主、降智、逻辑崩坏、设定冲突），直接导致读者弃书 | 立即修复 | -20分 |
| **P1** | 🟠 高 | 重要问题（节奏拖沓、爽点不足、人设OOC、战力崩坏），影响追读率 | 24小时内修复 | -10分 |
| **P2** | 🟡 中 | 一般问题（语言生硬、描写单薄、经济失衡），影响阅读体验 | 72小时内修复 | -5分 |
| **P3** | 🟢 低 | 细节问题（标点错误、用词不当），轻微影响体验 | 后续版本修复 | -2分 |

### 诊断优先级流程

**执行原则**：按优先级顺序执行，每步完成后评估是否需要触发后续步骤。P0级问题必须立即阻断并提示用户。

1. **第一步**：排查P0级致命问题（毒点、逻辑崩坏、设定冲突、角色境界冲突、伏笔编号重复）
   - 发现P0问题→立即输出告警→暂停后续诊断→等待用户确认修复方向
2. **第二步**：分析P1级重要问题（节奏、爽点、人设、战力、变更影响遗漏、英文混入）
3. **第三步**：检查P2级一般问题（语言、描写、经济、伏笔编号格式）
4. **第四步**：处理P3级细节问题
5. **第五步**：验证前后章衔接流畅度（读取前3-5章，逐项比对6项衔接指标）
6. **第六步**：验证大纲一致性（逐项比对5项一致性指标，标注偏离程度）
7. **第七步**：执行一致性规则诊断
8. **第八步**：执行神秘感与悬念层次诊断
9. **第九步**：执行世界观展开节奏诊断
10. **第十步**：执行人物群像深度诊断
11. **第十一步**：执行爽点质量诊断
12. **第十二步**：执行前期铺垫充分性诊断
13. **第十三步**：计算综合评分并定级
14. **第十四步**：执行技能自我诊断（加载 [shared/self-improvement.md](../shared/self-improvement.md)）
15. **第十五步**：大纲变更自动触发检测（加载 [mode-e/outline-plot-deep-diagnosis.md](mode-e/outline-plot-deep-diagnosis.md) 能力1）
    - 仅在检测到大纲实质性变更时触发
    - 输出变更影响报告，建议受影响章节范围
16. **第十六步**：番茄平台标准评估（加载 [mode-e/outline-plot-deep-diagnosis.md](mode-e/outline-plot-deep-diagnosis.md) 能力2）
    - 对当前诊断章节执行7维充实度量化
    - 判定扩充必要性等级，生成扩充方案
17. **第十七步**：大纲深度关联诊断（加载 [mode-e/outline-plot-deep-diagnosis.md](mode-e/outline-plot-deep-diagnosis.md) 能力3）
    - 提取章节设定元素→比对大纲覆盖→生成丰富方案
18. **第十八步**：跨章节联动修正检测（加载 [mode-e/outline-plot-deep-diagnosis.md](mode-e/outline-plot-deep-diagnosis.md) 能力4）
    - 仅在章节内容发生实质性变动时触发
    - 识别关联设定→生成级联修正方案→执行修正并验证
19. **第十九步**：触发优化功能（根据诊断结果自动触发对应优化模块）
    - **情感描写增强**：当人物群像深度评分<6或爽点质量评分<6时触发
    - **通感写作技法**：当写作质量评分<6或世界观展开节奏评分<6时触发
    - **长篇创作支持**：当大纲一致性评分<6或剧情结构评分<6时触发
