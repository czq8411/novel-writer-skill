### 优先级分级

| 级别 | 标识 | 调用时机 | 量化影响 |
|:---:|:---:|---------|:---:|
| P0 | 🔴 强制 | 必须调用，存在致命问题（战力崩坏、伏笔未回收） | -20分 |
| P1 | 🟠 高优 | 应该调用，关联度高（章节衔接不畅、爽点不足） | -10分 |
| P2 | 🟡 标准 | 建议调用，提升完整性 | -5分 |
| P3 | 🟢 可选 | 可选调用，性能优先时跳过 | -2分 |

### 执行流程

1. **输入解析** → 判定诊断意图  
2. **子技能识别** → 匹配核心/辅助子技能  
3. **优先级排序** → 计算优先级并排序  
4. **依赖分析** → 确定加载顺序  
5. **并行/顺序执行** → 独立技能并行，依赖技能按序  
6. **结果聚合** → 生成综合报告  
7. **优化触发** → 分析结果触发优化功能

### 协同模式

- **星型模式**：协同引擎为中心，子技能并行执行，结果汇总

## 自动化诊断修复模式

**触发词**：`自动修复`、`一键修复`、`自动优化`、`修复循环`

### 循环流程

```
诊断 → 分析问题 → 检查停止条件 → 自动修复 → 循环/停止
```

### 停止条件

| 条件 | 默认值 |
|-----|:---:|
| 连续2轮无P0/P1问题 | ✅ |
| 最大修复轮数 | 5轮 |
| 综合评分≥8分 | ✅ |
| 连续2轮评分无提升 | ✅ |

### 修复优先级

1. 🔴 P0级（致命问题）→ 立即修复  
2. 🟠 P1级（重要问题）→ 优先修复  
3. 🟡 P2级（一般问题）→ 按需修复  
4. 🟢 P3级（细节问题）→ 跳过/最后处理

### 控制命令

| 命令 | 功能 |
|------|------|
| `暂停` | 保存进度，可继续 |
| `继续` | 恢复暂停的循环 |
| `终止` | 保存进度，退出 |
| `取消` | 取消当前操作 |

### 异常处理

| 失败类型 | 处理方式 |
|---------|---------|
| 单问题修复失败 | 跳过，记录原因 |
| 连续失败≥3次 | 暂停等待用户确认 |
| 诊断失败 | 重试最多3次 |
| 系统异常 | 自动保存，安全退出 |

## 优先级排序机制

**执行原则**：按优先级顺序执行，P0级问题立即阻断。

1. **第一步**：排查P0级致命问题  
2. **第二步**：分析P1级重要问题  
3. **第三步**：检查P2级一般问题  
4. **第四步**：处理P3级细节问题  
5. **第五步**：验证前后章衔接流畅度  
6. **第六步**：验证大纲一致性  
7. **第七步**：执行剩余诊断维度  
8. **第八步**：计算综合评分并定级  
9. **第九步**：触发优化功能

### 优化功能触发规则

| 诊断条件 | 触发优化功能 |
|---------|-------------|
| 人物群像深度<6分 | 情感光谱系统 |
| 爽点质量<6分 | 情感光谱系统 |
| 写作质量<6分 | 通感写作技法 |
| 世界观展开节奏<6分 | 通感写作技法 |

## 大纲绑定强制机制

**前置检查**：诊断前必须执行
1. 读取大纲文件，提取核心情节/爽点类型/境界/人物阶段/伏笔  
2. 读取细纲文件，确认模块映射关系  
3. 读取前3-5章，确认人物状态连续性、伏笔一致性、剧情连贯性

**过程检查**：每个问题标注
- 大纲依据：[模块子节]  
- 偏离程度：[无偏离/轻微/中等/严重]  
- 是否违背大纲：[是/否]

**后置检查**：修改后重新验证大纲一致性

## 细纲诊断维度扩展

### 因果链完整性检查
| 检测项 | 标准 | 不通过处理 |
|--------|------|-----------|
| 前因缺失 | 每个转折有至少1处前文铺垫 | 标记为"因果断裂"，需补充铺垫 |
| 后果缺失 | 每个事件有后续影响 | 标记为"死胡同事件"，需设计后果 |
| 代价缺失 | 每次胜利有代价 | 标记为"无代价胜利"，需增加损失 |
| 因果链闭环 | 伏笔-揭示形成闭环 | 标记未闭环伏笔，规划回收章节 |
