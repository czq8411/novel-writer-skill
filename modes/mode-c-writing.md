## MCP联动
- **Filesystem**：读取当前章细纲+前一章结尾
- **Memory**：读取核心设定、剧情状态
- **Sequential Thinking**：推演因果链条，确保逻辑闭环
- **Memory**：更新剧情、伏笔、人物关系

---

## 写作流程
1. 加载写作流程模块，确认模式、章节、规则
2. 加载开头技巧模块，选择本章开头方式
3. 按需加载场景/角色/钩子模块
4. 正文创作（严格遵循细纲）
5. 完成后输入`诊断 第X章`路由至模式E

## 快速自检清单
- [ ] 细纲对齐：核心事件、爽点、人物状态一致？
- [ ] 开头有力：第一句是具体画面/动作？
- [ ] 感官丰富：至少2种非视觉感官描写？
- [ ] 对话自然：有潜台词？角色语音有辨识度？
- [ ] 心理克制：单次独白≤3行？穿插动作？
- [ ] 元数据隔离：无伏笔编号泄漏？
- [ ] 禁忌词检查：无AI味词汇？
- [ ] 钩子有力：章末300字有悬念/危机/反转？
- [ ] 移动端适配：每段≤3行？对话独立成段？
- [ ] 前文衔接：场景、状态、钩子自然承接？

## 文件输出与目录管理

### 目录结构
```
当前工作目录/
├── 大纲/
│   └── [书名]_大纲.md
├── 细纲/
│   ├── 第一卷_[卷名]/
│   │   └── [卷名]_细纲.md
│   └── ...
├── 正文/
│   ├── 第一卷_[卷名]/
│   │   ├── 第001章_[章节名].md
│   │   ├── 第002章_[章节名].md
│   │   └── ...
│   └── ...
└── metadata.json
```

### 文件输出规则
- **章节文件**：`正文/[卷名]/第[3位章节号]章_[章节名].md`
- **文件名格式**：`第001章_XXX.md`（章节号补零至3位）
- **触发条件**：每章创作完成后自动保存
- **文件格式**：UTF-8编码，Markdown格式

### 保存失败降级策略
| 失败类型 | 处理方式 |
|---------|---------|
| 目录创建失败 | 提示权限不足，引导用户检查目录权限 |
| 文件写入失败 | 重试1次，仍失败则提示用户手动保存 |
| 文件名非法 | 自动替换非法字符为 `-` |

### MCP联动增强
- **Filesystem**：自动创建卷目录并保存章节文件
- **Memory**：记录当前章节号、字数、修改时间等元数据
