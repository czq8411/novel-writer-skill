## 特殊场景模板库（v22.1新增）

> 以下模板覆盖四大高价值叙事场景。当正文创作遇到对应场景类型时，加载对应模板以确保情感峰值不降级。

### 模板1：角色退场场景（POV退场）

**适用时机**：核心配角/主要角色的终章、牺牲、永别场景

```
## 退场场景写作模板

### 写作前核对
- < > 该角色的核心性格是什么？（退场行为必须符合性格）
- < > 该角色与主角的核心羁绊是什么？（退场需要回扣羁绊）
- < > 该角色退场后对主线的影响是什么？（不是「死了就完了」）

### 四拍结构
| 拍节 | 内容 | 写作要点 |
|:----:|------|---------|
| 第一拍·回顾 | 3句话以内，点明该角色与主角的核心羁绊 | 用具体物件/地点触发回忆，不写旁白 |
| 第二拍·抉择 | 角色面对牺牲/放手/守护的瞬间 | 不说「我选择」，用动作（推开门/松开手/放下剑） |
| 第三拍·台词 | 最后一句台词，必须呼应角色的核心性格或全书主题 | 控制在10字以内，越短越有力 |
| 第四拍·余味 | 一个不写情绪的微小动作（倒下的茶杯/飘落的树叶/未送出的信） | 不要写心理描写，只写客观画面 |

### 执行自检
- < > 退场场景是否让读者觉得「这是这个人应有的结局」而不是「剧情杀」？
- < > 退场后是否有至少1章的「情绪留白」让读者消化（而不是立即跳入下一场战斗）？
- < > 退场的余味镜头是否能成为全书读者讨论时的「标志性画面」？
```

### 模板2：金句呼应场景

**适用时机**：全书核心金句/主题在不同阶段的重复出现，构成呼应弧线

```
## 金句呼应写作模板

### 全书金句定义
- **核心金句**：<<一句话，必须能概括全书主题>>
- **金句的种子版本**：<<原始版本（更短/更含蓄，在第一阶段出现）>>
- **金句的完整版本**：<<完整版本（在决战/高潮阶段出现）>>

### 五段呼应弧线设计
| 呼应轮次 | 章节 | 情感阶段 | 呼应用词 | 写作手法 | 预期效果 |
|:-------:|:----:|:-------:|:--------:|:--------:|:-------:|
| 种子 | 第X章 | 故事早期 | 原始版本 | 借他人之口说出，主角不一定听懂 | 埋下种子，读者初次接触 |
| 成长 | 第X章 | 中期突破 | 变体版本 | 主角自己说出，但还不确定 | 读者意识到「这句话在变】 |
| 至暗 | 第X章 | 最低谷 | 残缺版本（话只说一半） | 主视角崩溃时低语，话没说完 | 读者担心「会不会就这样结束」 |
| 升华 | 第X章 | 最终决战 | 完整版本 | 主角在决胜时刻完整说出 | 全书情感峰值，种子终于开花 |
| 回声 | 第X章 | 尾声 | 原始版本（致敬第1次） | 由其他人/环境回声引出 | 平静的圆满感 |

### 执行自检
- < > 每一次呼应在情感积累上是否比上一次更强？（不能递减）
- < > 至暗时刻的「残缺版本」是否让读者感受到主角的脆弱？
- < > 升华时刻的完整版本是否让读者觉得「这句话值得等N章」？
```

### 模板3：身份揭示场景

**适用时机**：主角/配角发现自己与某个人物/势力的隐藏关系

```
## 身份揭示写作模板

### 六拍结构
| 节奏节拍 | 内容 | 写法要点 |
|:--------:|------|---------|
| ① 触发 | <<什么触发了揭示（物件/对话/环境）>> | 感官先行，信息后至。「他看到...他听到...然后才意识到」 |
| ② 怀疑 | <<主角开始产生怀疑>> | 用动作代替心理描写：「他的手停在半空」而不是「他心里一惊」 |
| ③ 确认 | <<揭开真相的那一刻>> | 用环境/配角反应衬托内心震撼，不写主角的心理活动 |
| ④ 对抗 | <<主角面对真相的内心斗争>> | 短句+沉默+细节，给读者留白空间 |
| ⑤ 接受 | <<做出选择>> | 不说「我接受」——用行动表示：「他抬起头」/「他站了起来」 |
| ⑥ 余味 | <<走出房间/走出世界>> | 环境暗示改变了（天亮了/下雨了/镜子裂了/鸟不叫了） |

### 执行自检
- < > 揭示的真相是否在前文有至少3处伏笔？（没有伏笔的揭示是「机械降神」）
- < > 真相是否比读者猜测的更精彩？（否则读者会失望）
- < > 揭示之后的角色选择是否让读者觉得「换做是我也会这么选」？
```

## 写作流程

### 阶段一：细纲约束加载（v21.1新增）

> **核心改进**：强化细纲到正文的约束传递，防止约束衰减

#### 1.1 强制细纲读取

**必须读取的文件**（按优先级）：
| 优先级 | 文件 | 用途 | 缺失处理 |
|:------:|------|------|---------|
| P0 | 当前章细纲 | 核心事件、爽点、钩子 | 报错，要求生成细纲 |
| P0 | 前1章细纲 | 确保连贯性 | 跳过 |
| P1 | 前3章细纲 | 伏笔追踪 | 跳过 |
| P2 | 大纲文件 | 世界观、人物设定 | 跳过 |

#### 1.2 细纲约束提取

从细纲中提取以下**强制约束项**：

```yaml
约束清单:
  核心事件:
    必须呈现: true
    允许偏离: ≤10%
    检查点: 开头/中段/结尾
  
  爽点设计:
    类型: <必须匹配细纲>
    强度: <必须≥细纲设定-1>
    位置: <必须出现在细纲指定位置>
    配角反应: <必须呈现>
  
  伏笔管理:
    必须回收: <列表>
    必须埋设: <列表>
    禁止提前回收: <列表>
  
  人物状态:
    主角状态: <必须匹配>
    关键配角: <必须匹配>
    关系变化: <如有必须呈现>
  
  章末钩子:
    类型: <悬念/危机/反转>
    强度: <必须≥细纲设定-1>
    位置: <最后300字内>
```

#### 1.3 约束验证清单（生成前）

```markdown
## 细纲约束确认

### 本章必须呈现的内容
- < > 核心事件：<细纲中的核心事件描述>
- < > 爽点类型：<类型>，强度：<X/10>
- < > 伏笔回收：<列表>
- < > 伏笔埋设：<列表>
- < > 章末钩子：<类型>，强度：<X/10>

### 人物状态基准
- 主角：<当前状态>
- 关键配角：<状态列表>

### 约束违反处理
如生成过程中发现无法满足约束：
1. 立即暂停生成
2. 标记冲突项
3. 提供解决方案（A.调整细纲 B.强制满足 C.用户决策）
```

### 阶段二：正文创作（约束强制执行）

2. 加载写作流程模块，确认模式、章节、规则
3. 加载开头技巧模块，选择本章开头方式
4. **加载AI味防控模块，应用强化生成约束**
5. **正文创作（强制遵循细纲约束）**
   - 每完成一个段落，自检是否偏离细纲
   - 爽点段落必须达到强度要求
   - 钩子必须在最后300字内呈现

### 阶段三：细纲对齐验证（v21.1新增）

> **核心改进**：生成后自动验证与细纲的一致性

#### 5.1 约束满足度检查

| 检查项 | 权重 | 通过标准 | 未通过处理 |
|--------|:---:|:-------:|-----------|
| 核心事件完整度 | 30% | ≥90% | 标记重写 |
| 爽点强度达标 | 25% | ≥细纲设定-1 | 强化爽点 |
| 伏笔正确性 | 20% | 100% | 修正或标记 |
| 钩子强度 | 15% | ≥细纲设定-1 | 强化钩子 |
| 人物一致性 | 10% | 无OOC | 修正或标记 |

#### 5.2 细纲偏离度评分

```
细纲对齐评分 = Σ(检查项得分 × 权重)

评分等级:
- 9-10分: ✅ 优秀，直接通过
- 7-8分: ⚠️ 良好，建议微调
- 5-6分: 🔶 一般，需要修改
- <5分: 🔴 不合格，必须重写
```

#### 5.3 自动修正机制

**初稿评分<7分时触发**：
1. 识别偏离项
2. 生成修正方案
3. 自动重写偏离部分
4. 重新评分
5. 最多3轮，仍不达标则标记人工处理

### 阶段四：AI味检测与优化

6. **【自动执行】AI味检测与优化**
   - 初稿AI味≤3.0：直接通过
   - 初稿AI味3.0-4.0：普通优化
   - **初稿AI味>4.0：强化优化（逐段重写，最多3轮）**

### 阶段四（新增）：章节内高潮与伏笔检测

> **v21.2新增**：章节内级别的质量保障机制

#### 7. 章节内高潮检测

| 检测项 | 阈值 | 触发条件 | 处理方式 |
|-------|------|---------|---------|
| 高潮数量 | ≥1个/章 | 章节内无高潮预警 | 提示"本章缺少高潮" |
| 高潮位置 | 前/中/后 | 高潮位置与预期不符 | 调整高潮位置 |
| 高潮强度 | ≥细纲-2 | 高潮情感峰值不足 | 强化情感描写 |

**高潮识别规则**：
```
高潮特征标记：
- 战斗类：力量爆发、技能释放、绝境逆袭
- 情感类：告白、离别、重逢、揭秘
- 获得类：获得宝物、突破境界、身份揭露
- 反转型：真相揭示、计划成功、阴谋败露
```

#### 8. 章节内伏笔密度检测

| 检测项 | 阈值 | 触发条件 | 处理方式 |
|-------|------|---------|---------|
| 伏笔密度 | 2-5个/章 | 伏笔过少/过多预警 | 提示调整建议 |
| 伏笔埋设 | 每3-5章1个核心 | 埋设间隔过密/稀疏 | 标记优化 |
| 伏笔回收 | 回收率≥90% | 回收率过低 | 提示回收建议 |

**伏笔密度检测算法**：
```
伏笔检测规则：
- 扫描正文中的伏笔标记（如F001等）
- 统计本章埋设/回收数量
- 与前3章对比，检测伏笔密度异常
- 标记密度过高（影响阅读流畅）的伏笔
- 标记密度过低（节奏平淡）的章节
```

#### 9. 情感节奏检测

| 检测项 | 阈值 | 触发条件 | 处理方式 |
|-------|------|---------|---------|
| 情感峰值 | ≥2个/章 | 情感平淡预警 | 提示增加情感起伏 |
| 情感曲线 | 起点→升温→高潮→回落 | 曲线异常 | 建议调整节奏 |
| 情绪基调 | 与细纲匹配 | 基调偏离 | 提示修正 |

**检测输出格式**：
```markdown
## 章节内检测报告

### 高潮检测结果
- ✅ 高潮数量：<X>个（达标/不足）
- ✅ 高潮位置：<前/中/后段>
- ✅ 高潮强度：<X/10>

### 伏笔检测结果
- ✅ 伏笔密度：<X>个/章（正常/过密/稀疏）
- ✅ 埋设数量：<X>个
- ✅ 回收数量：<X>个

### 情感节奏结果
- ✅ 情感峰值：<X>个（达标/平淡）
- ✅ 情感曲线：<类型>
- ✅ 情绪基调：<基调>（匹配/偏离）

### 综合评估
- 章节质量评级：<S/A/B/C>
```

## 六、正文格式规范（v20.8新增）

### 6.1 正文纯净度要求

**严禁出现在正文中的内容**：

| 禁止项 | 示例 | 说明 |
|--------|------|------|
| **黏合元素标注** | `（情节回扣）`、`（感官叠接）`、`（时间桥接）` | 这些是写作指导用语，不是正文内容 |
| **Markdown分隔线** | `---` | 会造成视觉断裂，影响阅读体验 |
| **技能内部注释** | `【黏合】`、`【钩子】`、`【爽点】` | 所有技能标记都必须在生成前移除 |
| **细纲模块编号** | `<6.3>`、`F001` | 伏笔编号等元数据不得泄漏到正文 |
| **写作指导文本** | "本章开头使用了时间桥接词" | 任何解释性文字都不应出现在正文 |

### 6.2 正确的黏合开头写法

**错误示例**（包含标注）：
```markdown
照片还贴在胸口。（情节回扣）

沈默保持着这个姿势，在椅子上坐了一整夜。（空间站桩）
那股复杂的情绪还在胸腔里翻涌——震惊、疑惑、一丝难以言喻的希望。（情绪延续）
```

**正确示例**（纯净正文）：
```markdown
照片还贴在胸口。

沈默保持着这个姿势，在椅子上坐了一整夜。那股复杂的情绪还在胸腔里翻涌——震惊、疑惑、一丝难以言喻的希望。
```

### 6.3 章节结构规范

**正确的章节结构**：
```markdown
# 第002章：没有鬼

<开头段落：直接承接上文，无标注>

<发展段落：情节推进>

<高潮段落：爽点呈现>

<结尾段落：钩子设置>

---

**【第002章完】**

**字数：约2900字**

---

**章节传递卡片 - 第002章 → 第003章**
```

**注意**：
- 正文结束后使用 `---` 分隔线（这是元数据区域，不属于正文）
- CPC（章节传递卡片）放在分隔线之后，用于下一章参考

### 6.4 写作时的自检口诀

在生成正文前，默念三遍：
1. **"我是作家，不是编辑器"** —— 不要把写作指导语写进正文
2. **"读者看不到我的思考过程"** —— 只呈现最终的故事内容
3. **"黏合要自然，不要贴标签"** —— 让读者感受到连贯性，而不是读到标注

| 目录创建失败（其他原因） | 重试1次，仍失败则提示用户手动保存 |

| 文件名非法 | 自动替换非法字符为 `-` |

### MCP联动增强
- **Filesystem**：自动创建卷目录并保存章节文件，支持动态路径
- **Memory**：记录当前章节号、字数、修改时间、实际保存路径等元数据
