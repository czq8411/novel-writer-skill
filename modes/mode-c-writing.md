## 写作流程

### 阶段一：细纲约束加载（v21.1新增）

> **核心改进**：强化细纲到正文的约束传递，防止约束衰减

#### 1.1 强制细纲读取

**必须读取的文件**（按优先级）：
| 优先级 | 文件 | 用途 | 缺失处理 |
|:------:|------|------|---------|
| P0 | 当前章细纲 | 核心事件、爽点、钩子 | 报错，要求生成细纲 |
| P0 | 前1章细纲 | 确保连贯性 | 跳过 |
| P1 | 前3章细纲 | 伏笔追踪 | 跳过 |
| P2 | 大纲文件 | 世界观、人物设定 | 跳过 |

#### 1.2 细纲约束提取

从细纲中提取以下**强制约束项**：

```yaml
约束清单:
  核心事件:
    必须呈现: true
    允许偏离: ≤10%
    检查点: 开头/中段/结尾
  
  爽点设计:
    类型: [必须匹配细纲]
    强度: [必须≥细纲设定-1]
    位置: [必须出现在细纲指定位置]
    配角反应: [必须呈现]
  
  伏笔管理:
    必须回收: [列表]
    必须埋设: [列表]
    禁止提前回收: [列表]
  
  人物状态:
    主角状态: [必须匹配]
    关键配角: [必须匹配]
    关系变化: [如有必须呈现]
  
  章末钩子:
    类型: [悬念/危机/反转]
    强度: [必须≥细纲设定-1]
    位置: [最后300字内]
```

#### 1.3 约束验证清单（生成前）

```markdown
## 细纲约束确认

### 本章必须呈现的内容
- [ ] 核心事件：[细纲中的核心事件描述]
- [ ] 爽点类型：[类型]，强度：[X/10]
- [ ] 伏笔回收：[列表]
- [ ] 伏笔埋设：[列表]
- [ ] 章末钩子：[类型]，强度：[X/10]

### 人物状态基准
- 主角：[当前状态]
- 关键配角：[状态列表]

### 约束违反处理
如生成过程中发现无法满足约束：
1. 立即暂停生成
2. 标记冲突项
3. 提供解决方案（A.调整细纲 B.强制满足 C.用户决策）
```

### 阶段二：正文创作（约束强制执行）

2. 加载写作流程模块，确认模式、章节、规则
3. 加载开头技巧模块，选择本章开头方式
4. **加载AI味防控模块，应用强化生成约束**
5. **正文创作（强制遵循细纲约束）**
   - 每完成一个段落，自检是否偏离细纲
   - 爽点段落必须达到强度要求
   - 钩子必须在最后300字内呈现

### 阶段三：细纲对齐验证（v21.1新增）

> **核心改进**：生成后自动验证与细纲的一致性

#### 5.1 约束满足度检查

| 检查项 | 权重 | 通过标准 | 未通过处理 |
|--------|:---:|:-------:|-----------|
| 核心事件完整度 | 30% | ≥90% | 标记重写 |
| 爽点强度达标 | 25% | ≥细纲设定-1 | 强化爽点 |
| 伏笔正确性 | 20% | 100% | 修正或标记 |
| 钩子强度 | 15% | ≥细纲设定-1 | 强化钩子 |
| 人物一致性 | 10% | 无OOC | 修正或标记 |

#### 5.2 细纲偏离度评分

```
细纲对齐评分 = Σ(检查项得分 × 权重)

评分等级:
- 9-10分: ✅ 优秀，直接通过
- 7-8分: ⚠️ 良好，建议微调
- 5-6分: 🔶 一般，需要修改
- <5分: 🔴 不合格，必须重写
```

#### 5.3 自动修正机制

**初稿评分<7分时触发**：
1. 识别偏离项
2. 生成修正方案
3. 自动重写偏离部分
4. 重新评分
5. 最多3轮，仍不达标则标记人工处理

### 阶段四：AI味检测与优化

6. **【自动执行】AI味检测与优化**
   - 初稿AI味≤3.0：直接通过
   - 初稿AI味3.0-4.0：普通优化
   - **初稿AI味>4.0：强化优化（逐段重写，最多3轮）**

### 阶段五：优秀标准验证

7. **【自动执行】优秀标准验证**
   - 句式多样性≤40%？
   - 五感视觉占比≤50%？
   - 情感层次达标？
   - 细纲对齐评分≥7分？（v21.1新增）
   - 未达标则再次优化或标记重写

### 阶段六：输出与保存

8. 输出最终正文（含AI味评分、质量指标、**细纲对齐评分**）
9. **【自动执行】保存文件到指定目录**
10. （可选）用户输入`诊断 第X章`查看详细诊断报告

### 自动保存规则（v20.8-patch新增）

> 每章正文生成并优化完成后，必须自动保存到文件系统，无需用户手动确认。

#### 保存触发时机

| 时机 | 行为 | 用户提示 |
|------|------|---------|
| **单章生成完成后** | 自动检测路径→创建目录→保存文件 | 显示保存路径和文件大小 |
| **连续创作每章后** | 静默保存，仅显示进度 | 简洁进度提示 |
| **批量生成完成后** | 保存所有章节→显示汇总 | 显示总字数和文件列表 |

#### 保存路径规则

```
1. 获取目标路径（优先级：用户指定 > 当前工作目录 > 默认路径）
2. 检查路径权限
3. 无权限时自动降级到默认路径
4. 创建目录结构（如不存在）：正文/[卷名]/
5. 生成文件名：第XXX章_[章节标题].md
6. 执行保存（UTF-8编码）
7. 验证保存成功（检查文件存在+大小>0）
```

#### 保存成功提示格式

**单章保存成功**：
```
✅ 第004章保存成功
   路径：/Users/.../诡雾之下/正文/第一卷_诡雾初现/第004章_U盘里的真相.md
   大小：6.1KB
```

**连续创作保存成功**：
```
✅ 第005章完成并保存（3200字，AI味3.2）→ 自动进入第006章...
```

**保存失败处理**：
```
🔴 第004章保存失败
   错误：权限不足 / 磁盘空间不足 / 路径不存在
   
   建议：
   - 检查路径权限
   - 更换保存路径
   - 手动保存正文内容
```

#### 文件内容格式

保存的文件包含：
1. 章节标题（# 第X章：标题）
2. 纯净正文内容（无黏合标注、无技能注释、**无markdown分隔线**）
3. 章节结束标记（**【第X章完】**）
4. 字数统计
5. （可选）章节传递卡片（CPC）

**⚠️ 重要：正文纯净度规范（v20.8-patch2新增）**

正文内容中**严禁**出现以下元素：
- ❌ `---` 分隔线（仅允许在正文结束后、元数据前使用）
- ❌ `***` 分隔线
- ❌ `___` 分隔线
- ❌ `===` 标题线
- ❌ 任何markdown格式符号（如 `**强调**`、`> 引用`）
- ❌ HTML标签（如 `<br>`、`<div>`）

**正确的场景切换写法**：
```markdown
❌ 错误：使用分隔线
---
记忆到这里断裂了。
---

✅ 正确：使用自然过渡
记忆到这里断裂了。

沈默猛地睁开眼睛...
```

**正确的章节结构**：
```markdown
# 第006章：标题

[开头段落，直接承接上文]

[发展段落，情节推进]

[高潮段落，爽点呈现]

[结尾段落，钩子设置]

---
**【第006章完】**

**字数：约XXXX字**

---
**章节传递卡片...**
```

**注意**：`---` 只能出现在正文结束后，用于分隔正文和元数据（章节结束标记、字数统计、CPC）。正文内部禁止使用任何分隔线！

#### 保存验证

保存后自动验证：
- [ ] 文件存在
- [ ] 文件大小 > 0
- [ ] 文件可读
- [ ] 内容完整性（包含标题和结束标记）

验证失败时：
1. 重试保存（最多2次）
2. 仍失败则提示用户手动保存
3. 输出正文内容到对话，让用户复制保存

- [ ] 前文衔接：场景、状态、钩子自然承接？
- [ ] **正文纯净度：无黏合标注、无分隔线、无技能注释？** ⚠️ 关键检查项
  - 检查正文内部是否有 `---` / `***` / `___` 等分隔线

### AI味防控检查（v20.8新增）
- [ ] **句式多样性**：主谓宾结构占比≤50%？有倒装/省略/被动等变异句式？

- [ ] **情感层次**：无200字以上单一情绪？展示情绪矛盾性？
- [ ] **Cliché 检查**：无"血液凝固"/"眼泪滑落"/"像狼一样"等套路表达？
- [ ] **词汇独特性**：避免高频词重复？有独特用词和比喻？
- [ ] **世界观铺设**：第1章明确世界观背景？类型特征清晰？

## 六、正文格式规范（v20.8新增）

### 6.1 正文纯净度要求

**严禁出现在正文中的内容**：

| 禁止项 | 示例 | 说明 |
|--------|------|------|
| **黏合元素标注** | `（情节回扣）`、`（感官叠接）`、`（时间桥接）` | 这些是写作指导用语，不是正文内容 |
| **Markdown分隔线** | `---` | 会造成视觉断裂，影响阅读体验 |
| **技能内部注释** | `【黏合】`、`【钩子】`、`【爽点】` | 所有技能标记都必须在生成前移除 |
| **细纲模块编号** | `[6.3]`、`F001` | 伏笔编号等元数据不得泄漏到正文 |
| **写作指导文本** | "本章开头使用了时间桥接词" | 任何解释性文字都不应出现在正文 |

### 6.2 正确的黏合开头写法

**错误示例**（包含标注）：
```markdown
照片还贴在胸口。（情节回扣）

沈默保持着这个姿势，在椅子上坐了一整夜。（空间站桩）
那股复杂的情绪还在胸腔里翻涌——震惊、疑惑、一丝难以言喻的希望。（情绪延续）
```

**正确示例**（纯净正文）：
```markdown
照片还贴在胸口。

沈默保持着这个姿势，在椅子上坐了一整夜。那股复杂的情绪还在胸腔里翻涌——震惊、疑惑、一丝难以言喻的希望。
```

### 6.3 章节结构规范

**正确的章节结构**：
```markdown
# 第002章：没有鬼

[开头段落：直接承接上文，无标注]

[发展段落：情节推进]

[高潮段落：爽点呈现]

[结尾段落：钩子设置]

---

**【第002章完】**

**字数：约2900字**

---

**章节传递卡片 - 第002章 → 第003章**
```

**注意**：
- 正文结束后使用 `---` 分隔线（这是元数据区域，不属于正文）
- CPC（章节传递卡片）放在分隔线之后，用于下一章参考

### 6.4 写作时的自检口诀

在生成正文前，默念三遍：
1. **"我是作家，不是编辑器"** —— 不要把写作指导语写进正文
2. **"读者看不到我的思考过程"** —— 只呈现最终的故事内容
3. **"黏合要自然，不要贴标签"** —— 让读者感受到连贯性，而不是读到标注

## 七、写作规则

### 转折合理性检查
每个情节转折必须通过以下验证：
- [ ] 转折在前文是否有至少1处伏笔/铺垫？
- [ ] 转折后果是否在后续剧情中有量化体现？
- [ ] 转折是否改变了至少一个角色的处境/认知/关系？
- [ ] 转折后读者是否会感到"原来如此"而非"莫名其妙"？

## 文件输出与目录管理

### 目录结构
```
当前工作目录/
├── 大纲/
│   └── [书名]_大纲.md
├── 细纲/
│   ├── 第一卷_[卷名]/
│   │   └── [卷名]_细纲.md
│   └── ...
├── 正文/
│   ├── 第一卷_[卷名]/
│   │   ├── 第001章_[章节名].md
│   │   ├── 第002章_[章节名].md
│   │   └── ...
│   └── ...
└── metadata.json
```

### 文件输出规则
- **章节文件**：`正文/[卷名]/第[3位章节号]章_[章节名].md`
- **文件名格式**：`第001章_XXX.md`（章节号补零至3位）
- **触发条件**：每章创作完成后自动保存
- **文件格式**：UTF-8编码，Markdown格式
- **路径支持**：支持用户指定保存路径，详见下方"动态路径支持"

### 动态路径支持（v20.8新增）

**用户指定路径方式**：
- 在请求中明确指定路径，如：`将文件保存到 /Users/xxx/小说/`
- 使用当前工作目录（自动检测）
- 使用默认路径（有MCP权限的目录）

**路径解析优先级**：
```
1. 用户明确指定的绝对路径
2. 当前工作目录（pwd）
3. 默认路径：/Users/chenzhiqiang/Desktop/T/document/novel_1
```

**自动降级机制**：

当用户指定的路径无MCP写入权限时：
1. 尝试创建目录，捕获权限错误
2. 自动降级到默认路径
3. 向用户显示路径降级提示
4. 继续完成文件保存

**路径降级提示格式**：
```markdown
⚠️ 路径权限提示
─────────────────────────────────
目标路径: [用户指定路径]
状态: ❌ 无MCP写入权限

✅ 已使用备选方案保存到目标路径！

使用 RunCommand 工具成功创建文件，无路径限制。

📁 文件保存位置:
- 大纲: [用户指定路径]/大纲/
- 细纲: [用户指定路径]/细纲/
- 正文: [用户指定路径]/正文/
─────────────────────────────────
```

### 备选方案：使用 RunCommand 工具（v20.8新增）

当 MCP Filesystem 工具无法访问目标路径时，自动切换到 **RunCommand** 工具：

**实现方式**：
```bash
# 创建目录结构
mkdir -p [目标路径]/[书名]/{大纲,细纲/第一卷_XXX,正文/第一卷_XXX}

# 创建文件
echo "[文件内容]" > [目标路径]/[书名]/正文/[章节文件].md
```

**优势**：
- ✅ 无路径限制，可以保存到任意目录
- ✅ 支持动态路径，如 `/Users/xxx/Desktop/小说/novel_2/`
- ✅ 无需修改MCP配置
- ✅ 自动创建完整目录结构

**使用示例**：
```
用户：将小说保存到 /Users/chenzhiqiang/Desktop/T/document/novel_3/
AI：使用 RunCommand 创建目录和文件...
     ✅ 成功保存到 /Users/chenzhiqiang/Desktop/T/document/novel_3/诡雾之下/
```

### 保存失败降级策略
| 失败类型 | 处理方式 |
|---------|---------|
| 目录创建失败（权限不足） | 自动降级到默认路径，提示用户 |
| 目录创建失败（其他原因） | 重试1次，仍失败则提示用户手动保存 |
| 文件写入失败 | 重试1次，仍失败则提示用户手动保存 |
| 文件名非法 | 自动替换非法字符为 `-` |

### MCP联动增强
- **Filesystem**：自动创建卷目录并保存章节文件，支持动态路径
- **Memory**：记录当前章节号、字数、修改时间、实际保存路径等元数据
