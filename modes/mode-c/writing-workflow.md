# 模式C子模块：写作流程与项目管理

> 按需加载：首次进入模式C或切换写作模式时加载本文件。
> 全局规则：遵循主控文件第二章核心铁律
> **重要**：第2章起必须严格执行【上下文加载流程】，详见下方"上下文加载（第2章起强制）"

## 一、写作模式

| 模式 | 参数值 | 适用场景 | 说明 |
|------|--------|---------|------|
| **串行**（默认） | `serial` | 单章精写 | 逐章创作，质量优先 |
| **批量串行** | `batch` | 连续多章 | 一次性生成N章细纲后逐章创作正文 |
| **并行** | `parallel` | 多章同时产出 | 将连续N章分配给子Agent并行创作，通过`writing-plan.json`协调状态 |
| **连续创作** | `continuous` | 快速产出完整作品 | 确认起始章节后自动逐章创作直到完成，仅在发现问题时暂停 |

**并行模式额外要求**：
- 创作前统一读取大纲+细纲+人物设定，确保各Agent共享同一设定基线
- 创作后执行文风一致性校验：对比各章语言风格、人物性格、叙事节奏，偏差超过阈值则统一润色

### 连续创作模式详解

用户确认起始章节和终止章节（或"写到完"）后，AI自动逐章创作，全程无需逐章确认。

```
连续创作流程：
1. 用户确认：起始章节 + 终止章节（或"写到完"）
2. AI逐章创作，每章完成后静默执行7维自动校验
3. 校验通过 → 自动进入下一章，输出简短进度提示
4. 校验不通过 → 自动修复（最多2轮），仍不通过则暂停询问用户
5. 每5章输出一次进度摘要（含字数统计、爽点密度、伏笔状态）
6. 全部完成 → 输出创作总结
```

**连续创作进度提示格式**：
```
✅ 第042章完成（3850字）→ AI味检测通过 → 自动保存 → 自动进入第043章...
✅ 第043章完成（4100字）→ AI味检测通过 → 自动保存 → 自动进入第044章...
⚠️ 第044章字数不足（2800字），自动扩充中...
✅ 第044章修复完成（3650字）→ AI味检测通过 → 自动保存 → 自动进入第045章...
⚠️ 第046章AI味评分7.2（偏高），自动优化中...
✅ 第046章AI味优化完成（评分4.1）→ 自动保存 → 自动进入第047章...

📊 进度摘要（第041-045章）
| 章节 | 字数 | 爽点 | 伏笔 | AI味 | 保存 | 校验 |
|------|------|------|------|------|------|------|
| 041 | 4200 | ✅ | F12回收 | 3.2✅ | ✅ | ✅ |
| 042 | 3850 | ✅ | — | 2.8✅ | ✅ | ✅ |
| 043 | 4100 | ✅ | F15埋下 | 3.5✅ | ✅ | ✅ |
| 044 | 3650 | ✅ | — | 4.2✅ | ✅ | ⚠️修复后通过 |
| 045 | 3900 | ✅ | — | 2.9✅ | ✅ | ✅ |
```

**连续创作暂停条件**（仅在以下情况暂停询问用户）：
- 自动修复2轮后校验仍不通过
- AI味评分>7.0且自动优化2轮后仍>6.0
- 检测到与大纲/细纲的重大偏离
- 检测到人物OOC（性格行为严重不符）
- 检测到章间连贯性校验失败（开头黏合元素不足3个）
- 用户主动输入"暂停"

## 二、上下文加载（第2章起强制）

> **这是解决"每章像独立故事"问题的核心机制**。写第2章及以后章节时，必须按以下流程加载上下文。

### 2.1 加载流程

```
Step 1：读取上一章CPC（章节传递卡片）── 获取所有结构化状态信息
    ↓
Step 2：读取上一章全文最后800-1000字 ── 获取具体场景/对话/情绪的原文质感
    ↓
Step 3：读取大纲/细纲中本章定位 ── 获取本章核心事件
    ↓
Step 4：确认无冲突 ── CPC + 大纲 + 细纲三方信息交叉验证
    ↓
Step 5：开始写作
```

### 2.2 每个加载步骤的详细要求

#### Step 1：读取CPC（必须提取的信息）

上一章末尾必须包含CPC（章节传递卡片），从中提取：

| 必提取项 | 用途 | 本章必须体现 |
|---------|------|-------------|
| 上一章结尾场景（时间+空间） | 确定本章开头的时空位置 | 本章开头必须从此时此地开始 |
| 上一章结尾情绪 | 本章开头延续该情绪或自然过渡 | 开头必须体现该情绪或其直接延续 |
| 上一章未解决冲突 | 本章前1000字必须推进或提及 | 必须出现相关人物或事件 |
| 角色状态快照 | 位置/情绪/伤势/物品/关系绑定 | 角色状态必须与CPC一致 |
| 待回收伏笔清单 | 按时效推进或回收 | 检查是否有需要在本章处理的伏笔 |
| 钩子信息 | 章末钩子+下一章何时揭示 | 本章开头必须回应或延续钩子 |

#### Step 2：读取上一章结尾800-1000字（原文质感延续）

**为什么从500字增加到800-1000字？**
- 500字往往只包含结尾的高潮或对话，缺少足够的上下文
- 800-1000字能覆盖结尾的完整场景，包括：场景铺垫、关键对话/动作、情绪变化、结尾意象

**阅读重点**：
- 感受语言节奏、句式特点、氛围基调
- 记住结尾意象（"烛火""雨声""风声"），本章可自然复用
- 确认最后一句话的具体内容，本章开头必须能直接承接

#### Step 3：读取细纲（本章定位）

确认本章在细纲中的：
- 核心事件
- 场景切换要求
- 人物状态变化
- 伏笔操作（回收/埋下）

#### Step 4：交叉验证（防冲突检查）

| 检查项 | 检查方法 | 冲突处理 |
|-------|---------|---------|
| 时间冲突 | CPC中的"累计故事时间"与细纲是否一致 | 优先遵循CPC，记录偏差 |
| 空间冲突 | CPC中的"本章结束地点"与细纲中的"本章起始地点"是否一致 | 如不一致，写场景过渡 |
| 角色冲突 | CPC中的角色状态与细纲是否一致 | 优先遵循CPC |
| 伏笔冲突 | 是否需要在本章回收某个已达时效的伏笔 | 必须在本章处理 |

## 三、章节自动编号与项目管理

> 首次创作时自动创建项目目录结构，后续章节自动编号，无需用户手动管理。
> **重要**：支持用户指定保存路径，如果路径无权限则自动降级到默认路径。

### 项目初始化

首次进入模式C时，自动检测并创建项目结构：

```
检测流程：
1. 获取目标路径（优先级：用户指定 > 当前工作目录 > 默认路径）
2. 检查目标路径是否存在且有写入权限
3. 若无权限 → 降级到默认路径（/Users/chenzhiqiang/Desktop/T/document/novel_1）
4. 检查是否存在 ./正文/ 目录
5. 若不存在 → 创建完整项目结构（正文/、细纲/、大纲/均在目标路径下）
6. 若存在 → 检测已有章节，自动分配下一章编号
```

**路径优先级规则**：

| 优先级 | 路径来源 | 说明 |
|-------|---------|------|
| 1 | 用户指定 | 用户明确提供的绝对路径 |
| 2 | 当前工作目录 | 执行 pwd 获取的目录 |
| 3 | 默认路径 | `/Users/chenzhiqiang/Desktop/T/document/novel_1`（有MCP权限） |

**自动降级机制**：

当用户指定的路径无MCP写入权限时，自动降级到默认路径，并提示用户：

```
⚠️ 路径权限提示
─────────────────────────────────
目标路径: [用户指定路径]
状态: ❌ 无写入权限

已自动降级到默认路径:
/Users/chenzhiqiang/Desktop/T/document/novel_1/[书名]/

文件已成功保存到上述位置。
如需移动到其他位置，请手动操作或使用符号链接。
─────────────────────────────────
```

**自动创建的目录结构**（在最终确定的目标路径下）：
```
[目标路径]/
├── 记忆库.md
├── 大纲/
│   └── [书名]_核心大纲.md
├── 细纲/
│   └── 第一卷_XXX/
│       ├── 第001章_[标题].md
│       └── 细纲目录.md

    └── 第一卷_XXX/
        └── 第001章_[标题].md
```

| 操作 | 行为 |
|------|------|

## 五、前置要求（撰写前必读）

> **核心原则**：大纲信息已通过细纲间接引用，写正文时**严格遵循当前章细纲**即可，无需直接查阅大纲。

### 第一层：上下文加载（第2章起强制）

写第2章及以后章节时，必须完成：

| 步骤 | 执行要求 | 检查点 |
|-----|---------|--------|
| **读取CPC** | 提取上一章CPC中的所有字段 | 是否记录了时间/空间/角色状态/情节/情绪/伏笔/钩子？ |
| **读取结尾** | 读取上一章最后800-1000字 | 是否能感受到原文质感？最后一句话是什么？ |
| **交叉验证** | CPC + 细纲信息对比 | 是否有冲突？如何处理？ |

### 第二层：细纲执行（核心聚焦）

细纲已包含大纲所有关键信息，写作时严格遵循以下指令：

| 细纲要素 | 执行要求 | 检查点 |
|---------|---------|--------|
| **核心事件** | 必须成为正文主线骨架 | 是否偏离细纲一句话剧情？ |
| **阶段拆分** | 3-5个阶段对应正文段落结构 | 字数分配是否符合细纲？ |
| **爽点指令** | 每个爽点必须逐一实现，不得遗漏或淡化 | 四段式结构是否完整？ |
| **伏笔操作** | 回收（FXX）和埋下（FXX）必须精确执行 | 伏笔编号是否正确？ |
| **人物状态** | 表格中每个人物的情绪/状态必须准确呈现 | 是否符合细纲表格？ |
| **感官锚点** | 每个场景的感官锚点必须在正文中体现 | 是否调用非视觉感官？ |
| **大纲引用** | 细纲中标注的大纲模块编号已足够 | 仅在细纲与大纲冲突时回查 |

### 第三层：前文衔接（精简聚焦）

基于CPC和上一章结尾，确认以下衔接点：

| 衔接要素 | 确认内容 | 处理方式 |
|---------|---------|---------|
| **场景终点** | 前一章结尾场景是什么？ | 本章开头场景必须能自然承接 |
| **人物状态** | 主角境界、装备、情绪如何？ | 本章开头状态必须一致 |
| **未解决钩子** | 前一章留下了什么悬念？ | 本章开头必须回应或延续 |
| **文风延续** | 前一章的叙述节奏、语言风格 | 保持文风统一 |

## 六、排版与视觉要求

- **移动端适配**：每段不超过3行（手机屏幕宽度），段落间留空行
- **对话独立**：对话必须独立成段，严禁大段对话堆砌
- **视觉留白**：关键剧情转折处，使用单句成段，制造节奏感

**通用规则**：严格遵循主控文件第二章核心铁律（2.3去AI味、2.4沉浸式体验、2.5数据导向），包括禁忌词汇黑名单、质感描写词库、五感调用、潜台词设计、动作代替标签、语言纯正性等全部约束。

### AI味自动防控规则（v20.8新增）

> 每章正文生成后，必须自动执行AI味检测和优化，无需用户手动触发。

#### 自动生成流程（内置AI味检测）

```
用户请求生成正文
    ↓
生成初稿（遵循所有写作规则）
    ↓
【自动执行】AI味检测（无需用户干预）
    ↓
AI味评分 ≤ 5.0？──是──▶ 直接输出最终稿
    ↓
    否
    ↓
【自动执行】AI味优化（最多2轮）
    ↓
优化后评分 ≤ 5.0？──是──▶ 输出优化后的最终稿
    ↓
    否
    ↓
标记警告，但仍输出（让用户决定）
```

#### AI味检测触发点

| 触发时机 | 检测内容 | 处理方式 |
|---------|---------|---------|
| **每章生成完成后** | 句式多样性、五感平衡、情感层次、Cliché密度 | 自动检测+自动优化 |
| **第1章生成后** | 额外检测世界观铺设、类型特征 | 自动检测+提示优化 |
| **连续创作每章后** | 同上，静默执行 | 仅当AI味>7.0时提示 |
| **用户输入"诊断"时** | 全面检测（包含AI味） | 输出详细报告 |

#### AI味优化自动执行规则

| AI味评分 | 处理策略 | 是否暂停 |
|---------|---------|---------|
| 0-3.0（优秀） | 直接输出，无需优化 | 否 |
| 3.0-5.0（良好） | 轻微优化后输出 | 否 |
| 5.0-7.0（中等） | 重点优化，最多2轮 | 否（连续模式）/ 是（单章模式） |
| 7.0-9.0（严重） | 深度优化，2轮后仍>6.0则暂停 | 是 |
| 9.0-10.0（极重） | 建议重写 | 是 |

#### AI味优化重点项（按优先级）

**P0（必须修复）**：
- [ ] 句式单一：主谓宾结构占比>60%
- [ ] 五感严重失衡：视觉占比>80%
- [ ] 明显Cliché：每千字>3个

**P1（重点修复）**：
- [ ] 情感机械：单一情绪持续>300字
- [ ] 词汇平庸：高频词占比>40%
- [ ] 逻辑过清：缺乏人物思维跳跃性

**P2（建议修复）**：
- [ ] 描写套路化
- [ ] 对话缺乏个性
- [ ] 世界观铺设不足（第1章）

### 元数据隔离规则

大纲/细纲中的人物设定标签、伏笔编号（如V1-014、F01）、元数据标记（如#记忆更新）、细纲指令（如【爽点】）是**作者参考信息**，严禁直接输出到正文。

**过滤清单**（输出前扫描删除）：
- 人物属性标签格式（"姓名。身份。血脉。编号。"）
- 伏笔编号（V1-XXX、FXX、"伏笔：XXX"）
- 元数据标记（#记忆更新、#伏笔回收）
- 细纲指令标记（【爽点】、【钩子】）
- 作者备注（（此处需加强）、（待补充））

**转化示例**：
- ❌ "李铁牛。猎户之子。远古战族血脉。V1-014。"
- ✅ "那少年叫李铁牛，猎户的儿子。林凡的感知扫过他时，察觉到那具普通身躯里沉睡着某种古老而狂暴的力量。"
- ❌ "埋下伏笔V1-014"
- ✅ "李铁牛低头看着掌心那道暗红色的纹路，像一条沉睡的蛇。"

### 心理描写限制

- **限制长度**：单次心理独白严禁超过3行
- **穿插动作**：心理描写必须穿插在动作或对话中，严禁大段静止思考

### 节奏与爽点

遵循主控文件2.5数据导向原则，对照细纲中的爽点指令执行。

### 细纲逐项核对机制

写作中必须实时对照细纲执行，确保核心事件、爽点类型、人物状态、伏笔、场景、境界、时间线与细纲完全对齐。核对流程：写作前读取细纲 → 写作中逐项对照 → 完成后自检。

## 八、章间连贯性校验清单（第2章起强制）

每章写完后，必须执行以下自检：

| 检查维度 | 检查方法 | 通过标准 |
|---------|---------|---------|
| **开头黏合度** | 把本章开头和上章结尾并列阅读 | 包含≥3个黏合元素（时间/空间/情绪/情节/感官） |
| **CPC一致性** | 对照上一章CPC检查 | 时间/空间/角色状态与CPC一致 |
| **情绪连续性** | 画情绪变化曲线 | 无断崖式突变，有明确过渡 |
| **钩子处理** | 检查上一章钩子 | 本章开头必须回应或延续钩子 |
| **伏笔回收** | 对照CPC伏笔清单 | 即时型100%，短程型≥80% |
| **时间一致性** | 对照CPC时间锚点 | 无时间bug，累计时间正确 |
| **角色状态一致** | 逐项对照角色状态表 | 无"自动痊愈"，物品不凭空出现/消失 |

**未通过校验的处理**：
- 开头黏合元素不足 → 重写开头，强制添加黏合元素
- 角色状态不一致 → 修改正文，使其与CPC一致
- 钩子未处理 → 在开头添加钩子回应

## 九、情节偏差累积检测机制（v20.9新增）

> **核心目的**：解决AI长篇创作中情节逐渐偏离预设大纲的问题，通过累计偏差追踪实现早发现、早预警。

### 9.1 检测维度与指标

| 维度 | 指标 | 计算方法 | 预警阈值 |
|------|------|---------|---------|
| **核心事件偏离** | 本章实际核心事件与细纲的匹配度 | AI语义相似度对比 | <70% |
| **爽点偏差** | 实际爽点类型与细纲的差异 | 爽点类型匹配数量 | >2种不匹配 |
| **伏笔操作偏差** | 未按计划埋设/回收伏笔 | 对比伏笔操作清单 | 任意遗漏 |
| **角色行为偏差** | 角色行为与设定不符（OOC） | 人物设定知识库匹配 | 任意OOC |
| **场景偏差** | 实际场景与细纲设定不一致 | 场景描述相似度 | <60% |

### 9.2 单章偏差值计算

```
单章偏差值 = (核心事件偏离度×40% + 爽点偏差度×25% + 伏笔偏差度×20% + 角色行为偏差度×10% + 场景偏差度×5%) × 100
```

### 9.3 累积偏差追踪

每章完成后自动计算并累积偏差：

| 累积偏差范围 | 预警级别 | 响应策略 |
|-------------|---------|---------|
| 0-20% | ✅ 正常 | 无需干预 |
| 20%-30% | ⚠️ 轻度预警 | 提示作者检查情节方向 |
| 30%-50% | ⚠️ 中度预警 | 自动生成情节修正建议 |
| 50%-70% | 🔴 重度预警 | 暂停创作，要求人工确认 |
| >70% | 🔴 严重预警 | 强制暂停，需重写调整 |

### 9.4 累积偏差计算规则

```
累积偏差 = (最近5章偏差值之和 / 5) × 衰减系数
衰减系数 = 0.9^(距离当前章节数)  // 越远的章节权重越低
```

### 9.5 预警输出格式

**轻度预警**：
```
⚠️ 情节偏差预警（第023章）
当前累积偏差：24%（轻度）
最近偏差趋势：↗上升

建议：请检查本章核心事件是否符合细纲设定
```

**中度预警**：
```
⚠️ 情节偏差预警（第035章）
当前累积偏差：38%（中度）
最近偏差趋势：↗上升

偏差详情：
1. 核心事件匹配度：62%（细纲：[原核心事件]，实际：[实际内容]）
2. 爽点类型不匹配：[原爽点类型] → [实际爽点类型]

修正建议：
1. 调整第035章核心事件，使其回归细纲主线
2. 在下一章（第036章）增加对细纲关键节点的呼应
```

**重度预警**：
```
🔴 情节严重偏离（第048章）
当前累积偏差：56%（重度）
最近偏差趋势：↗急剧上升

已自动暂停创作，请确认：
1. 是否继续当前方向（输入"继续"）
2. 是否需要回归细纲（输入"回归"，将自动生成修正建议）
3. 是否需要重新审视大纲（输入"重审大纲"）
```

### 9.6 情节修正建议生成

当检测到中度及以上偏差时，自动生成修正建议：

| 偏差类型 | 修正策略 |
|---------|---------|
| 核心事件偏离 | 建议在下一章增加对原细纲关键节点的呼应 |
| 爽点类型偏差 | 建议调整爽点呈现方式，使其符合细纲要求 |
| 伏笔遗漏 | 建议在后续章节补设或提前回收相关伏笔 |
| 角色OOC | 建议修正角色行为，使其符合人物设定 |
