# 模式C子模块：章节版本管理

> 按需加载：涉及版本对比、回退、清理时加载本文件。
> 全局规则：遵循主控文件第二章核心铁律

---

## 一、自动版本记录

| 触发条件 | 行为 |
|---------|------|
| 每次修改已有章节 | 自动将当前版本保存为 `第XXX章_<标题>_v<N>.md` |
| 首次保存 | 版本号为v1 |
| 再次修改 | 版本号递增（v2, v3...） |
| 用户输入 `保存版本` | 手动创建当前版本的快照 |

**版本文件命名**：`第XXX章_<标题>_v<N>.md`
**版本文件路径**：`./正文/第X卷_XXX/版本历史/`

## 二、版本对比

输入 `对比第X章 v1 v2` 触发版本对比：

```markdown

| 段落数 | <<N>> | <<N>> | <<±N>> |
| 爽点数 | <<N>> | <<N>> | <<±N>> |
| 对话占比 | <<X%>> | <<X%>> | <<±X%>> |

> <新增内容摘要>

<基于对比分析，推荐保留哪个版本及理由>
```

## 三、版本回退

| 命令 | 行为 |
|------|------|
| `回退第X章到v2` | 将v2版本恢复为当前版本，当前版本自动保存为新版本 |
| `撤销回退` | 撤销最近一次回退操作 |
| `确认版本` | 确认当前版本为最终版本，清理更早版本 |

## 四、版本清理

| 命令 | 行为 |
|------|------|
| `清理版本` | 每章保留最近3个版本，删除更早版本 |
| `清理全部版本` | 每章仅保留当前版本，删除所有历史版本（需确认） |
| `归档版本` | 将所有历史版本打包归档到 `./正文/版本归档/` |

## 五、分支创作

> 支持从任意章节创建分支版本，探索不同剧情走向。分支间可独立发展，满意后合并回主干。

### 5.1 创建分支

| 命令 | 行为 |
|------|------|
| `从第X章创建分支 <分支名>` | 以第X章为起点创建新分支，分支独立发展 |
| `查看分支` | 列出所有分支及状态 |
| `切换到分支 <分支名>` | 切换到指定分支继续创作 |

**分支命名**：`分支_<类型>_<描述>`，如：`分支_结局_主角黑化`、`分支_感情线_双女主`

### 5.2 分支类型

| 分支类型 | 说明 | 示例 |
|---------|------|------|
| **结局分支** | 探索不同结局走向 | `分支_结局_悲剧` `分支_结局_开放` |
| **感情线分支** | 探索不同CP走向 | `分支_感情线_女二上位` |
| **剧情分支** | 探索不同剧情发展 | `分支_剧情_主角不原谅` |
| **风格分支** | 探索不同写作风格 | `分支_风格_暗黑向` |

### 5.3 分支管理

```markdown
## 分支状态

| 分支名 | 起点章节 | 当前章节 | 状态 | 创建日期 |
|--------|:---:|:---:|------|---------|
| 主干 | — | 第50章 | ⏳进行中 | — |
| 分支_结局_主角黑化 | 第40章 | 第48章 | ⏳进行中 | 2026-05-01 |
| 分支_感情线_双女主 | 第30章 | 第35章 | ⏸️暂停 | 2026-04-20 |
| 分支_结局_开放 | 第45章 | 第50章 | ✅已完成 | 2026-04-15 |
```

## 七、合并回主干

### 7.1 合并操作

| 命令 | 行为 |
|------|------|
| `合并分支 <分支名> 到主干` | 将分支内容合并到主干，自动处理章节编号顺延 |
| `预览合并 <分支名>` | 预览合并后的章节结构，不实际执行 |
| `放弃分支 <分支名>` | 删除分支（需确认） |

### 7.2 合并策略

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **完全替换** | 用分支内容完全替换主干对应章节 | 分支质量明显优于主干 |
| **选择性合并** | 逐章选择保留主干还是分支版本 | 分支部分章节优于主干 |
| **融合合并** | 将分支的亮点元素融入主干 | 分支有可取之处但整体不如主干 |

### 7.3 合并后处理

```markdown
## 合并完成：分支_结局_主角黑化 → 主干

### 合并摘要
- 替换章节：第41-48章（8章）
- 保留主干章节：第1-40章
- 新增章节编号：第41-48章（自动顺延）

### 影响分析
| 影响维度 | 变化 |
|---------|------|
| 总章节数 | 50→50（不变） |
| 总字数 | +3,200字 |
| 伏笔状态 | 新增F21-F23，F12-F15需重新规划回收 |
| 人物关系 | 主角-萧天：同盟→敌对 |

### 后续操作建议
1. 检查第40-41章衔接是否自然
2. 重新规划F12-F15的回收方案
3. 更新人物关系矩阵
```

## 八、创作快照

### 8.1 自动快照

| 触发条件 | 快照内容 |
|---------|---------|
| 每卷完结 | 全作品快照（大纲+细纲+正文+记忆库） |
| 大高潮完成 | 当前进度快照（正文+伏笔状态+人物关系） |
| 重要转折完成 | 当前进度快照 |
| 每50章 | 全作品快照 |

### 8.2 快照管理

| 命令 | 行为 |
|------|------|
| `创建快照 <描述>` | 手动创建当前进度的快照 |
| `查看快照` | 列出所有快照及时间 |
| `恢复到快照 <快照名>` | 将整个项目恢复到指定快照状态（需确认） |
