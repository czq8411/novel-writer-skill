# 模式C子模块：章节版本管理

> 按需加载：涉及版本对比、回退、清理时加载本文件。
> 全局规则：遵循主控文件第二章核心铁律

---

## 自动版本记录

| 触发条件 | 行为 |
|---------|------|
| 每次修改已有章节 | 自动将当前版本保存为 `第XXX章_[标题]_v[N].md` |
| 首次保存 | 版本号为v1 |
| 再次修改 | 版本号递增（v2, v3...） |
| 用户输入 `保存版本` | 手动创建当前版本的快照 |

**版本文件命名**：`第XXX章_[标题]_v[N].md`
**版本文件路径**：`./正文/第X卷_XXX/版本历史/`

## 版本对比

输入 `对比第X章 v1 v2` 触发版本对比：

```markdown
## 版本对比：第X章 v1 → v2

### 变更概览
| 指标 | v1 | v2 | 变化 |
|------|----|----|------|
| 总字数 | [N] | [N] | [±N] |
| 段落数 | [N] | [N] | [±N] |
| 爽点数 | [N] | [N] | [±N] |
| 对话占比 | [X%] | [X%] | [±X%] |

### 新增段落（[N]处）
> [新增内容摘要]

### 删除段落（[N]处）
> [删除内容摘要+删除原因推测]

### 修改段落（[N]处）
| 位置 | v1原文 | v2修改 | 变化类型 |
|------|--------|--------|---------|
| [段落位置] | [原文摘要] | [修改摘要] | [润色/扩写/删减/重写] |

### 版本选择建议
[基于对比分析，推荐保留哪个版本及理由]
```

## 版本回退

| 命令 | 行为 |
|------|------|
| `回退第X章到v2` | 将v2版本恢复为当前版本，当前版本自动保存为新版本 |
| `撤销回退` | 撤销最近一次回退操作 |
| `确认版本` | 确认当前版本为最终版本，清理更早版本 |

## 版本清理

| 命令 | 行为 |
|------|------|
| `清理版本` | 每章保留最近3个版本，删除更早版本 |
| `清理全部版本` | 每章仅保留当前版本，删除所有历史版本（需确认） |
| `归档版本` | 将所有历史版本打包归档到 `./正文/版本归档/` |

## 版本管理目录结构

```
./正文/第X卷_XXX/
├── 正文/
│   └── 第001章_[标题].md          ← 当前版本
├── 版本历史/
│   ├── 第001章_[标题]_v1.md       ← 初版
│   ├── 第001章_[标题]_v2.md       ← 第一次修改
│   └── 第001章_[标题]_v3.md       ← 第二次修改
└── 版本归档/                       ← 清理后的归档
    └── 归档_2026-05-12/
```
