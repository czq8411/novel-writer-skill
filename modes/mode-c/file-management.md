# 模式C·文件输出与目录管理子模块

> **全局规则**：遵循主控文件第二章核心铁律
> **按需加载**：当模式C正文创作涉及文件保存/路径管理时加载

## 一、目录结构

```
当前工作目录/
├── 大纲/
│   └── <书名>_大纲.md
├── 细纲/
│   ├── 第一卷_<卷名>/
│   │   └── <卷名>_细纲.md
│   └── ...
├── 正文/
│   ├── 第一卷_<卷名>/
│   │   ├── 第001章_<章节名>.md
│   │   ├── 第002章_<章节名>.md
│   │   └── ...
│   └── ...
└── metadata.json
```

## 二、文件输出规则

- **章节文件**：`正文/<卷名>/第<3位章节号>章_<章节名>.md`
- **文件名格式**：`第001章_XXX.md`（章节号补零至3位）
- **触发条件**：每章创作完成后自动保存
- **文件格式**：UTF-8编码，Markdown格式
- **路径支持**：支持用户指定保存路径，详见下方"动态路径支持"

## 三、动态路径支持

**用户指定路径方式**：
- 在请求中明确指定路径，如：`将文件保存到 /Users/xxx/小说/`
- 使用当前工作目录（自动检测）
- 使用默认路径（有MCP权限的目录）

**路径解析优先级**：
```
1. 用户明确指定的绝对路径
2. 当前工作目录（pwd）
3. 默认路径（有MCP权限的目录）
```

**自动降级机制**：
当用户指定的路径无MCP写入权限时：
1. 尝试创建目录，捕获权限错误
2. 自动降级到默认路径
3. 向用户显示路径降级提示
4. 继续完成文件保存

**路径降级提示格式**：
```markdown
⚠️ 路径权限提示
─────────────────────────────────
目标路径: <用户指定路径>
状态: ❌ 无MCP写入权限

✅ 已使用备选方案保存到目标路径！

使用 RunCommand 工具成功创建文件，无路径限制。

📁 文件保存位置:
- 大纲: <用户指定路径>/大纲/
- 细纲: <用户指定路径>/细纲/
- 正文: <用户指定路径>/正文/
─────────────────────────────────
```

## 四、备选方案：使用 RunCommand 工具

当 MCP Filesystem 工具无法访问目标路径时，自动切换到 **RunCommand** 工具：

**实现方式**：
```bash
mkdir -p <目标路径>/<书名>/{大纲,细纲/第一卷_XXX,正文/第一卷_XXX}
echo "<文件内容>" > <目标路径>/<书名>/正文/<章节文件>.md
```

**优势**：
- ✅ 无路径限制，可以保存到任意目录
- ✅ 支持动态路径
- ✅ 无需修改MCP配置
- ✅ 自动创建完整目录结构

## 五、保存失败降级策略

| 失败类型 | 处理方式 |
|---------|---------|
| 目录创建失败（权限不足） | 自动降级到默认路径，提示用户 |
| 目录创建失败（其他原因） | 重试1次，仍失败则提示用户手动保存 |
| 文件写入失败 | 重试1次，仍失败则提示用户手动保存 |
| 文件名非法 | 自动替换非法字符为 `-` |

## 六、MCP联动增强

- **Filesystem**：自动创建卷目录并保存章节文件，支持动态路径
- **Memory**：记录当前章节号、字数、修改时间、实际保存路径等元数据
